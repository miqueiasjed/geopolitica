## Especificação

### O que fazer:

**`EleicaoController`:**

`GET /api/eleicoes` — lista de eleições:
- Parâmetros: `?ano=2026` (obrigatório, padrão: ano atual), `?relevancia=alta` (opcional)
- Cache Redis 1h: `Cache::remember("eleicoes_{ano}_{relevancia}", 3600, fn() => ...)`
- Aplicar scopes `porAno` e `porRelevancia` conforme parâmetros
- Ordenar por `data_eleicao ASC`
- Retornar: `id`, `pais`, `codigo_pais`, `data_eleicao`, `tipo_eleicao`, `relevancia`

`GET /api/eleicoes/{id}` — detalhe:
- Cache Redis 1h: `Cache::remember("eleicao_{id}", 3600, fn() => ...)`
- Retornar todos os campos da `Eleicao`
- 404 se não encontrado

**`CriarEleicaoRequest`:**
```php
'pais' => 'required|string|max:100',
'codigo_pais' => 'required|string|size:2',
'data_eleicao' => 'required|date',
'tipo_eleicao' => 'required|string|max:100',
'relevancia' => 'required|in:alta,media,baixa',
'contexto_geopolitico' => 'required|string',
'impacto_brasil' => 'required|string',
'candidatos_principais' => 'nullable|array',
'candidatos_principais.*.nome' => 'required|string',
'candidatos_principais.*.partido' => 'nullable|string',
'content_slug' => 'nullable|string|max:255',
```

**`AtualizarEleicaoRequest`:**
- Mesmas regras com `sometimes` em todos os campos
- Herdar de `CriarEleicaoRequest` ou duplicar com `sometimes`

**`EleicaoAdminController` (CRUD completo):**

`POST /api/admin/eleicoes`:
- Usar `CriarEleicaoRequest`
- Criar eleição
- Invalidar cache: `Cache::forget("eleicoes_{ano}_*")` (tags ou flush seletivo)
- Retornar 201 com a eleição criada

`PATCH /api/admin/eleicoes/{id}`:
- Usar `AtualizarEleicaoRequest`
- Atualizar e invalidar cache da eleição e da lista
- Retornar 200 com a eleição atualizada

`DELETE /api/admin/eleicoes/{id}`:
- Deletar e invalidar cache
- Retornar 204

**Rotas em `routes/api.php`:**
```php
// Rotas públicas autenticadas
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/eleicoes', [EleicaoController::class, 'index']);
    Route::get('/eleicoes/{id}', [EleicaoController::class, 'show']);
});

// Rotas admin
Route::middleware(['auth:sanctum', 'role:admin'])->prefix('admin')->group(function () {
    Route::post('/eleicoes', [EleicaoAdminController::class, 'store']);
    Route::patch('/eleicoes/{id}', [EleicaoAdminController::class, 'update']);
    Route::delete('/eleicoes/{id}', [EleicaoAdminController::class, 'destroy']);
});
```

### Regras de negócio aplicáveis:
- Cache 1h para lista e detalhe
- Invalidar cache após CREATE/UPDATE/DELETE
- Rotas admin protegidas por `role:admin` (Spatie Permission)
- Ano padrão = ano atual se não informado

### Critérios de aceitação:
- [ ] `GET /api/eleicoes?ano=2026` retorna 15 eleições do seed
- [ ] `GET /api/eleicoes?ano=2026&relevancia=alta` filtra corretamente
- [ ] `POST /api/admin/eleicoes` cria nova eleição (admin autenticado)
- [ ] `PATCH /api/admin/eleicoes/{id}` atualiza campos parcialmente
- [ ] `DELETE /api/admin/eleicoes/{id}` remove e retorna 204
- [ ] Usuário não-admin recebe 403 nas rotas admin
- [ ] Cache Redis com TTL 1h funcionando
