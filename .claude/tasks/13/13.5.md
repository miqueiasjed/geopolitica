## Especificação

### O que fazer:

**`EleicaoDetailPanel.tsx`:**
Props: `eleicaoId: number | null`, `onClose: () => void`
- Query interna: `useQuery(['eleicao-detalhe', eleicaoId], ...)` fetch `GET /api/eleicoes/{id}`
- `enabled: !!eleicaoId`
- Painel deslizante da direita (Framer Motion, mesma animação do `CriseDetailPanel` do M07):
  ```tsx
  initial={{ x: '100%' }} animate={{ x: 0 }} exit={{ x: '100%' }}
  transition={{ type: 'spring', damping: 25, stiffness: 200 }}
  className="fixed right-0 top-0 h-full w-[440px] bg-[#1C1F26] overflow-y-auto z-50"
  ```
- Overlay clicável para fechar
- Skeleton enquanto carrega
- **Conteúdo:**
  - Header: bandeira emoji + nome do país + badge de relevância
  - Data e tipo da eleição formatados
  - Candidatos principais (se houver): lista com nome + partido
  - Contexto geopolítico (texto)
  - Impacto no Brasil (texto com destaque visual diferente)
  - Botão "Ver na Biblioteca →" se `content_slug` presente

**`AdminEleicaoForm.tsx`:**
Props: `eleicao?: EleicaoDetalhe` (undefined = modo criação), `onSuccess: () => void`, `onCancel: () => void`

Campos do formulário:
- `pais` (input text)
- `codigo_pais` (input text, maxLength 2, uppercase automático)
- `data_eleicao` (input date)
- `tipo_eleicao` (input text com sugestões: "Presidencial", "Parlamentar", "Referendo", "Municipal", "Legislativa")
- `relevancia` (select: Alta / Média / Baixa)
- `contexto_geopolitico` (textarea, linhas 4)
- `impacto_brasil` (textarea, linhas 3)
- `candidatos_principais` (campo dinâmico: lista de pares nome+partido, botões + Adicionar / × Remover)
- `content_slug` (input text, opcional)
- Botões: "Salvar" (submit) + "Cancelar"

Mutations:
- Criação: `POST /api/admin/eleicoes`
- Edição: `PATCH /api/admin/eleicoes/{id}`
- Invalidar `['eleicoes']` após sucesso
- Chamar `onSuccess()` após salvar

**`AdminEleicoes.tsx` (página admin):**
- Header: "Gerenciar Eleições" + botão "Nova Eleição"
- Tabela com colunas: País, Data, Tipo, Relevância (badge), Ações (Editar | Excluir)
- `useEleicoes({ ano: anoAtual })` + select de ano para filtrar
- Ao clicar "Nova Eleição": exibir `AdminEleicaoForm` em modal ou drawer
- Ao clicar "Editar": buscar detalhe e exibir `AdminEleicaoForm` pré-preenchido
- Ao clicar "Excluir": confirmation dialog → `DELETE /api/admin/eleicoes/{id}` + invalidar query

**`RadarEleicoes.tsx` (página principal):**
Estado local: `filtros: FiltrosEleicao = { ano: anoAtual }`, `eleicaoIdSelecionada: number | null`
- `useEleicoes(filtros)` para buscar dados
- `EleicaoFilterBar` no topo
- `RadarGrid` com dados e callback de clique
- `EleicaoDetailPanel` com `eleicaoId={eleicaoIdSelecionada}` e `onClose`
- Skeleton durante carregamento
- Contadores por relevância: "X alta | Y média | Z baixa"

**Rotas:**
```tsx
{ path: '/dashboard/eleicoes', element: <RadarEleicoes /> }
{ path: '/admin/eleicoes', element: <AdminEleicoes /> } // dentro do layout admin
```

**`TopNav.tsx`:**
- Adicionar link "Radar de Eleições" com ícone de calendário/urna
- Rota: `/dashboard/eleicoes`

### Regras de negócio aplicáveis:
