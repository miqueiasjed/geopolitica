## Especificação

### O que fazer:

**`src/types/eleicao.ts`:**
```typescript
export type RelevanciaEleicao = 'alta' | 'media' | 'baixa';

export interface CandidatoPrincipal {
  nome: string;
  partido?: string;
}

export interface Eleicao {
  id: number;
  pais: string;
  codigo_pais: string;
  data_eleicao: string;
  tipo_eleicao: string;
  relevancia: RelevanciaEleicao;
}

export interface EleicaoDetalhe extends Eleicao {
  contexto_geopolitico: string;
  impacto_brasil: string;
  candidatos_principais: CandidatoPrincipal[];
  content_slug: string | null;
  created_at: string;
  updated_at: string;
}

export interface FiltrosEleicao {
  ano: number;
  relevancia?: RelevanciaEleicao;
}

// Mapa de cores por relevância
export const CORES_RELEVANCIA: Record<RelevanciaEleicao, string> = {
  alta: '#EF4444',    // vermelho
  media: '#FACC15',   // amarelo
  baixa: 'rgba(232, 228, 220, 0.2)', // branco 20%
};
```

**`useEleicoes`:**
- Parâmetros: `filtros: FiltrosEleicao`
- Query key: `['eleicoes', filtros]`
- Fetch: `GET /api/eleicoes?ano={ano}&relevancia={relevancia}`
- `staleTime: 60 * 60 * 1000` (1h — consistente com cache Redis)
- Retornar: `{ eleicoes: Eleicao[], isLoading, error }`
- Nota: `relevancia` é opcional — omitir da URL se não definida

**`EleicaoFilterBar.tsx`:**
Props: `filtros: FiltrosEleicao`, `onChange: (filtros: FiltrosEleicao) => void`

Dois controles:
1. **Select de ano:** opções de 2024 até 2028
   - Valor atual: `filtros.ano`
   - Ao mudar: `onChange({ ...filtros, ano: Number(value) })`
   - Estilo: select com borda dourada no foco

2. **Toggle buttons de relevância:** "Todas" | "Alta" | "Média" | "Baixa"
   - "Todas" → limpa o filtro de relevância (`relevancia: undefined`)
   - Cada botão tem a cor correspondente ao nível (vermelho, amarelo, branco)
   - Botão ativo: borda sólida + fundo levemente mais claro
   - Ao clicar: `onChange({ ...filtros, relevancia: value })`

Layout: flex row, gap-4, alinhado à esquerda

### Regras de negócio aplicáveis:
- Ano padrão = ano atual (2026)
- Filtro de relevância é opcional
- `CORES_RELEVANCIA` exportado para reutilização em outros componentes

### Critérios de aceitação:
- [ ] Types TypeScript sem erros de compilação
- [ ] `useEleicoes({ ano: 2026 })` busca sem filtro de relevância
- [ ] `useEleicoes({ ano: 2026, relevancia: 'alta' })` inclui `?relevancia=alta` na URL
- [ ] `EleicaoFilterBar` renderiza select + 4 botões de toggle
- [ ] Botão "Todas" limpa filtro de relevância
- [ ] Botão ativo visualmente diferenciado
