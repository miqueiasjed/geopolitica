# Task 31.1 – Migration de Unificação de Roles + RolesSeeder

## Objetivo
Criar migration que converte `assinante_essencial`, `assinante_pro` e `assinante_reservado` → `assinante` na tabela `model_has_roles` do Spatie, e atualizar o `RolesSeeder` para criar apenas a role `assinante`.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/YYYY_MM_DD_unificar_role_assinante.php` → criar
- `database/seeders/RolesSeeder.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `database/seeders/RolesSeeder.php` → ver roles existentes criadas e permissões associadas
- `app/Providers/AuthServiceProvider.php` → verificar se há referências às roles específicas no boot

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
1. Criar migration idempotente:
   - Obter o ID da role `assinante` (criar se não existir).
   - Para cada usuário que tenha `assinante_essencial`, `assinante_pro` ou `assinante_reservado`:
     - Inserir `assinante` em `model_has_roles` (se ainda não tiver).
     - Remover as três roles específicas.
   - Remover as três roles específicas da tabela `roles`.
   - Limpar cache de permissões: `app()['cache']->forget('spatie.permission.cache')`.
2. Atualizar `RolesSeeder`:
   - Remover criação de `assinante_essencial`, `assinante_pro`, `assinante_reservado`.
   - Garantir criação de `assinante` (sem permissões específicas — controle é por `assinantes.plano`).

### Regras de negócio aplicáveis:
- A migration deve ser segura para produção: idempotente, não deve falhar se já foi parcialmente executada.
- Um usuário não pode ter a mesma role duas vezes (use `firstOrCreate` ou `insertOrIgnore`).
- Spatie usa `guard_name = 'web'` por padrão — verificar na tabela `roles`.

### Critérios de aceitação:
- [ ] `php artisan migrate` executa sem erro em banco limpo e em banco com dados reais.
- [ ] Após a migration, nenhum registro com `assinante_essencial/pro/reservado` existe em `model_has_roles`.
- [ ] Role `assinante` existe na tabela `roles`; as 3 específicas não existem.
- [ ] `php artisan db:seed --class=RolesSeeder` recria apenas `assinante` sem erros.

## Teste esperado
- Comando: `php artisan migrate --pretend` para revisão visual antes de aplicar.
- Verificação manual: `select * from roles where name like 'assinante%'` deve retornar apenas `assinante`.
