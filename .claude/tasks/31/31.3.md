# Task 31.3 – Refactor ImportacaoAssinantesService + TrocarPlanoAssinanteJob + ImportarAddonsJob

## Objetivo
Atualizar os três componentes de importação/processamento em background para atribuírem a role `assinante` em vez de roles específicas por plano.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/ImportacaoAssinantesService.php` → modificar
- `app/Jobs/TrocarPlanoAssinanteJob.php` → modificar
- `app/Jobs/ImportarAddonsJob.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `app/Services/ImportacaoAssinantesService.php` → ver onde atribui role (mapa de roles específicas)
- `app/Jobs/TrocarPlanoAssinanteJob.php` → entender lógica de troca de roles por plano
- `app/Jobs/ImportarAddonsJob.php` → linha ~102 onde gera `assinante_`+plano

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**ImportacaoAssinantesService:**
1. Localizar o mapa de roles (ex: `['essencial' => 'assinante_essencial', ...]`).
2. Substituir por atribuição direta: `$usuario->syncRoles(['assinante'])`.
3. Garantir que `assinantes.plano` continua sendo preenchido corretamente (não alterar).

**TrocarPlanoAssinanteJob:**
1. Remover qualquer lógica que faz `removeRole('assinante_antigo')` + `assignRole('assinante_novo')`.
2. Simplificar para apenas garantir que o usuário tem a role `assinante`:
   ```php
   $usuario->syncRoles(['assinante']);
   ```
3. A atualização de `assinantes.plano` já ocorre antes — só confirmar que a role está correta.

**ImportarAddonsJob:**
1. Localizar linha ~102 onde gera `'assinante_' . $plano` ou similar.
2. Substituir por `'assinante'`.
3. Usar `$usuario->syncRoles(['assinante'])`.

### Regras de negócio aplicáveis:
- A importação CSV/XLSX pode criar usuários novos — eles devem receber a role `assinante` independentemente do plano indicado na planilha.
- O plano continua sendo salvo em `assinantes.plano`; a role só indica "é assinante".
- Jobs devem ser idempotentes: chamar `syncRoles(['assinante'])` em usuário que já tem a role não causa problema.

### Critérios de aceitação:
- [ ] `grep -r "assinante_" app/Services/ImportacaoAssinantesService.php app/Jobs/TrocarPlanoAssinanteJob.php app/Jobs/ImportarAddonsJob.php` retorna vazio.
- [ ] Usuário importado via CSV recebe role `assinante` e plano correto em `assinantes.plano`.
- [ ] `TrocarPlanoAssinanteJob` não contém lógica de swap de roles por nome de plano.

## Teste esperado
- Verificação manual via tinker:
  ```php
  $job = new TrocarPlanoAssinanteJob($usuario, 'pro');
  $job->handle();
  $usuario->hasRole('assinante'); // true
  ```
