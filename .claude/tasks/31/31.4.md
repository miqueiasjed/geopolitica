# Task 31.4 – Refactor AdminAssinanteService + EntregaAlertaService

## Objetivo
Corrigir `AdminAssinanteService` para usar `'assinante'` fixo em vez de derivar role do plano, e atualizar `EntregaAlertaService` para filtrar destinatários via `assinante.plano` em vez de role Spatie.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/AdminAssinanteService.php` → modificar
- `app/Services/EntregaAlertaService.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `app/Services/AdminAssinanteService.php` → ver linha ~20 com `$plano?->role ?? ('assinante_'.$novoPlano)`
- `app/Services/EntregaAlertaService.php` → ver uso de `role('assinante_reservado')` para filtrar destinatários

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**AdminAssinanteService:**
1. Localizar linha ~20: `$plano?->role ?? ('assinante_'.$novoPlano)`.
2. Substituir por `'assinante'` (string fixa).
3. Verificar se há outros pontos no arquivo que derivam role a partir do plano e corrigir.

**EntregaAlertaService:**
1. Localizar o ponto que usa `role('assinante_reservado')` para buscar usuários destinatários.
2. Substituir por query via `assinante.plano`:
   ```php
   // Antes:
   $usuarios = User::role('assinante_reservado')->get();
   
   // Depois:
   $usuarios = User::whereHas('assinante', fn($q) => $q->where('plano', 'reservado'))
                   ->whereHas('roles', fn($q) => $q->where('name', 'assinante'))
                   ->get();
   ```
3. Verificar se há outros filtros por role específica de assinante no arquivo.

### Regras de negócio aplicáveis:
- Apenas usuários com plano `reservado` em `assinantes.plano` devem receber alertas do nível reservado.
- A role `assinante` é necessária mas não suficiente — ainda é preciso verificar `assinante.plano`.
- O relacionamento `User::assinante` já existe e é `hasOne(Assinante::class)`.

### Critérios de aceitação:
- [ ] `AdminAssinanteService` não contém `assinante_essencial`, `assinante_pro`, `assinante_reservado`.
- [ ] `EntregaAlertaService` não usa `role('assinante_reservado')` — usa `whereHas('assinante', ...)`.
- [ ] Usuário com plano `reservado` em `assinantes.plano` é selecionado como destinatário de alertas reservados.

## Teste esperado
