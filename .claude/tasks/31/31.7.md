# Task 31.7 – Atualizar Commands + DadosProducaoFakeSeeder

## Objetivo
Corrigir os dois Artisan Commands (`CorrigirRolesAssinantesSemRole` e `CorrigirPlanoAddonOnly`) e o `DadosProducaoFakeSeeder` para usarem a role `assinante` em vez de roles específicas por plano.

- `app/Console/Commands/CorrigirRolesAssinantesSemRole.php` → modificar
- `app/Console/Commands/CorrigirPlanoAddonOnly.php` → modificar
- `database/seeders/DadosProducaoFakeSeeder.php` → modificar

## Contexto necessário (arquivos para ler antes)

- `app/Console/Commands/CorrigirPlanoAddonOnly.php` → ver lógica similar
- `database/seeders/DadosProducaoFakeSeeder.php` → ver linhas 55–57, 76, 86, 96

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**CorrigirRolesAssinantesSemRole:**
1. Este command busca usuários sem role de assinante e os corrige.
2. Atualizar a lógica para:
   - Verificar se o usuário tem a role `assinante` (não mais as específicas).
   - Se não tiver, atribuir `assinante` (independentemente do plano em `assinante.plano`).
3. Remover qualquer mapeamento `plano → role`.

**CorrigirPlanoAddonOnly:**
1. Idem: atualizar para atribuir `assinante` em vez de derivar role do plano.
2. Verificar se o command ainda faz sentido após a unificação — ajustar a mensagem de log se necessário.

**DadosProducaoFakeSeeder:**
1. Linhas 55–57, 76, 86, 96: substituir `assignRole('assinante_essencial')`, `assignRole('assinante_pro')`, `assignRole('assinante_reservado')` por `syncRoles(['assinante'])`.
2. O seeder continua variando os valores de `assinantes.plano` (`essencial`, `pro`, `reservado`) — apenas a role muda para `assinante` em todos os casos.

### Regras de negócio aplicáveis:
- O command de correção deve ser idempotente — rodar duas vezes não causa duplicação.
- O seeder deve gerar dados de teste realistas: usuários com planos diferentes mas todos com role `assinante`.

### Critérios de aceitação:
- [ ] `php artisan assinantes:corrigir-roles` atribui role `assinante` a usuários sem role, sem referenciar roles específicas.
- [ ] `php artisan assinantes:corrigir-plano-addon` idem.
- [ ] `DadosProducaoFakeSeeder` usa apenas `assinante` como role (nenhuma das 3 específicas).
- [ ] `grep "assinante_" app/Console/Commands/ database/seeders/DadosProducaoFakeSeeder.php` retorna vazio.

## Teste esperado
- Executar os commands em ambiente de dev e verificar output: `php artisan assinantes:corrigir-roles --dry-run` (se suportado) ou com banco de teste.
- Verificar resultado: `select * from model_has_roles join roles on roles.id = model_has_roles.role_id where roles.name like 'assinante%'` — apenas `assinante`.
