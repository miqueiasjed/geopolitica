# Task 31.8 – Atualizar Testes de Webhook e Admin (HotmartHandlerTest + WebhookLastlinkTest + AdminHotmartTest)

## Objetivo
Atualizar os três arquivos de teste para substituir atribuições e assertions de roles específicas (`assinante_essencial/pro/reservado`) por `assinante`.

## Tipo
teste

## Arquivos a criar/modificar
- `tests/Feature/HotmartHandlerTest.php` → modificar
- `tests/Feature/WebhookLastlinkTest.php` → modificar
- `tests/Feature/AdminHotmartTest.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `tests/Feature/HotmartHandlerTest.php` → ver todas as assertions de role e fixtures de usuário
- `tests/Feature/WebhookLastlinkTest.php` → idem para Lastlink
- `tests/Feature/AdminHotmartTest.php` → ver criação de usuários de teste com roles específicas

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Para cada arquivo:
1. Localizar todas as ocorrências de `assinante_essencial`, `assinante_pro`, `assinante_reservado` (em `assignRole`, `hasRole`, `assertHasRole`, fixtures, factories).
2. Substituir por `assinante`.
3. Verificar se algum teste valida o *nome da role como output do endpoint* — atualizar o valor esperado.
4. Se houver fixtures de usuário criados com `Role::create(['name' => 'assinante_pro'])` etc., substituir por `Role::findOrCreate('assinante')`.
5. Testes que verificavam "usuário com plano X deve ter role assinante_X" passam a verificar "usuário com plano X deve ter role assinante".

### Regras de negócio aplicáveis:
- Manter a lógica do teste intacta — apenas o nome da role muda.
- Se um teste verificava que dois usuários com planos diferentes tinham roles diferentes, isso deixa de ser verdade — ambos agora têm `assinante`. Atualizar a assertion.
- Não adicionar novos testes nesta task — apenas corrigir os existentes.

### Critérios de aceitação:
- [ ] `php artisan test --filter=HotmartHandlerTest` passa sem erros.
- [ ] `php artisan test --filter=WebhookLastlinkTest` passa sem erros.
- [ ] `php artisan test --filter=AdminHotmartTest` passa sem erros.
- [ ] Nenhuma ocorrência de `assinante_essencial/pro/reservado` permanece nos 3 arquivos.

- Comando: `php artisan test tests/Feature/HotmartHandlerTest.php tests/Feature/WebhookLastlinkTest.php tests/Feature/AdminHotmartTest.php`
