# Task 31.9 – Atualizar Testes de Auth/Feed/Perfil (FeedTest + PerfilTest + AuthTest)

## Objetivo
Atualizar os três arquivos de teste restantes para substituir atribuições e assertions de roles específicas por `assinante`.

## Tipo
teste

## Arquivos a criar/modificar
- `tests/Feature/FeedTest.php` → modificar
- `tests/Feature/PerfilTest.php` → modificar
- `tests/Feature/AuthTest.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `tests/Feature/FeedTest.php` → ver criação de usuários de teste com roles específicas e assertions
- `tests/Feature/PerfilTest.php` → idem
- `tests/Feature/AuthTest.php` → idem; verificar se há testes de login que validam a role retornada

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Para cada arquivo:
1. Localizar todas as ocorrências de `assinante_essencial`, `assinante_pro`, `assinante_reservado`.
2. Substituir por `assinante`.
3. **AuthTest:** verificar se o endpoint de login retorna o campo `role` — o valor esperado deve ser `assinante` (não mais a role específica por plano). Atualizar assertions de JSON como `assertJsonFragment(['role' => 'assinante_pro'])` → `assertJsonFragment(['role' => 'assinante'])`.
4. **FeedTest:** se testa acesso ao feed com usuário de plano específico, garantir que o usuário é criado com `role('assinante')` + `assinante.plano = 'pro'` (separados).
5. **PerfilTest:** idem — garantir que o perfil retorna o plano de `assinante.plano`, não da role.

### Regras de negócio aplicáveis:
- Se um teste criava um "usuário pro" com `assignRole('assinante_pro')`, deve criar com `syncRoles(['assinante'])` + `Assinante::factory()->state(['plano' => 'pro'])`.
- Testes que validavam controle de acesso por role (ex: "assinante_essencial não acessa X") passam a validar por `assinante.plano`.
- Não adicionar novos testes nesta task — apenas corrigir os existentes.

### Critérios de aceitação:
- [ ] `php artisan test --filter=FeedTest` passa sem erros.
- [ ] `php artisan test --filter=PerfilTest` passa sem erros.
- [ ] `php artisan test --filter=AuthTest` passa sem erros.
- [ ] Nenhuma ocorrência de `assinante_essencial/pro/reservado` permanece nos 3 arquivos.

## Teste esperado
- Comando: `php artisan test tests/Feature/FeedTest.php tests/Feature/PerfilTest.php tests/Feature/AuthTest.php`
- Todos os testes devem passar (verde).
