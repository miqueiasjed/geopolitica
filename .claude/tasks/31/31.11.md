# Task 31.11 – Frontend: AdminPlanos — Remover Coluna Role + Reformular PainelRecursos

## Objetivo
Remover a coluna "Role" da tabela de planos em `AdminPlanos.tsx` e reformular `PainelRecursos`/`LinhaRecurso` para exibir o estado de ativação (Ativo/Inativo) de cada recurso antes do valor/detalhe.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminPlanos.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/AdminPlanos.tsx` → ver estrutura atual da tabela, componentes PainelRecursos e LinhaRecurso
- `frontend/src/services/adminPlanos.ts` → ver tipo do plano retornado (campo `role` presente?)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**Tabela de planos:**
1. Localizar o cabeçalho da coluna "Role" (ou similar) na tabela de listagem de planos.
2. Remover a coluna "Role" — header e célula correspondente.
3. Não remover outras colunas (nome, preço, status, ações).

**PainelRecursos / LinhaRecurso:**
1. Localizar os componentes (podem ser sub-componentes dentro do arquivo ou importados).
2. Reformular cada `LinhaRecurso` para exibir na ordem:
   - **Badge de Status:** `Ativo` (verde) ou `Inativo` (cinza/vermelho) — baseado em se o recurso está habilitado para aquele plano.
   - **Nome do Recurso:** label legível.
   - **Valor/Detalhe** (condicional — só exibe quando `Ativo`): ex. "até 5 usuários", "ilimitado", etc.
3. Exemplo de layout de uma linha de recurso:
   ```
   [Ativo]  Feed de Tensões        (sem detalhe adicional)
   [Ativo]  Licenciamento B2B      até 5 usuários
   [Inativo] Relatório por IA
   ```
4. Garantir que recursos inativos não exibem o campo de valor/detalhe (evitar confusão visual).
5. Usar os componentes do design system (Badge, Chip) já existentes no projeto.

### Regras de negócio aplicáveis:
- O estado Ativo/Inativo é determinado pelos dados do plano (`recurso.ativo`, `recurso.habilitado` ou campo equivalente na API).
- O detalhe/valor (ex: limites numéricos) só é relevante quando o recurso está ativo — não exibir quando inativo.
- Manter a responsividade — a tabela deve funcionar em telas menores.

### Critérios de aceitação:
- [ ] Coluna "Role" removida da tabela de listagem de planos.
- [ ] Cada recurso na listagem de plano exibe badge Ativo/Inativo como primeiro elemento.
- [ ] Recursos inativos não exibem valor/detalhe.
- [ ] Sem erros de TypeScript ou console em `/admin/planos`.

## Teste esperado
- Abrir `/admin/planos` no browser.
- Verificar que a coluna "Role" sumiu da tabela.
- Clicar em um plano para ver detalhes — verificar layout de recursos: Ativo/Inativo antes do detalhe.
- Verificar em plano com recursos mistos (alguns ativos, alguns inativos).
