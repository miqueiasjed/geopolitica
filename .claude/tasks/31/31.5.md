# Task 31.5 – Refactor AdminUsuarioController + AdminPlanoController

## Objetivo
Remover `planoFromRole()` do `AdminUsuarioController` (derivar plano de `assinante.plano`) e parar de gerar role ao criar/editar planos no `AdminPlanoController`.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Api/Admin/AdminUsuarioController.php` → modificar
- `app/Http/Controllers/Api/Admin/AdminPlanoController.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `app/Http/Controllers/Api/Admin/AdminUsuarioController.php` → localizar método `planoFromRole()` e todos os seus usos
- `app/Http/Controllers/Api/Admin/AdminPlanoController.php` → ver onde cria/atribui role ao salvar plano

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**AdminUsuarioController:**
1. Remover o método `planoFromRole()` (que derivava o nome do plano a partir da role do usuário).
2. Em todos os pontos que chamavam `planoFromRole($usuario)`, substituir por:
   ```php
   $usuario->assinante?->plano
   ```
3. Nos retornos de JSON (`UserResource` ou array), garantir que o campo `plano` vem de `assinante.plano`, não da role.
4. No método `roles()` (lista de roles para o frontend), retornar `['assinante']` como única role de assinante disponível.

**AdminPlanoController:**
1. Localizar qualquer linha que:
   - Cria uma role Spatie ao criar um plano (ex: `Role::firstOrCreate(['name' => 'assinante_'.$plano->slug])`).
   - Atualiza o campo `role` da tabela `planos` com valor significativo de role Spatie.
2. Remover essa lógica de geração/atribuição de role.
3. O campo `planos.role` pode ficar em branco/null — não remover a coluna (está fora do escopo deste plano).

### Regras de negócio aplicáveis:
- O plano de um usuário é SEMPRE lido de `assinante.plano`, nunca inferido da role.
- O `AdminUsuarioController` não deve mais conhecer o mapeamento de roles para planos.
- `AdminPlanoController` não deve criar/sincronizar roles Spatie ao salvar planos.

### Critérios de aceitação:
- [ ] Método `planoFromRole()` removido do `AdminUsuarioController`.
- [ ] Nenhum uso de `assinante_essencial/pro/reservado` permanece nos dois controllers.
- [ ] `GET /api/admin/usuarios/{id}` retorna `plano` a partir de `assinante.plano` corretamente.
- [ ] `POST /api/admin/planos` não cria role Spatie ao salvar um plano.

## Teste esperado
- Verificação: `grep "assinante_" app/Http/Controllers/Api/Admin/AdminUsuarioController.php app/Http/Controllers/Api/Admin/AdminPlanoController.php` retorna vazio.
- Verificar resposta JSON de `GET /api/admin/usuarios/{id}` via tinker ou Postman.
