# Task 9.1 – Migrations, Models e Seeder de Indicadores

## Objetivo
Criar as migrations das tabelas `indicadores` e `indicadores_historico`, os Models correspondentes e o Seeder com os 6 registros iniciais dos indicadores de mercado.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_indicadores_table.php` → criar
- `database/migrations/xxxx_create_indicadores_historico_table.php` → criar
- `app/Models/Indicador.php` → criar
- `app/Models/IndicadorHistorico.php` → criar
- `database/seeders/IndicadoresSeeder.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/Conteudo.php` → ver padrão de Model adotado no projeto

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

1. **Migration `indicadores`**:
   - `id`
   - `simbolo` (string, unique — ex: `BZ=F`, `USDBRL=X`, `NG=F`, `ZS=F`, `ZW=F`, `TIO=F`)
   - `nome` (string — ex: "Petróleo Brent", "Câmbio BRL/USD")
   - `valor` (decimal 15,4, nullable)
   - `moeda` (string — `USD` | `BRL`)
   - `unidade` (string — ex: "USD/barril", "R$/USD")
   - `variacao_pct` (decimal 8,4, nullable — ex: -1.23 para -1,23%)
   - `variacao_abs` (decimal 15,4, nullable)
   - `atualizado_em` (timestamp, nullable)
   - `timestamps`

2. **Migration `indicadores_historico`**:
   - `id`
   - `simbolo` (string — FK lógica, sem constraint para performance)
   - `valor` (decimal 15,4)
   - `registrado_em` (timestamp)
   - `timestamps` (somente `created_at` necessário; usar `$table->timestamp('created_at')`)
   - Índice composto em `(simbolo, registrado_em)`

3. **Model `Indicador`**:
   - `$fillable` com todos os campos
   - Casts: `valor` e `variacao_pct` e `variacao_abs` para float, `atualizado_em` para datetime
   - Scope `porOrdem()`: retorna na ordem fixa Brent, BRL/USD, Gás, Soja, Trigo, Minério (ORDER BY FIELD ou array de ids)
   - Relacionamento `historico()`: hasMany `IndicadorHistorico`, foreignKey `simbolo`, ownerKey `simbolo`

4. **Model `IndicadorHistorico`**:
   - `$fillable`: `simbolo`, `valor`, `registrado_em`
   - Cast `registrado_em` para datetime
   - `$timestamps = false` (apenas `created_at` manual)

5. **`IndicadoresSeeder`**:
   - Inserir 6 registros em `indicadores` com `valor = 0` (serão populados pelo job):
     - `BZ=F` → "Petróleo Brent", moeda USD, unidade "USD/barril"
     - `USDBRL=X` → "Câmbio BRL/USD", moeda BRL, unidade "R$/USD"
     - `NG=F` → "Gás Natural", moeda USD, unidade "USD/MMBtu"
     - `ZS=F` → "Soja", moeda USD, unidade "USD/bushel"
     - `ZW=F` → "Trigo", moeda USD, unidade "USD/bushel"
     - `TIO=F` → "Minério de Ferro", moeda USD, unidade "USD/t"

### Regras de negócio aplicáveis:
- `simbolo` é o identificador único e chave de relacionamento entre tabelas
- `indicadores_historico` guarda histórico de 7 dias; registros com mais de 8 dias devem ser limpos (pela Service)
- Ordem de exibição dos indicadores é fixa: Brent → BRL/USD → Gás → Soja → Trigo → Minério

### Critérios de aceitação:
- [ ] `php artisan migrate` cria ambas as tabelas sem erros
- [ ] `php artisan db:seed --class=IndicadoresSeeder` insere 6 registros
- [ ] `Indicador::porOrdem()->get()` retorna na ordem correta
- [ ] `Indicador::first()->historico` acessa o relacionamento sem erro
