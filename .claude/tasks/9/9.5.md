# Task 9.5 – IndicatorsBar e integração no DashboardLayout

## Objetivo
Criar a barra horizontal de indicadores de mercado e integrá-la no topo do `DashboardLayout`, visível em todas as páginas do dashboard.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `src/components/indicadores/IndicatorsBar.tsx` → criar
- `src/components/layout/DashboardLayout.tsx` → modificar (adicionar IndicatorsBar)

## Contexto necessário (arquivos para ler antes)
- `src/components/indicadores/IndicatorCard.tsx` → interface de props
- `src/hooks/useIndicadores.ts` → assinatura do hook
- `src/components/layout/DashboardLayout.tsx` → ver estrutura atual do layout

## Skills necessárias
- frontend-design-system
- animacoes-motion

## Especificação

### O que fazer:

1. **`IndicatorsBar.tsx`**:
   - `h-16 bg-[#0d0d0f] border-b border-zinc-800`
   - Label "Mercados" em `text-xs text-zinc-500` no canto esquerdo
   - 6 `IndicatorCard` em flex row com `gap-4 px-4`
   - Scroll horizontal em telas pequenas (`overflow-x-auto`, esconder scrollbar com `scrollbar-hide`)
   - Timestamp "atualizado X min atrás" calculado a partir de `ultimaAtualizacao`:
     - < 1 min: "agora mesmo"
     - 1–59 min: "X min atrás"
     - ≥ 60 min: "há X h"
   - Timestamp no canto direito em `text-xs text-zinc-600`
   - Skeleton loading: 6 cards placeholder quando `isLoading = true` (retângulos animados)
   - Passa `historico` para cada card via `useIndicadorHistorico(simbolo)` individual

2. **`DashboardLayout.tsx`** — inserir `<IndicatorsBar />`:
   - Posicionar **acima** da `TopNav`, como primeira linha do layout
   - Não deve ser condicional — exibir para todos os assinantes autenticados
   - Ajustar `min-h` ou padding do conteúdo principal se necessário

### Regras de negócio aplicáveis:
- `IndicatorsBar` visível em todas as páginas do dashboard para todos os roles
- Timestamp recalculado a cada renderização (não precisa de interval próprio — o hook já tem `refetchInterval`)
- Skeleton impede CLS (Cumulative Layout Shift) ao carregar

### Critérios de aceitação:
- [ ] Barra aparece em todas as páginas do dashboard
- [ ] 6 cards renderizam com valores e sparklines
- [ ] Timestamp mostra "X min atrás" corretamente
- [ ] Skeleton visível durante carregamento inicial
- [ ] Scroll horizontal funciona em mobile sem scrollbar visível
- [ ] Layout não quebra após adicionar a barra

## Teste esperado
- Verificação visual no browser: navegar entre páginas do dashboard e confirmar que a barra persiste
