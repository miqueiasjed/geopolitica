# Task 9.2 – MarketFetcherService, IndicadoresService, Job e Scheduler

## Objetivo
Implementar o serviço de busca de dados de mercado (Yahoo Finance + BCB PTAX), o serviço de upsert/cache/histórico, o Job de fila e a configuração do scheduler para atualização a cada 15 minutos.

## Especificação

### O que fazer:

1. **`MarketFetcherService`**:

   - `buscarYahooFinance(array $simbolos): array`
     - URL: `https://query1.finance.yahoo.com/v8/finance/spark?symbols=BZ=F,NG=F,ZS=F,ZW=F,TIO=F&range=1d&interval=15m`
     - HTTP GET via `Http::withHeaders(['User-Agent' => 'Mozilla/5.0'])->get($url)`
     - Retorna array keyed por símbolo: `['BZ=F' => ['valor' => 85.42, 'variacao_pct' => -0.32, 'variacao_abs' => -0.27]]`
     - Em caso de erro HTTP: retorna array vazio e loga warning

   - `buscarCambioBCB(): array`
     - URL: `https://economia.awesomeapi.com.br/json/last/USD-BRL` (API pública, sem autenticação)
     - Retorna `['USDBRL=X' => ['valor' => 5.23, 'variacao_pct' => 0.15, 'variacao_abs' => 0.008]]`
     - Em caso de erro: retorna array vazio e loga warning

2. **`IndicadoresService`**:

   - `atualizarTodos(): void`
     - Chama `MarketFetcherService::buscarYahooFinance()` e `buscarCambioBCB()`
     - Para cada indicador retornado: chama `upsertValor()`
     - Limpa histórico com mais de 8 dias: `IndicadorHistorico::where('registrado_em', '<', now()->subDays(8))->delete()`
     - Invalida cache Redis ao final

   - `upsertValor(string $simbolo, float $valor, float $variacaoPct, float $variacaoAbs): void`
     - `Indicador::where('simbolo', $simbolo)->update(['valor' => ..., 'variacao_pct' => ..., 'variacao_abs' => ..., 'atualizado_em' => now()])`
     - Insere em `indicadores_historico`: `['simbolo' => $simbolo, 'valor' => $valor, 'registrado_em' => now()]`

   - `listarComCache(): array`
     - Cache Redis key `'indicadores:lista'`, TTL 15 minutos
     - Se cache miss: busca `Indicador::porOrdem()->get()` e armazena
     - Retorna array de `Indicador`

   - `historicoPorSimbolo(string $simbolo): array`
     - Retorna últimos 7 dias de `IndicadorHistorico` para o símbolo
     - Orderby `registrado_em` asc

3. **`AtualizarIndicadoresJob`**:
   - `implements ShouldQueue`
   - `$queue = 'redis'`, `$tries = 3`, `$timeout = 60`, `$backoff = 30`
   - `handle()`: chama `IndicadoresService::atualizarTodos()`

4. **`Console/Kernel.php`** — adicionar ao scheduler:
   ```php
   $schedule->job(new AtualizarIndicadoresJob)
            ->everyFifteenMinutes()
            ->withoutOverlapping()
            ->onFailure(fn() => Log::error('AtualizarIndicadoresJob falhou'));
   ```

### Regras de negócio aplicáveis:
- Yahoo Finance não requer auth mas exige User-Agent no header
- BCB PTAX via awesomeapi é gratuita e sem chave
- Cache Redis deve ser invalidado após cada atualização bem-sucedida
- `withoutOverlapping()` evita execuções simultâneas do scheduler

### Critérios de aceitação:
- [ ] `MarketFetcherService::buscarYahooFinance(['BZ=F'])` retorna dados estruturados
- [ ] `IndicadoresService::atualizarTodos()` atualiza os 6 indicadores
- [ ] Após `atualizarTodos()`, registros aparecem em `indicadores_historico`
- [ ] Registros com mais de 8 dias são removidos
- [ ] `IndicadoresService::listarComCache()` retorna do Redis na segunda chamada
- [ ] Job despacha e executa sem erros: `AtualizarIndicadoresJob::dispatch()`
