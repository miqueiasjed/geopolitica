# Task 6.2 – GdeltFetcherService, MapaIntensidadeService, Job e Scheduler

## Objetivo
Implementar o `GdeltFetcherService` que busca dados da GDELT API v2 e normaliza o tom, o `MapaIntensidadeService` que agrega dados com cache Redis 15 min, o `AtualizarGdeltJob` e o scheduler no minuto :30.

## Especificação

### O que fazer:
1. Criar `GdeltFetcherService`:
   - Método `fetch(): array` — busca eventos da GDELT API v2
   - Endpoint: `https://api.gdeltproject.org/api/v2/summary/summary?d=web&t=summary&k=&ts=custom&sdt=&edt=&svt=zoom&stab=1&mode=pointdata&fmt=json`
   - Parsear resposta para extrair por país: `codigo_pais`, `nome_pais`, `total_eventos`, `tom_medio`
   - Normalizar `tom_medio`: `intensidade_gdelt = round(((tom * -1) + 100) / 20) + 1`
   - Clamp resultado entre 1 e 10
   - Em caso de erro: logar warning e retornar array vazio
2. Criar `MapaIntensidadeService`:
   - Método `obter(): array` — retorna dados da view com cache Redis
   - Cache key: `mapa:intensidade`, TTL: 15 minutos
   - Se cache hit: retornar direto
   - Se cache miss: `DB::table('mapa_intensidade')->get()` + cachear
   - Método `invalidar(): void` — limpar cache Redis
3. Criar `AtualizarGdeltJob`:
   - `$tries = 3`, `$timeout = 120`, fila `default`
   - `handle()`: chamar `GdeltFetcherService::fetch()`, fazer upsert em `gdelt_cache`, chamar `MapaIntensidadeService::invalidar()`
   - Usar `GdeltCache::upsert()` com `codigo_pais` como chave única
4. Registrar no scheduler (minuto :30, sem conflito com M01 que roda no :00):
   ```php
   Schedule::job(new AtualizarGdeltJob)->hourlyAt(30)->withoutOverlapping();
   ```

### Regras de negócio aplicáveis:
- GDELT roda no minuto :30 para não coincidir com o M01 (que roda no :00)
- Cache Redis 15 min para o mapa — o endpoint é consultado com frequência pelo frontend
- `upsert()` com `codigo_pais` como chave para atualização idempotente

### Critérios de aceitação:
- [ ] `GdeltFetcherService::fetch()` retorna array com dados de países
- [ ] `AtualizarGdeltJob` popula `gdelt_cache` sem erros
- [ ] `MapaIntensidadeService::obter()` retorna dados com cache funcionando
- [ ] Segunda chamada em 15 min retorna dado cacheado (sem query no banco)
- [ ] Scheduler lista o job GDELT no minuto :30
- [ ] `withoutOverlapping()` configurado
