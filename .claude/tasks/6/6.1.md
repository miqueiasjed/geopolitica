# Task 6.1 – Migration gdelt_cache, View SQL mapa_intensidade e Model

## Objetivo
Criar a migration da tabela `gdelt_cache`, a view SQL `mapa_intensidade` que combina dados M01 + GDELT com fórmula de peso, e o Model `GdeltCache`.

## Tipo

## Especificação

### O que fazer:
1. Migration `create_gdelt_cache_table`:
   - `id`, `codigo_pais` (char(2), unique — ISO 3166-1 alpha-2), `nome_pais` (string)
   - `total_eventos` (integer, default: 0)
   - `tom_medio` (decimal 5,2, nullable — tom negativo do GDELT)
   - `intensidade_gdelt` (decimal 5,2 — normalizado 1–10)
   - `atualizado_em` (timestamp, nullable)
   - `timestamps`
2. Migration `create_mapa_intensidade_view` (usar `DB::statement` para criar view):
   ```sql
   CREATE OR REPLACE VIEW mapa_intensidade AS
   SELECT
     g.codigo_pais,
     g.nome_pais,
     g.intensidade_gdelt,
     COALESCE(AVG(e.impact_score), 0) AS score_m1,
     ROUND(
       (COALESCE(AVG(e.impact_score), 0) * 0.6) + (g.intensidade_gdelt * 0.4),
       2
     ) AS intensidade_final
   FROM gdelt_cache g
   LEFT JOIN events e ON LOWER(e.regiao) LIKE CONCAT('%', LOWER(g.nome_pais), '%')
     AND e.relevante = 1
     AND e.publicado_em >= NOW() - INTERVAL 48 HOUR
   GROUP BY g.codigo_pais, g.nome_pais, g.intensidade_gdelt
   ```
3. Criar `GdeltCache` Model:
   - `$fillable` com todos os campos
   - Cast `intensidade_gdelt` e `tom_medio` para float
   - Cast `atualizado_em` para datetime
   - `$table = 'gdelt_cache'`

### Regras de negócio aplicáveis:
- Fórmula: `(score_m1 × 0.6) + (intensidade_gdelt × 0.4)`
- Normalização GDELT: `round(((tom × -1) + 100) / 20) + 1` (resultado 1–10)
- A view deve ser recriada com `CREATE OR REPLACE` para ser idempotente

### Critérios de aceitação:
- [ ] `php artisan migrate` cria tabela `gdelt_cache` e view `mapa_intensidade`
- [ ] `DB::select('SELECT * FROM mapa_intensidade LIMIT 1')` executa sem erro
- [ ] `GdeltCache::create([...])` funciona
- [ ] View retorna `intensidade_final` como decimal entre 1 e 10
