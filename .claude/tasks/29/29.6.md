# Task 29.6 – Router admin + Sidebar: rotas e links de navegação

## Objetivo
Registrar as rotas admin das novas páginas no Router e adicionar os links "Produtos" e "Importar Addons" no Sidebar admin, com navegação funcionando corretamente.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/router/` (arquivo de rotas admin) → modificar
- `frontend/src/components/admin/AdminSidebar.tsx` (ou equivalente) → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/router/` → ver como rotas admin são registradas atualmente
- `frontend/src/components/admin/AdminSidebar.tsx` → ver estrutura de links existentes

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**Router — adicionar rotas:**
```tsx
// Dentro do grupo admin (AuthGuard + role admin)
{ path: 'produtos', element: <AdminProdutos /> }
{ path: 'assinantes/importar-addons', element: <AdminImportarAddons /> }
```

**Sidebar admin — adicionar links:**
- "Produtos" — posicionar após "Planos" na navegação principal
- Ícone: `PackageIcon` ou similar
- "Importar Addons" — posicionar na seção de assinantes, após o link de listagem de assinantes
- Ícone: `UploadIcon` ou similar

**Integração AdminAssinanteAddons:**
- Na página de detalhe do assinante, adicionar aba/seção "Addons" que renderiza `<AdminAssinanteAddons userId={userId} />`
- Verificar onde a página de detalhe do assinante está localizada

### Regras de negócio aplicáveis:
- Rotas devem estar dentro do AuthGuard com role admin
- Nomes das rotas devem ser consistentes com a convenção do projeto
- Links do sidebar com estado ativo (classe active) quando rota corresponde

### Critérios de aceitação:
- [ ] Navegar para `/admin/produtos` carrega `AdminProdutos`
- [ ] Navegar para `/admin/assinantes/importar-addons` carrega `AdminImportarAddons`
- [ ] Sidebar exibe "Produtos" e "Importar Addons" nos lugares corretos
- [ ] Link ativo destacado na navegação atual
- [ ] `AdminAssinanteAddons` acessível a partir do detalhe do assinante

## Teste esperado
- Verificação visual: navegar pelos links do sidebar, confirmar rotas corretas
- Verificação: F5 nas rotas, confirmar não quebram (sem 404)
