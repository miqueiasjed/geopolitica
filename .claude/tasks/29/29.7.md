# Task 29.7 – Polimento visual e estados de loading/erro globais

## Objetivo
Revisar todas as páginas admin criadas nos Planos 29.1–29.6 para garantir consistência visual, estados de erro, toasts de feedback e comportamento correto em mobile/tablet.

frontend-pagina

## Skills necessárias
- `frontend-design-system`
- `animacoes-motion`

## Especificação

### O que fazer:
**Verificar e corrigir em cada página:**
1. **Toast de sucesso** após criar/editar/excluir/importar (ex: "Produto criado com sucesso")
2. **Toast de erro** para falhas de rede (não apenas 422)
3. **Estado de loading** consistente (mesmo padrão em todas as páginas)
4. **Responsividade:** tabelas com scroll horizontal em telas pequenas
5. **Acessibilidade:** labels em todos os inputs, ARIA em modais, foco correto ao abrir/fechar modal
6. **Animações:** verificar que Framer Motion está aplicado consistentemente (`animacoes-motion` skill)

**Cheklist de consistência visual:**
- Paleta gold `#C9B882` usada nos títulos de todas as páginas admin
- Tipografia, espaçamentos e bordas iguais às demais páginas admin existentes
- Botões com hover states corretos

### Regras de negócio aplicáveis:
- Não adicionar funcionalidades novas — apenas polimento e consistência
- Manter padrão visual das demais páginas admin (`/admin/planos`, `/admin/assinantes`, etc.)

### Critérios de aceitação:
- [ ] Toast de sucesso após cada operação CRUD
- [ ] Toast de erro em falhas de rede (500, timeout)
- [ ] Tabelas com scroll horizontal em viewport < 768px
- [ ] Todos os inputs têm labels acessíveis
- [ ] Animações de entrada/saída de modais funcionando
- [ ] Visual consistente com demais páginas admin

## Teste esperado
- Verificação visual completa: percorrer todas as funcionalidades das 3 páginas
- Verificação responsiva: testar em viewport 375px, 768px, 1280px
