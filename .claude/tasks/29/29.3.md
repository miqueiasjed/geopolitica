# Task 29.3 – AdminAssinanteAddons.tsx

## Objetivo
Criar a página/componente `AdminAssinanteAddons.tsx` que exibe e gerencia os addons de um assinante específico, acessível a partir da página de detalhe do assinante.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminAssinanteAddons.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/` → ver página de detalhe do assinante e como é estruturada
- `frontend/src/services/adminAssinanteAddons.ts` → importar service (Task 28.2)
- `frontend/src/services/adminProdutos.ts` → listar produtos ativos para o select
- `frontend/src/types/produto.ts` → tipos relevantes

## Skills necessárias
- `frontend-design-system`
- `animacoes-motion`

## Especificação

### O que fazer:
**Componente/Página** que recebe `userId: number` como prop ou parâmetro de rota:

**Listagem:**
- `useQuery` chamando `adminAssinanteAddons.listar(userId)`
- Tabela/lista: addon_key, status (badge colorido), fonte, iniciado_em, expira_em, ações

**Modal "+ Adicionar Addon":**
- Select de `addon_key`: buscar `adminProdutos.listar()`, exibir apenas ativos
- Campos: status (select), fonte (select — default "manual"), iniciado_em (date), expira_em (date)
- Submit: `adminAssinanteAddons.adicionar(userId, payload)`

**Editar status na linha:**
- Botão de edição inline ou modal simplificado
- Chamar `adminAssinanteAddons.atualizar(userId, addonId, { status })`

**Remover addon:**
- Botão com confirmação
- Chamar `adminAssinanteAddons.remover(userId, addonId)`
- Retorno 204 → remover da lista local

**Badges de status:**
- `ativo` → verde
- `cancelado` → âmbar
- `expirado` → cinza
- `reembolsado` → vermelho

### Regras de negócio aplicáveis:
- Select de addon_key mostra apenas produtos `ativo=true`
- Fonte padrão "manual" para addons criados via painel
- Sincronização via backend (não precisa atualizar `assinantes.addons[]` manualmente no frontend)

### Critérios de aceitação:
- [ ] Lista addons do assinante corretamente
- [ ] Modal de adicionar carrega produtos ativos no select
- [ ] Após adicionar addon com status ativo, lista atualiza
- [ ] Editar e remover addon funcional
- [ ] Badges de status com cores corretas

## Teste esperado
- Verificação visual: acessar assinante no admin, ver e gerenciar seus addons
- Verificação: adicionar addon elections, confirmar aparece na lista
