# Task 29.1 – AdminProdutos.tsx (tabela de listagem)

## Objetivo
Criar a página `AdminProdutos.tsx` com a tabela de listagem de produtos, exibindo colunas de nome, chave, preço, links, status, ordem e ações (Editar, Excluir).

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminProdutos.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/` → ver padrão de páginas admin existentes (layout, paleta gold `#C9B882`)
- `frontend/src/services/adminProdutos.ts` → importar `adminProdutos.listar()` (Task 28.2)
- `frontend/src/types/produto.ts` → usar tipo `AdminProduto` (Task 28.1)

## Skills necessárias
- `frontend-design-system`
- `animacoes-motion`

## Especificação

### O que fazer:
Página com:
- `useQuery` chamando `adminProdutos.listar()`
- Tabela com colunas: Nome, Chave, Preço (preco_label), Links (ícones clicáveis para link_compra e link_reativar), Status (badge Ativo/Inativo), Ordem, Ações
- Botão "+ Novo Produto" no topo (abre modal — implementado na Task 29.2)
- Estado vazio: mensagem "Nenhum produto cadastrado"
- Estado loading: skeleton ou spinner consistente com o padrão admin
- Ações: botões "Editar" e "Excluir" por linha

**Links na tabela:**
- Ícone de link clicável quando não null; ícone desabilitado quando null
- Tooltip com URL ao hover

**Paleta:** usar `#C9B882` gold para títulos/destaques do painel admin.

**Excluir:**
- Confirmar antes de deletar (dialog simples)
- Chamar `adminProdutos.excluir(id)`
- Se receber 422: exibir toast com mensagem do backend ("N assinante(s) com addon ativo")
- Invalidar query após sucesso

### Regras de negócio aplicáveis:
- Exibir todos os produtos (ativos e inativos) — admin vê tudo
- Badge verde para ativo, cinza para inativo
- Erro 422 no excluir deve ser visível ao usuário

### Critérios de aceitação:
- [ ] Tabela exibe lista de produtos com todas as colunas
- [ ] Badge de status correto (verde/cinza)
- [ ] Ícones de link clicáveis e desabilitados conforme valor
- [ ] Botão excluir com confirmação e tratamento de 422
- [ ] Estado de loading e vazio implementados

## Teste esperado
- Verificação visual: navegar para `/admin/produtos`, confirmar tabela e dados
- Verificação: excluir produto com addon ativo, confirmar mensagem de erro
