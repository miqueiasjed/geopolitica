# Task 29.5 – AdminImportarAddons.tsx: relatório pós-importação

## Objetivo
Adicionar à página `AdminImportarAddons.tsx` a exibição do relatório retornado após importação, mostrando o total importado e lista de erros por linha com motivo.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminImportarAddons.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/AdminImportarAddons.tsx` → estado após Task 29.4
- Formato da resposta do backend: `{ importados: number, erros: [{ linha: number, motivo: string }] }` (Plano 27)

## Skills necessárias
- `frontend-design-system`
- `animacoes-motion`

## Especificação

### O que fazer:
Após submit bem-sucedido da importação, exibir seção de relatório:

**Resumo:**
- Card verde: "✓ N registro(s) importado(s) com sucesso"
- Se houver erros: card âmbar: "⚠ N erro(s) encontrado(s)"

**Lista de erros:**
- Tabela ou lista com colunas: Linha, Motivo
- Cor de destaque para o número da linha
- Scroll caso haja muitos erros

**Importação assíncrona (job):**
- Se backend retornar `{ mensagem: 'Importação em andamento' }`: exibir estado de "processando" com spinner
- Não implementar polling (fora do escopo — informar ao usuário para aguardar e recarregar)

**Após novo upload:**
- Limpar relatório anterior ao selecionar novo arquivo

**Animação:**
- Entrada suave do relatório usando Framer Motion (fade + slide up)

### Regras de negócio aplicáveis:
- Relatório substituído a cada importação (não acumular)
- Erros não impedem sucesso parcial — mostrar ambos (N importados + M erros)
- Para arquivos grandes (job), apenas informar que está em processamento

### Critérios de aceitação:
- [ ] Relatório exibido com total de importados após submit
- [ ] Lista de erros com número de linha e motivo
- [ ] Relatório limpo ao selecionar novo arquivo
- [ ] Mensagem de "em andamento" para importações via job
- [ ] Animação de entrada do relatório funcionando

## Teste esperado
- Verificação visual: importar CSV com erros e acertos, confirmar relatório correto
- Verificação: importar CSV válido, confirmar apenas card de sucesso (sem erros)

## Estimativa de complexidade
baixa
~60 linhas de código | ~1 arquivo
