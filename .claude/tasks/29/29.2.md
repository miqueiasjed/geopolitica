# Task 29.2 – AdminProdutos.tsx: Modal Criar/Editar Produto

## Objetivo
Adicionar o modal de criar e editar produto à página `AdminProdutos.tsx`, com formulário validado e integração com os endpoints de store/update.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminProdutos.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/AdminProdutos.tsx` → estado após Task 29.1
- `frontend/src/services/adminProdutos.ts` → usar `criar()` e `atualizar()`
- Padrão de modal existente no admin (Radix Dialog ou componente do projeto)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**Modal com campos:**
- `chave`: text input — obrigatório, read-only quando editando, validar `^[a-z0-9_-]+$` no frontend
- `nome`: text input — obrigatório
- `descricao`: textarea — opcional
- `preco_label`: text input — opcional (ex: "R$ 49/mês")
- `link_compra`: URL input — opcional
- `link_reativar`: URL input — opcional
- `ativo`: switch/checkbox — padrão true
- `ordem`: number input — padrão 0

**Comportamento:**
- Ao clicar "Editar" na tabela: abrir modal preenchido com dados do produto
- Ao clicar "+ Novo Produto": abrir modal vazio
- No submit: chamar `adminProdutos.criar()` ou `adminProdutos.atualizar()`
- Após sucesso: fechar modal + invalidar query `['admin-produtos']`
- Em caso de erro 422: exibir erros de validação inline

### Regras de negócio aplicáveis:
- Campo `chave` read-only no modo edição (preenchido mas não editável)
- Feedback imediato de erro de validação (sem aguardar submit do backend)
- Invalidar cache após criação/edição para refletir mudanças na tabela

### Critérios de aceitação:
- [ ] Modal abre vazio ao criar, preenchido ao editar
- [ ] Campo `chave` é read-only no modo edição
- [ ] Submit chama endpoint correto (POST no create, PUT no edit)
- [ ] Tabela atualiza após fechar modal com sucesso
- [ ] Erros 422 exibidos inline nos campos correspondentes

## Teste esperado
- Verificação visual: criar produto com dados válidos, verificar aparece na tabela
- Verificação: editar produto, confirmar `chave` não pode ser alterada
