# Task 4.1 – Migrations Sources e Events com índices

## Objetivo
Criar as migrations das tabelas `sources` e `events` com todos os índices necessários (incluindo FULLTEXT), e os Models correspondentes com scopes, casts e relacionamentos.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_sources_table.php` → criar
- `database/migrations/xxxx_create_events_table.php` → criar
- `app/Models/Source.php` → criar
- `app/Models/Event.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/Assinante.php` → ver padrão de Model adotado no projeto

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Migration `sources`:
   - `id`, `nome` (string), `rss_url` (string, unique), `categoria` (string)
   - `ativo` (boolean, default: true)
   - `ultima_coleta_em` (timestamp, nullable)
   - `timestamps`
   - Índice em `categoria` e `ativo`
2. Migration `events`:
   - `id`, `titulo` (string), `resumo` (text), `analise_ia` (text, nullable)
   - `fonte` (string — nome da fonte), `fonte_url` (string, unique — para deduplicação)
   - `regiao` (string, nullable), `impact_score` (tinyInteger 1–10)
   - `impact_label` (string — `CRÍTICO` | `ALTO` | `MÉDIO` | `MONITORAR`)
   - `categorias` (json — array de categorias)
   - `relevante` (boolean, default: false)
   - `publicado_em` (timestamp)
   - `timestamps`
   - Índice em `regiao`, `impact_score`, `relevante`, `publicado_em`
   - FULLTEXT index em `titulo` e `resumo` (usar `$table->fullText(['titulo', 'resumo'])`)
3. Criar `Source` Model:
   - `$fillable` com todos os campos
   - Cast `ativo` para boolean, `ultima_coleta_em` para datetime
   - Scope `ativos()`: `where('ativo', true)`
4. Criar `Event` Model:
   - `$fillable` com todos os campos
   - Cast `categorias` para array, `relevante` para boolean, `publicado_em` para datetime
   - Scope `relevantes()`: `where('relevante', true)`
   - Scope `ultimas48h()`: `where('publicado_em', '>=', now()->subHours(48))`
   - Scope `porRegiao(string $regiao)`: filtro por região

### Regras de negócio aplicáveis:
- `fonte_url` deve ser único — é o mecanismo de deduplicação de eventos
- FULLTEXT index é necessário para buscas futuras por texto
- `impact_label` é derivado do `impact_score`: 8–10 = CRÍTICO, 6–7 = ALTO, 4–5 = MÉDIO, 1–3 = MONITORAR

### Critérios de aceitação:
- [ ] `php artisan migrate` cria ambas as tabelas sem erros
- [ ] FULLTEXT index existe na tabela `events`
- [ ] `Event::ultimas48h()->count()` executa sem erro
- [ ] `Source::ativos()->get()` filtra corretamente
- [ ] Tentativa de inserir dois eventos com mesma `fonte_url` lança `UniqueConstraintViolationException`

## Teste esperado

- Verificação manual: `php artisan tinker` → `Schema::getColumnListing('events')` lista todas as colunas
