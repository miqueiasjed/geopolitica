# Task 4.3 – AiAnalyzerService (análise Claude API em lotes)

## Objetivo
Implementar o `AiAnalyzerService` que envia lotes de até 5 itens RSS para a Claude API, recebe a análise estruturada (relevância, score, label, resumo, regiões, categorias) e retorna os dados prontos para persistência.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/AiAnalyzerService.php` → criar

## Contexto necessário (arquivos para ler antes)
- `config/claude.php` → ver modelo e configurações da API (task 1.2)
- `app/Models/Event.php` → ver campos esperados no retorno (task 4.1)

## Skills necessárias
- laravel-arquitetura
- claude-api

## Especificação

### O que fazer:
1. Criar `AiAnalyzerService` com método `analisar(array $itens): array`
   - Recebe array de até 5 itens: `[['titulo', 'resumo', 'fonte_url', 'fonte', 'publicado_em'], ...]`
   - Monta prompt com context de relevância para investidores brasileiros:
     - Critérios: energia (petróleo, gás), câmbio (dólar, real), alimentos (soja, milho, trigo), sanções, eleições em países com impacto no Brasil, conflitos em rotas comerciais
   - Solicita JSON estruturado por item:
     ```json
     {
       "relevante": true,
       "impact_score": 7,
       "impact_label": "ALTO",
       "analise_ia": "Breve análise de 2-3 frases em português...",
       "regiao": "Europa",
       "categorias": ["energia", "sanções"]
     }
     ```
   - Usar `claude-sonnet-4-5` via config `config('claude.model')`
   - Incluir prompt caching (system prompt como `cache_control: ephemeral`)
   - Parsear a resposta JSON; se parse falhar, marcar item como `relevante: false`
   - Retornar array com os mesmos itens enriquecidos com os campos de análise

### Regras de negócio aplicáveis:
- Análise em lotes de **máximo 5 itens** para controlar rate limit da Claude API
- Itens marcados como `relevante: false` são persistidos com `impact_score = 1`
- `impact_label` deve ser derivado do `impact_score`: 8–10 = CRÍTICO, 6–7 = ALTO, 4–5 = MÉDIO, 1–3 = MONITORAR
- Usar prompt caching no system prompt para reduzir custos de tokens

### Critérios de aceitação:
- [ ] `AiAnalyzerService::analisar([$item])` retorna array com campos `relevante`, `impact_score`, `impact_label`, `analise_ia`, `regiao`, `categorias`
- [ ] Lote de 5 itens retorna 5 resultados com dados individuais
- [ ] Notícia sobre esporte retorna `relevante: false`
- [ ] Notícia sobre sanções ao petróleo retorna `relevante: true` com `impact_score >= 6`
- [ ] Falha no parse JSON da Claude não quebra o serviço
