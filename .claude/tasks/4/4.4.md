# Task 4.4 – FeedUpdaterService, Job e Scheduler

## Objetivo
Implementar o `FeedUpdaterService` que orquestra coleta + análise + persistência, o `ProcessFeedUpdateJob` assíncrono para execução via fila Redis, e o `UpdateFeedCommand` para trigger manual. Configurar o scheduler para rodar a cada hora.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/FeedUpdaterService.php` → criar
- `app/Jobs/ProcessFeedUpdateJob.php` → criar
- `app/Console/Commands/UpdateFeedCommand.php` → criar
- `routes/console.php` → modificar (ou `app/Console/Kernel.php`) — registrar schedule

## Contexto necessário (arquivos para ler antes)

## Especificação

### O que fazer:
1. Criar `FeedUpdaterService::atualizar(): array`:
   - Buscar `Source::ativos()->get()`
   - Para cada source: chamar `RssFetcherService::fetchSource($source)`
   - Acumular todos os itens novos, processar em lotes de 5 via `AiAnalyzerService::analisar()`
   - Salvar cada item analisado em `Event::create([...])`
   - Retornar `['coletados' => N, 'novos' => N, 'salvos' => N]`
2. Criar `ProcessFeedUpdateJob`:
   - Implements `ShouldQueue`
   - `$tries = 3`, `$timeout = 300`
   - Fila: `default`
   - Método `handle()`: instanciar e chamar `FeedUpdaterService::atualizar()`
   - Logar resultado via `Log::info()`
3. Criar `UpdateFeedCommand` (`feed:update`):
   - Despachar `ProcessFeedUpdateJob::dispatch()`
   - Opção `--sync` para rodar sincronamente (sem fila) em dev
4. Registrar no scheduler (em `routes/console.php`):
   ```php
   Schedule::job(new ProcessFeedUpdateJob)->hourly()->withoutOverlapping();
   ```

### Regras de negócio aplicáveis:
- `withoutOverlapping()` é obrigatório — evita múltiplas execuções simultâneas
- O retorno `{ coletados, novos, salvos }` permite monitorar a eficiência do pipeline
- `salvos` < `novos` indica falha de persistência — logar como warning

### Critérios de aceitação:
- [ ] `php artisan feed:update --sync` executa o pipeline completo sem erros
- [ ] `php artisan feed:update` despacha o job para a fila Redis
- [ ] `php artisan queue:work --once` processa o job com sucesso
- [ ] Scheduler configurado: `php artisan schedule:list` mostra o job horário
- [ ] Segundo dispatch simultâneo é bloqueado por `withoutOverlapping()`
- [ ] `FeedUpdaterService` retorna `{ coletados, novos, salvos }`

## Teste esperado
