# Task 4.5 – FeedController, EventResource e rotas com controle por role

## Objetivo
Implementar o `FeedController` com GET paginado por cursor e filtros, o `EventResource` para transformação da resposta, e as rotas com controle de acesso por role (essencial limita a 20 eventos/48h, reservado tem histórico completo).

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/FeedController.php` → criar
- `app/Http/Requests/FeedFilterRequest.php` → criar
- `app/Http/Resources/EventResource.php` → criar
- `app/Http/Resources/EventCollection.php` → criar
- `routes/api.php` → modificar (adicionar rotas feed)

## Especificação

### O que fazer:
1. Criar `FeedFilterRequest` com validação:
   - `categoria`: nullable, string, in:[energia,alimentos,cambio,conflitos,sancoes,eleicoes,commodities]
   - `regiao`: nullable, string
   - `label`: nullable, in:[CRÍTICO,ALTO,MÉDIO,MONITORAR]
   - `cursor`: nullable, string (cursor de paginação)
   - `limite`: nullable, integer, between:5,50 (default: 20)
2. Criar `EventResource` retornando:
   ```json
   { "id", "titulo", "resumo", "analise_ia", "fonte", "regiao", "impact_score", "impact_label", "categorias", "publicado_em" }
   ```
3. Criar `FeedController::index(FeedFilterRequest $request)`:
   - Query base: `Event::relevantes()->ultimas48h()` para `assinante_essencial` e `assinante_pro`
   - Para `assinante_reservado` e `admin`: sem filtro de 48h (histórico completo)
   - Para `assinante_essencial`: limitar a 20 eventos
   - Aplicar filtros: categoria (in `categorias` json), regiao, label
   - Paginação cursor-based via `cursorPaginate($limite)`
   - Retornar `EventCollection`
4. Criar `FeedController::atualizar(Request $request)`:
   - Apenas role `admin`
   - Despachar `ProcessFeedUpdateJob::dispatch()`
   - Retornar `{ message: 'Atualização em andamento' }` com 202
5. Registrar rotas:
   ```php
   Route::middleware(['auth:sanctum', 'assinante.ativo'])->group(function () {
       Route::get('/feed', [FeedController::class, 'index']);
       Route::post('/feed/atualizar', [FeedController::class, 'atualizar'])
            ->middleware('role:admin');
   });
   ```

### Regras de negócio aplicáveis:
- `assinante_essencial` recebe no máximo 20 eventos das últimas 48h
- `assinante_reservado` e `admin` recebem histórico sem limite temporal
- Paginação cursor-based é mais eficiente que offset para feeds em tempo real

### Critérios de aceitação:
- [ ] `GET /api/feed` sem token retorna 401
- [ ] `GET /api/feed` com assinante inativo retorna 403
- [ ] `GET /api/feed` com token válido retorna lista paginada de eventos
- [ ] Filtro `?categoria=energia` retorna apenas eventos da categoria energia
- [ ] Assinante essencial recebe no máximo 20 eventos
- [ ] `POST /api/feed/atualizar` com role admin retorna 202
- [ ] `POST /api/feed/atualizar` com role não-admin retorna 403
