# Task 4.6 – Error handling global e testes do pipeline

## Objetivo
Configurar o handler de erros global para retornar 401/403 padronizados em JSON, e criar os testes automatizados do pipeline de feed (Service + endpoint).

## Tipo
teste

## Arquivos a criar/modificar
- `app/Exceptions/Handler.php` → modificar (renderizar 401/403 como JSON)
- `tests/Feature/FeedTest.php` → criar
- `tests/Unit/AiAnalyzerServiceTest.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Http/Controllers/FeedController.php` → ver endpoints a testar (task 4.5)
- `app/Services/AiAnalyzerService.php` → ver interface para mock (task 4.3)
- `app/Models/Event.php` → ver factories necessárias

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Atualizar `Handler.php` (ou `bootstrap/app.php` no Laravel 11):
   - Capturar `AuthenticationException`: retornar `{ message: 'Não autenticado.' }` com 401
   - Capturar `AuthorizationException`: retornar `{ message: 'Sem permissão.' }` com 403
   - Garantir que todas as respostas de API são JSON (header `Accept: application/json` ou detectar rota `/api/`)
2. Criar `EventFactory` (se não existir):
   - Gerar eventos com dados realistas
   - States: `relevante()`, `critico()`, `ultimas48h()`
3. Criar `FeedTest` (Feature):
   - `test_unauthenticated_returns_401`
   - `test_inactive_subscriber_returns_403`
   - `test_essencial_subscriber_gets_max_20_events`
   - `test_reservado_subscriber_gets_full_history`
   - `test_category_filter_works`
   - `test_admin_can_trigger_update`
4. Criar `AiAnalyzerServiceTest` (Unit):
   - Mock da Claude API para retornar JSON de análise
   - `test_sport_news_marked_as_not_relevant`
   - `test_sanctions_news_marked_as_relevant_high_score`
   - `test_invalid_json_response_handled_gracefully`

### Regras de negócio aplicáveis:
- Testes de feature devem usar `RefreshDatabase`
- A Claude API deve ser mockada nos testes para não gerar custos reais
- O error handler deve funcionar para TODAS as rotas `/api/*`

### Critérios de aceitação:
- [ ] `php artisan test --filter=FeedTest` passa todos os testes
- [ ] `php artisan test --filter=AiAnalyzerServiceTest` passa todos os testes
- [ ] Request sem `Accept: application/json` para rota 401 ainda retorna JSON
- [ ] `EventFactory` gera dados válidos
- [ ] `Handler.php` retorna JSON para erros 401, 403, 404 e 422 em rotas `/api/`

## Teste esperado
- Comando: `php artisan test --filter=FeedTest`
- Todos os testes devem passar em verde

média
~150 linhas de código | ~3 arquivos
