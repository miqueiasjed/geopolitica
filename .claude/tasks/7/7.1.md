# Task 7.1 – Migration, Model Conteudo e Seeder

## Objetivo
Criar a migration da tabela `conteudos` com índices FULLTEXT e UNIQUE, o Model `Conteudo` com scopes e geração de slug, e o Seeder com 9 conteúdos de teste.

## Tipo
backend-estrutura

- `database/migrations/xxxx_create_conteudos_table.php` → criar
- `app/Models/Conteudo.php` → criar
- `database/seeders/ConteudoSeeder.php` → criar

## Contexto necessário (arquivos para ler antes)

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

1. **Migration `conteudos`**:
   - `id`
   - `tipo` (string — `briefing` | `mapa` | `tese`)
   - `titulo` (string)
   - `slug` (string, unique)
   - `corpo` (longText — HTML)
   - `resumo` (text)
   - `regiao` (string, nullable)
   - `tags` (json, nullable — array de strings)
   - `tese_manchete` (string, nullable — só para tipo `tese`)
   - `plano_minimo` (string — `essencial` | `pro` | `reservado`)
   - `publicado` (boolean, default: false)
   - `publicado_em` (timestamp, nullable)
   - `timestamps`
   - Índice composto em `tipo`, `regiao`, `publicado`, `publicado_em`
   - FULLTEXT index em `titulo` e `corpo` (`$table->fullText(['titulo', 'corpo'])`)
   - UNIQUE index em `slug`

2. **Model `Conteudo`**:
   - `$fillable` com todos os campos
   - Casts: `tags` → array, `publicado` → boolean, `publicado_em` → datetime
   - Scope `publicados()`: `where('publicado', true)->whereNotNull('publicado_em')`
   - Scope `acessivelPor(string $role)`: filtra por `plano_minimo` conforme hierarquia e janela de 90 dias:
     - `assinante_essencial`: `plano_minimo = 'essencial'` + `publicado_em >= now()->subDays(90)`
     - `assinante_pro`: `plano_minimo IN ('essencial', 'pro')` + `publicado_em >= now()->subDays(90)`
     - `assinante_reservado` ou `admin`: sem filtro de plano ou data
   - Método estático `gerarSlug(string $titulo): string`: transforma título em slug + appenda timestamp único (`Str::slug($titulo) . '-' . now()->timestamp`)

3. **ConteudoSeeder**:
   - 9 conteúdos: 1 briefing + 1 mapa + 1 tese para cada nível de plano (`essencial`, `pro`, `reservado`)
   - `publicado = true`, `publicado_em` variando entre hoje e 100 dias atrás (para testar restrição de 90 dias)
   - Usar `Conteudo::gerarSlug()` para gerar slugs únicos

### Regras de negócio aplicáveis:
- Slug deve ser único e gerado com timestamp para evitar colisões
- FULLTEXT index é obrigatório para buscas via `MATCH AGAINST`
- A janela de 90 dias se aplica apenas a `essencial` e `pro`; `reservado` e `admin` veem histórico completo
- `plano_minimo = 'reservado'` é o nível mais restrito (só assinante_reservado e admin podem ver)

### Critérios de aceitação:
- [ ] `php artisan migrate` cria a tabela `conteudos` sem erros
- [ ] FULLTEXT index existe em `titulo` e `corpo`
- [ ] UNIQUE index existe em `slug`
- [ ] `Conteudo::publicados()->count()` executa sem erro
- [ ] `Conteudo::publicados()->acessivelPor('assinante_essencial')->get()` retorna apenas briefings com ≤90 dias
- [ ] `Conteudo::gerarSlug('Meu Titulo')` retorna slug válido com timestamp
- [ ] `php artisan db:seed --class=ConteudoSeeder` insere 9 registros
