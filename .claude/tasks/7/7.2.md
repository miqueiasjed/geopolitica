# Task 7.2 – ConteudoService e ConteudoPolicy

## Objetivo
Implementar toda a lógica de negócio da Biblioteca no `ConteudoService` e as regras de autorização por role na `ConteudoPolicy`.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/ConteudoService.php` → criar
- `app/Policies/ConteudoPolicy.php` → criar
- `app/Providers/AuthServiceProvider.php` → modificar (registrar Policy)

## Contexto necessário (arquivos para ler antes)
- `app/Models/Conteudo.php` → entender scopes disponíveis e método `gerarSlug()`
- `app/Services/EventoService.php` → ver padrão de retorno `['success', 'message', 'data']` adotado no projeto

## Skills necessárias
- laravel-arquitetura
- permissoes-e-sync

## Especificação

### O que fazer:

1. **`ConteudoService`**:

   - `listar(array $filtros, string $role): array`
     - Aplica scopes `publicados()` e `acessivelPor($role)`
     - Filtros opcionais: `q` (busca FULLTEXT via `whereRaw('MATCH(titulo, corpo) AGAINST(? IN BOOLEAN MODE)', [$q])`), `tipo`, `regiao`, `de` (data início), `ate` (data fim)
     - Paginação cursor-based: recebe `cursor` (id do último item) e `limite` (default: 20, max: 50)
     - Retorna `['data' => [...], 'next_cursor' => id|null]`
     - Usa `ConteudoCardResource` para formatar itens

   - `buscarPorSlug(string $slug, string $role): ?Conteudo`
     - Busca por slug em `publicados()`
     - Verifica se o usuário tem acesso pelo plano (lógica baseada em `plano_minimo` e janela de tempo)
     - Retorna `null` quando acesso insuficiente (o Controller retorna 404 — não revela que o conteúdo existe)

   - `criar(array $dados): Conteudo`
     - Gera slug via `Conteudo::gerarSlug($dados['titulo'])`
     - Cria e retorna o Model

   - `atualizar(Conteudo $conteudo, array $dados): Conteudo`
     - Atualiza os campos fornecidos
     - Se `publicado = true` e `publicado_em` ainda for null, define `publicado_em = now()`
     - Retorna o Model atualizado

   - `despublicar(Conteudo $conteudo): Conteudo`
     - Define `publicado = false` (não apaga o registro)
     - Retorna o Model atualizado

2. **`ConteudoPolicy`**:
   - `verPublico(User $user, Conteudo $conteudo): bool`
     - Verifica se o usuário tem role suficiente para o `plano_minimo` do conteúdo
     - Hierarquia: `assinante_essencial` < `assinante_pro` < `assinante_reservado` = `admin`
   - `gerenciarAdmin(User $user): bool`
     - Retorna `true` apenas se `$user->hasRole('admin')`

3. **Registrar a Policy** em `AuthServiceProvider`:
   - `Conteudo::class => ConteudoPolicy::class`

### Regras de negócio aplicáveis:
- O Service nunca chama `Auth::user()` — recebe `$role` como parâmetro
- HTTP 404 (não 403) quando acesso insuficiente — não revelar que o conteúdo existe
- `despublicar()` usa soft-delete lógico (apenas `publicado = false`), nunca deleta fisicamente
- Paginação cursor-based: `WHERE id > $cursor ORDER BY id ASC LIMIT $limite+1` (o +1 detecta se há próxima página)

### Critérios de aceitação:
- [ ] `ConteudoService::listar(['tipo' => 'briefing'], 'assinante_essencial')` retorna apenas briefings públicos dentro de 90 dias
- [ ] `ConteudoService::listar(['q' => 'guerra'], 'assinante_pro')` usa FULLTEXT sem erro
- [ ] `ConteudoService::buscarPorSlug('slug-reservado', 'assinante_pro')` retorna `null`
- [ ] `ConteudoService::criar([...])` gera slug único
- [ ] `ConteudoService::despublicar($conteudo)` define `publicado = false` sem deletar
- [ ] Policy registrada: `Gate::check('verPublico', $conteudo)` funciona
