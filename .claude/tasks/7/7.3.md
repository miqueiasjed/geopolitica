# Task 7.3 – FormRequests e Resources

## Objetivo
Criar os FormRequests de validação (listagem, criação e atualização) e os Resources de transformação de resposta (`ConteudoResource` completo e `ConteudoCardResource` resumido).

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Http/Requests/BibliotecaFiltroRequest.php` → criar
- `app/Http/Requests/CriarConteudoRequest.php` → criar
- `app/Http/Requests/AtualizarConteudoRequest.php` → criar
- `app/Http/Resources/ConteudoResource.php` → criar
- `app/Http/Resources/ConteudoCardResource.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/Conteudo.php` → ver campos disponíveis e tipos esperados
- `app/Http/Resources/EventoResource.php` → ver padrão de Resource adotado no projeto

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

1. **`BibliotecaFiltroRequest`**:
   - `q` (string, nullable, max:200) — termo de busca full-text
   - `tipo` (string, nullable, in:`briefing`,`mapa`,`tese`)
   - `regiao` (string, nullable, max:100)
   - `de` (date, nullable) — data início
   - `ate` (date, nullable, after_or_equal:de) — data fim
   - `cursor` (integer, nullable, min:1) — id do último item da página anterior
   - `limite` (integer, nullable, min:1, max:50) — itens por página

2. **`CriarConteudoRequest`**:
   - `titulo` (required, string, max:255)
   - `tipo` (required, string, in:`briefing`,`mapa`,`tese`)
   - `corpo` (required, string) — HTML
   - `resumo` (required, string, max:500)
   - `regiao` (nullable, string, max:100)
   - `tags` (nullable, array)
   - `tags.*` (string, max:50)
   - `tese_manchete` (nullable, string, max:255) — obrigatório quando `tipo = tese`; usar `Rule::requiredIf`
   - `plano_minimo` (required, string, in:`essencial`,`pro`,`reservado`)
   - `publicado` (boolean, default: false)
   - `authorize()`: retorna `$this->user()->hasRole('admin')`

3. **`AtualizarConteudoRequest`**:
   - Mesmos campos de `CriarConteudoRequest`, mas todos `nullable` (PATCH parcial)
   - `authorize()`: retorna `$this->user()->hasRole('admin')`

4. **`ConteudoResource`** (resposta completa para `GET /api/biblioteca/{slug}`):
   - `id`, `tipo`, `titulo`, `slug`, `corpo`, `resumo`
   - `regiao`, `tags`, `tese_manchete`, `plano_minimo`
   - `publicado_em` (formato ISO 8601)

5. **`ConteudoCardResource`** (resposta resumida para listagem):
   - `id`, `tipo`, `titulo`, `slug`, `resumo`
   - `regiao`, `tags`, `tese_manchete`
   - `publicado_em` (formato ISO 8601)
   - Omite `corpo` e `plano_minimo`

### Regras de negócio aplicáveis:
- `tese_manchete` é obrigatório quando `tipo = tese`
- FormRequests de admin verificam role na `authorize()`
- `ConteudoCardResource` nunca expõe `corpo` (HTML completo) na listagem

### Critérios de aceitação:
- [ ] `POST /api/admin/conteudos` sem `titulo` retorna HTTP 422 com erro de validação
- [ ] `POST /api/admin/conteudos` com `tipo = tese` e sem `tese_manchete` retorna HTTP 422
- [ ] `ConteudoCardResource` não contém chave `corpo` no JSON
- [ ] `ConteudoResource` contém chave `corpo` no JSON
- [ ] `GET /api/biblioteca?limite=101` retorna HTTP 422
