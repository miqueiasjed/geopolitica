# Task 7.4 – BibliotecaController e ConteudoController (endpoints públicos)

## Objetivo
Implementar os dois controllers de leitura pública da Biblioteca: listagem com busca/filtros e leitura individual com controle de acesso por plano, além das rotas correspondentes.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Api/BibliotecaController.php` → criar
- `app/Http/Controllers/Api/ConteudoController.php` → criar
- `routes/api.php` → modificar (adicionar rotas públicas de biblioteca)

## Contexto necessário (arquivos para ler antes)

- `app/Http/Controllers/Api/FeedController.php` → ver padrão de controller fino adotado no projeto
- `routes/api.php` → ver estrutura de rotas e middlewares existentes

## Skills necessárias

- permissoes-e-sync

## Especificação

### O que fazer:

1. **`BibliotecaController`**:
   - `index(BibliotecaFiltroRequest $request): JsonResponse`
     - Obtém a role do usuário autenticado: `$role = $request->user()->getRoleNames()->first()`
     - Chama `ConteudoService::listar($request->validated(), $role)`
     - Retorna `response()->json(['data' => $resultado['data'], 'next_cursor' => $resultado['next_cursor']])`
   - Middleware no construtor: `auth:sanctum`

2. **`ConteudoController`**:
   - `show(string $slug): JsonResponse`
     - Obtém role do usuário autenticado
     - Chama `ConteudoService::buscarPorSlug($slug, $role)`
     - Se retornar `null` → `abort(404)`
     - Se encontrado → retorna `new ConteudoResource($conteudo)`
   - Middleware no construtor: `auth:sanctum`

3. **Rotas em `routes/api.php`** (grupo com `auth:sanctum`):
   ```php
   Route::middleware('auth:sanctum')->group(function () {
       Route::get('/biblioteca', [BibliotecaController::class, 'index']);
       Route::get('/biblioteca/{slug}', [ConteudoController::class, 'show']);
   });
   ```

### Regras de negócio aplicáveis:
- Controllers devem ser finos: zero lógica de negócio, apenas orquestração
- HTTP 404 (não 403) quando acesso insuficiente ao conteúdo — não revelar que existe
- Role obtida diretamente do usuário autenticado pelo Sanctum

### Critérios de aceitação:
- [ ] `GET /api/biblioteca` com token válido de `assinante_essencial` retorna lista paginada
- [ ] `GET /api/biblioteca?tipo=mapa` com token de `assinante_essencial` retorna lista vazia (sem acesso a mapas)
- [ ] `GET /api/biblioteca/{slug-reservado}` com token de `assinante_pro` retorna HTTP 404
- [ ] `GET /api/biblioteca/{slug-essencial}` com token de `assinante_essencial` retorna HTTP 200 com `ConteudoResource`
- [ ] `GET /api/biblioteca` sem token retorna HTTP 401
- [ ] `GET /api/biblioteca?limite=20` retorna `next_cursor` quando há mais itens
