# Task 8.3 – SearchBar, FilterBar e ContentReader

## Objetivo
Criar a barra de busca com debounce, a barra de filtros combinados e o componente de leitura de conteúdo HTML renderizado com estilos tipográficos.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `src/components/biblioteca/SearchBar.tsx` → criar
- `src/components/biblioteca/FilterBar.tsx` → criar
- `src/components/biblioteca/ContentReader.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `src/types/biblioteca.ts` → tipos `BibliotecaFiltros` e `TipoConteudo`
- `src/components/feed/FeedFilters.tsx` → ver padrão de filtros existente no projeto (se houver)

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

1. **`SearchBar.tsx`** (props: `value: string`, `onChange: (v: string) => void`, `placeholder?: string`):
   - Input com ícone de lupa
   - Debounce de 500ms: usar `useEffect` + `setTimeout` interno ou hook `useDebounce`
   - Chama `onChange` apenas após 500ms sem nova digitação
   - Botão "×" para limpar (visível apenas quando `value !== ''`)
   - Estilo: `bg-zinc-900 border border-zinc-800 rounded-lg`

2. **`FilterBar.tsx`** (props: `filtros: BibliotecaFiltros`, `onChange: (f: BibliotecaFiltros) => void`):
   - Select de `tipo`: opções Todos / Briefing / Mapa / A Tese
   - Select de `regiao`: opções pré-definidas (América do Sul, Europa, Oriente Médio, Ásia, África, América do Norte, Global)
   - Inputs de data `de` e `ate` para período
   - Botão "Limpar filtros" — visível apenas quando algum filtro está ativo (`tipo || regiao || de || ate`)
   - Ao clicar "Limpar filtros": chama `onChange({})` resetando tudo
   - Layout: flex row, gap-2, scroll horizontal em mobile

3. **`ContentReader.tsx`** (props: `corpo: string`, `titulo: string`):
   - Renderiza `corpo` (HTML gerado pelo TipTap) via `dangerouslySetInnerHTML`
   - Container com classe `prose prose-invert prose-zinc max-w-none` (Tailwind Typography)
   - Customizações CSS: `prose-h2:text-white prose-a:text-blue-400 prose-strong:text-white`
   - Título `h1` acima do corpo em fonte grande
   - Wrapper com `max-w-3xl mx-auto`

### Regras de negócio aplicáveis:
- Debounce de 500ms obrigatório na busca — não disparar a cada tecla
- "Limpar filtros" só aparece quando há filtro ativo
- `ContentReader` usa `prose-invert` para tema escuro do projeto

### Critérios de aceitação:
- [ ] `SearchBar` não dispara `onChange` a cada tecla — só após 500ms
- [ ] Botão "×" limpa o campo e chama `onChange('')`
- [ ] "Limpar filtros" ausente quando `filtros = {}`
- [ ] "Limpar filtros" presente quando qualquer filtro está ativo
- [ ] `ContentReader` renderiza HTML com estilos Tailwind Typography
- [ ] TypeScript compila sem erros
