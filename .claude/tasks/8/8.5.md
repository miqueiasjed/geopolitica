# Task 8.5 – AdminEditor (TipTap) e AdminNovoConteudo

## Objetivo
Criar o editor TipTap com barra de formatação e a página admin de criação/publicação de novo conteúdo.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `src/components/biblioteca/AdminEditor.tsx` → criar
- `src/pages/admin/AdminNovoConteudo.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `src/types/biblioteca.ts` → tipos `TipoConteudo` e `PlanoMinimo`
- `src/pages/admin/` → ver estrutura de páginas admin existentes

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

1. **`AdminEditor.tsx`** (props: `value: string`, `onChange: (html: string) => void`, `placeholder?: string`):
   - Usa `useEditor` do `@tiptap/react` com extensões:
     - `StarterKit` (inclui Bold, Italic, Heading, HorizontalRule)
     - `Placeholder.configure({ placeholder: props.placeholder })`
   - Barra de ferramentas acima do editor:
     - Botão **N** (negrito) — `editor.chain().focus().toggleBold().run()`
     - Botão *I* (itálico) — `editor.chain().focus().toggleItalic().run()`
     - Botão **H2** (subtítulo) — `editor.chain().focus().toggleHeading({ level: 2 }).run()`
     - Botão **—** (divisor horizontal) — `editor.chain().focus().setHorizontalRule().run()`
   - Cada botão recebe classe `is-active` quando o formato está ativo no cursor
   - `EditorContent` com min-height de 300px
   - Chama `onChange(editor.getHTML())` a cada mudança
   - Estilo: `bg-zinc-900 border border-zinc-800 rounded-lg`

2. **`AdminNovoConteudo.tsx`** (rota: `/admin/novo-conteudo`):
   - Formulário com os campos:
     - `tipo` (Select: Briefing / Mapa / A Tese)
     - `titulo` (Input)
     - `tese_manchete` (Input) — visível apenas quando `tipo === 'tese'`
     - `regiao` (Input)
     - `tags` (Input de texto livre, separado por vírgulas)
     - `resumo` (Textarea, max 500 chars com contador)
     - `plano_minimo` (Select: Essencial / Pro / Reservado)
     - `corpo` → `<AdminEditor />`
     - Toggle "Publicar agora" (boolean, default off) — quando on, envia `publicado: true`
   - Submit: `POST /api/admin/conteudos` via `useMutation` do TanStack Query
   - Tags: converter string separada por vírgulas em array antes de enviar
   - Após sucesso: navegar para `/admin/biblioteca` e exibir toast de sucesso
   - Validação client-side: `titulo` e `corpo` obrigatórios; `tese_manchete` obrigatório se `tipo === 'tese'`

### Regras de negócio aplicáveis:
- Campo `tese_manchete` só aparece quando `tipo === 'tese'`
- Tags são enviadas como array de strings (split por vírgula + trim)
- Toggle "Publicar agora" controla o campo `publicado` no payload

### Critérios de aceitação:
- [ ] Editor TipTap renderiza e botões de formatação funcionam
- [ ] Botões da barra ficam destacados quando o formato está ativo
- [ ] Campo `tese_manchete` aparece/desaparece conforme `tipo`
- [ ] Submit chama `POST /api/admin/conteudos` com payload correto
- [ ] Toggle "Publicar agora" inclui `publicado: true` no payload quando ativo
- [ ] TypeScript compila sem erros
