# Task 12.2 – TimelineController, TimelineDetailController, rotas e cache Redis

## Objetivo
Implementar os dois endpoints da timeline — lista combinada (crises + eventos M01) e detalhe de crise — com cache Redis de 24h para os dados históricos estáticos.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Api/TimelineController.php` → criar
- `app/Http/Controllers/Api/TimelineDetailController.php` → criar
- `routes/api.php` → modificar (adicionar rotas de timeline)

## Contexto necessário (arquivos para ler antes)
- `app/Models/CriseHistorica.php` → scopes disponíveis (12.1)
- `app/Models/Event.php` → campos disponíveis (do Plano 4)
- `routes/api.php` → ver middleware e prefixos já definidos

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**`TimelineController@index` — `GET /api/timeline`:**

Parâmetros opcionais:
- `?periodo_inicio=1990&periodo_fim=2024` — filtro de período (anos)
- `?categoria=guerra` — filtro de categoria (para crises históricas)

Retorno combinado:
```json
{
  "crises": [...],
  "eventos": [...]
}
```

**Crises históricas:**
- Buscar do Redis: `Cache::remember('timeline_crises_{hash_params}', 86400, fn() => ...)`
- Hash params: `md5(serialize([$periodoinicio, $periodofim, $categoria]))`
- Aplicar scopes: `scopePorPeriodo` e `scopePorCategoria` conforme parâmetros
- Campos a retornar: `id`, `titulo`, `slug`, `ano`, `data_inicio`, `data_fim`, `categorias`, `content_slug`
- Ordenar por `data_inicio ASC`

**Eventos do M01 (faixa inferior):**
- Sem cache (dado em tempo real)
- Buscar `Event` dos últimos 365 dias (ou filtrado pelo período)
- Campos: `id`, `titulo`, `nivel_tensao`, `created_at`
- Ordenar por `created_at ASC`
- Limitar a 100 eventos mais recentes

**`TimelineDetailController@show` — `GET /api/timeline/crise/{slug}`:**
- Cache Redis 24h: `Cache::remember("timeline_crise_{slug}", 86400, fn() => ...)`
- Buscar por `slug`; 404 se não encontrar
- Retornar todos os campos do `CriseHistorica`
- Incluir `content_slug` para link na Biblioteca

**Rotas em `routes/api.php`:**
```php
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/timeline', [TimelineController::class, 'index']);
    Route::get('/timeline/crise/{slug}', [TimelineDetailController::class, 'show']);
});
```

### Regras de negócio aplicáveis:
- Cache de 24h para dados históricos (são estáticos após o seeder)
- Eventos do M01 sempre frescos (sem cache)
- Filtros de período e categoria são opcionais (sem filtro = retornar tudo)

### Critérios de aceitação:
- [ ] `GET /api/timeline` retorna objeto com `crises` e `eventos`
- [ ] `GET /api/timeline?categoria=guerra` retorna apenas crises da categoria
- [ ] `GET /api/timeline/crise/invasao-russa-da-ucrania` retorna detalhes completos
- [ ] Cache Redis com TTL de 24h funcionando (verificar no Redis CLI)
- [ ] 404 para slug inexistente
