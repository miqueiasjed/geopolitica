# Task 22.2 – Blade templates PDF: layout base, briefing e alerta

## Objetivo
Criar os três primeiros templates Blade para geração de PDFs: o layout base com capa/header/footer e os templates de briefing e alerta preditivo.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/resources/views/pdf/layout.blade.php` → criar
- `backend/resources/views/pdf/briefing.blade.php` → criar
- `backend/resources/views/pdf/alerta.blade.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/Conteudo.php` → entender campos disponíveis para o template de briefing
- `backend/app/Models/AlertaPreditivo.php` → entender campos disponíveis para o template de alerta

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

#### `pdf/layout.blade.php`
Template base com `@yield('content')` que encapsula todos os outros. Deve conter:
- `<meta charset="utf-8">` e CSS inline (DomPDF não carrega CSS externo por padrão)
- **Header:** logo "Geopolítica para Investidores" (texto) à esquerda + data de geração à direita. Se `$companyLogo` for passado (URL string), exibir `<img>` com height 28px no lugar do logo texto.
- **Footer:** disclaimer regulatório fixo: _"Este documento tem caráter exclusivamente analítico e informativo. Não constitui recomendação de investimento, consultoria financeira ou indicação de compra e venda de ativos. © Geopolítica para Investidores."_
- Fonte body: Georgia, serif. Tamanho base: 10pt. Cor de destaque: `#C9B882`. Background branco, texto `#1a1a1b`.
- Separador header: `border-bottom: 2px solid #C9B882`.

#### `pdf/briefing.blade.php`
Recebe variável `$conteudo` (Model Conteudo). Exibe:
- Tag de categoria (uppercase, tracking, dourado) + tipo de conteúdo
- Título em serif, 18pt, negrito
- Subtítulo se houver
- Data de publicação formatada em pt-BR
- Corpo do conteúdo em HTML (campo `corpo` ou `descricao`) com `{!! $conteudo->corpo !!}`
- Rodapé de seção com nome do autor se disponível

#### `pdf/alerta.blade.php`
Recebe variável `$alerta` (Model AlertaPreditivo). Exibe:
- Badge de nível (`CRÍTICO` / `ALTO` / `MODERADO`) com cor: crítico = vermelho, alto = laranja, moderado = amarelo
- Título do alerta em serif
- Campos: `sinais_detectados` (lista), `analise_ia`, `nivel_confianca`, `probabilidade_ocorrencia`
- Data de criação

### Regras de negócio aplicáveis:
- Todo CSS deve ser inline ou em `<style>` no `<head>` — DomPDF não carrega arquivos externos.
- Imagens de logo B2B devem usar `src="{{ $companyLogo }}"` — `enable_remote` está habilitado.
- Não usar Tailwind ou classes CSS externas — somente CSS puro no `<style>`.

### Critérios de aceitação:
- [ ] `pdf/layout.blade.php` com header (logo/data), seção de conteúdo e footer com disclaimer
- [ ] Header exibe logo B2B quando `$companyLogo` é passado, logo texto caso contrário
- [ ] `pdf/briefing.blade.php` renderiza título, tipo, corpo e data corretamente
- [ ] `pdf/alerta.blade.php` renderiza badge de nível, título, sinais e análise
- [ ] Nenhum CSS externo referenciado (apenas inline/`<style>`)
