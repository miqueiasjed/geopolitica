# Task 22.4 – PdfTemplateService: busca de dados e geração do PDF

## Objetivo
Criar o `PdfTemplateService` responsável por buscar os dados corretos no banco conforme o tipo solicitado, selecionar o template Blade correspondente, e gerar e retornar o buffer do PDF usando DomPDF.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `backend/app/Services/PdfTemplateService.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/Conteudo.php` → campos e relacionamentos
- `backend/app/Models/AlertaPreditivo.php` → campos disponíveis
- `backend/app/Models/PerfilPais.php` → campos disponíveis
- `backend/app/Models/ChatMensagem.php` → campos e relacionamento com sessão
- `backend/app/Models/LicencaB2b.php` → campos company_name e company_logo_url

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
Criar `App\Services\PdfTemplateService` com o método principal `gerar(string $tipo, string $id, ?string $companySlug, int $userId): string` que retorna o buffer do PDF.

Estrutura do service:

```php
public function gerar(string $tipo, string $id, ?string $companySlug, int $userId): string
{
    $dados   = $this->buscarDados($tipo, $id, $userId);
    $logoB2B = $this->buscarLogoB2B($companySlug);
    $view    = $this->selecionarView($tipo, $dados, $logoB2B);
    return $this->gerarPdf($view);
}
```

#### `buscarDados(string $tipo, string $id, int $userId): array`
- `'briefing'` → busca em `Conteudo` onde `id = $id` e publicado
- `'alerta'` → busca em `AlertaPreditivo` onde `id = $id`
- `'pais'` → busca em `PerfilPais` onde `id = $id` ou `codigo = $id`
- `'chat'` → busca em `ChatMensagem` onde `id = $id`, valida que a sessão pertence ao `$userId`
- Lança `\Illuminate\Database\Eloquent\ModelNotFoundException` se não encontrado

#### `buscarLogoB2B(?string $companySlug): ?string`
- Se `$companySlug` for nulo, retorna `null`
- Busca `LicencaB2b::where('company_slug', $companySlug)->first()`
- Retorna a URL do logo (`company_logo_url`) ou `null`

#### `selecionarView(string $tipo, array $dados, ?string $logoB2B): string`
- Usa `view("pdf.{$tipo}", array_merge($dados, ['companyLogo' => $logoB2B]))->render()`
- Mapeia tipo para view: `briefing` → `pdf.briefing`, `alerta` → `pdf.alerta`, etc.

#### `gerarPdf(string $html): string`
- Usa `\Barryvdh\DomPDF\Facade\Pdf::loadHTML($html)->output()`
- Retorna o buffer como string

### Regras de negócio aplicáveis:
- Para `chat`, validar que `$sessao->user_id === $userId` — usuário só exporta suas próprias conversas.
- Para `briefing`, verificar que o conteúdo está publicado (não exportar rascunhos).
- Tipos inválidos devem lançar `\InvalidArgumentException`.

### Critérios de aceitação:
- [ ] `gerar()` retorna string binária do PDF para os 4 tipos
- [ ] `buscarDados()` lança `ModelNotFoundException` para IDs inexistentes
- [ ] Validação de ownership para tipo `chat`
- [ ] Logo B2B incluído quando `company_slug` é válido
- [ ] Tipo inválido lança `InvalidArgumentException`
