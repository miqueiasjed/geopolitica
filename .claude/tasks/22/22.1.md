# Task 22.1 – Instalar e configurar barryvdh/laravel-dompdf

## Objetivo
Instalar o pacote `barryvdh/laravel-dompdf` no backend Laravel, publicar e ajustar a configuração para gerar PDFs A4 com suporte completo a UTF-8 e fonte Georgia.

## Tipo
config

## Arquivos a criar/modificar
- `backend/config/dompdf.php` → modificar (publicado pelo pacote, ajustar charset, paper, fontes)
- `backend/composer.json` → modificar (adicionar dependência)

## Contexto necessário (arquivos para ler antes)
- `backend/composer.json` → ver versão do Laravel e dependências existentes

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Rodar `composer require barryvdh/laravel-dompdf` no diretório `backend/`.
2. Publicar o config com `php artisan vendor:publish --provider="Barryvdh\DomPDF\ServiceProvider"`.
3. Editar `config/dompdf.php` com os seguintes valores:
   - `'default_paper_size' => 'a4'`
   - `'default_paper_orientation' => 'portrait'`
   - `'default_font' => 'serif'`
   - `'enable_unicode' => true`
   - `'enable_remote' => true` (necessário para carregar logos B2B por URL)
   - `'chroot' => realpath(base_path())` (segurança para assets locais)
4. Criar o diretório `backend/resources/views/pdf/` para os templates Blade (será populado na próxima task).

### Regras de negócio aplicáveis:
- `enable_remote => true` é necessário para renderizar logos de empresas B2B hospedados em CDN externo.
- Configuração de charset UTF-8 garante suporte a caracteres especiais do português.

### Critérios de aceitação:
- [ ] `composer require barryvdh/laravel-dompdf` executado com sucesso
- [ ] `config/dompdf.php` publicado e ajustado com paper A4, UTF-8 e remote habilitado
- [ ] Diretório `resources/views/pdf/` criado
- [ ] `php artisan config:clear` executa sem erros

## Teste esperado
- Verificação manual: `php artisan config:show dompdf` retorna os valores configurados
- `composer show barryvdh/laravel-dompdf` confirma instalação
