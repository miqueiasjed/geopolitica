# Task 26.1 – Migration e Model Produto

## Objetivo
Criar a migration da tabela `produtos` e o Model `Produto` com fillable, casts e scopes, fornecendo a estrutura de dados base para todo o módulo Multi-Produtos.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/database/migrations/2026_05_18_000001_create_produtos_table.php` → criar
- `backend/app/Models/Produto.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/AssinanteAddon.php` → entender padrões de Model existentes (fillable, casts, constantes)
- `backend/database/migrations/` → ver convenção de nomenclatura e estrutura

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Criar migration com colunas:
- `id` (bigIncrements)
- `chave` (string, unique) — ex: `elections`, `war`
- `nome` (string)
- `descricao` (text, nullable)
- `preco_label` (string, nullable) — ex: "R$ 49/mês"
- `link_compra` (string, nullable)
- `link_reativar` (string, nullable)
- `ativo` (boolean, default true)
- `ordem` (unsignedSmallInteger, default 0)
- `timestamps`

Model `Produto`:
- `$fillable` com todas as colunas acima (exceto id, timestamps)
- `$casts`: `ativo` → `boolean`, `ordem` → `integer`
- Scope `scopeAtivo(Builder $query)` — where('ativo', true)
- Scope `scopeOrdenado(Builder $query)` — orderBy('ordem')

### Regras de negócio aplicáveis:
- `chave` deve ser imutável após criação (validado no FormRequest, não no Model)
- `ativo` e `ordem` têm valores padrão; não são obrigatórios no create

### Critérios de aceitação:
- [ ] `php artisan migrate` executa sem erros
- [ ] `php artisan migrate:rollback` executa sem erros
- [ ] `Produto::ativo()->ordenado()->get()` retorna instâncias do Model correto
- [ ] Casts aplicados: `$produto->ativo` é booleano, `$produto->ordem` é inteiro
