# Task 26.6 – AdminProdutoController destroy com proteção 422

## Objetivo
Implementar o método `destroy` no `AdminProdutoController` que bloqueia exclusão de produtos com addons ativos, retornando 422 com mensagem descritiva.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Api/Admin/AdminProdutoController.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/Api/Admin/AdminProdutoController.php` → estado após Task 26.5
- `backend/app/Models/AssinanteAddon.php` → ver campos e relação com produto via `addon_key`

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Adicionar método `destroy(Produto $produto)`:

```php
public function destroy(Produto $produto): JsonResponse
{
    $totalAtivos = AssinanteAddon::where('addon_key', $produto->chave)
        ->where('status', 'ativo')
        ->count();

    if ($totalAtivos > 0) {
        return response()->json([
            'message' => "Não é possível excluir: {$totalAtivos} assinante(s) com este addon ativo.",
        ], 422);
    }

    $produto->delete();

    return response()->json(null, 204);
}
```

### Regras de negócio aplicáveis:
- Produto com 1 ou mais addons `status=ativo` **não pode** ser excluído
- A verificação é por `AssinanteAddon.addon_key` === `Produto.chave`
- Retornar 422 com contagem de assinantes afetados
- Exclusão bem-sucedida retorna 204 (no content)

### Critérios de aceitação:
- [ ] `DELETE /admin/produtos/{id}` com addons ativos retorna 422 com mensagem
- [ ] `DELETE /admin/produtos/{id}` sem addons ativos exclui e retorna 204
- [ ] Produto com addons cancelados/expirados (não ativos) pode ser excluído
- [ ] Usuário não-admin recebe 403
