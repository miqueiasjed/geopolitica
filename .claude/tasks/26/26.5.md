# Task 26.5 – AdminProdutoController (index, store, update) + Rotas Admin

## Objetivo
Criar o controller admin de produtos com operações de listagem, criação e atualização, e registrar as rotas no grupo admin.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Api/Admin/AdminProdutoController.php` → criar
- `backend/routes/api.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/Api/Admin/` → ver padrão de controllers admin existentes
- `backend/routes/api.php` → ver estrutura do grupo admin
- `backend/app/Http/Requests/Admin/CriarProdutoRequest.php` → confirmar interface (Task 26.4)
- `backend/app/Http/Requests/Admin/AtualizarProdutoRequest.php` → confirmar interface (Task 26.4)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**AdminProdutoController:**
- `index()` → `Produto::ordenado()->get()` (inclui inativos para admin ver todos)
- `store(CriarProdutoRequest $request)` → `Produto::create($request->validated())` → retornar 201
- `update(AtualizarProdutoRequest $request, Produto $produto)` → `$produto->update($request->validated())` → retornar 200

**Rotas** no grupo admin (`middleware(['auth:sanctum', 'role:admin'])`):
```
GET    /admin/produtos          → AdminProdutoController@index
POST   /admin/produtos          → AdminProdutoController@store
PUT    /admin/produtos/{produto} → AdminProdutoController@update
DELETE /admin/produtos/{produto} → AdminProdutoController@destroy
```

### Regras de negócio aplicáveis:
- Admin vê todos os produtos (inclusive inativos) no index
- Ordenar por `ordem` no index
- `store` retorna status 201 com o produto criado
- `update` retorna o produto atualizado

### Critérios de aceitação:
- [ ] `GET /admin/produtos` retorna todos os produtos (ativos e inativos)
- [ ] `POST /admin/produtos` com dados válidos cria produto e retorna 201
- [ ] `POST /admin/produtos` com `chave` duplicada retorna 422
- [ ] `PUT /admin/produtos/{id}` atualiza produto e retorna 200
- [ ] Usuário não autenticado recebe 401
- [ ] Usuário sem role admin recebe 403
