# Task 26.2 – ProdutoSeeder + DatabaseSeeder + AssinanteAddon::FONTES

## Objetivo
Criar o `ProdutoSeeder` que semeia `elections` e `war` a partir de env vars, registrá-lo no `DatabaseSeeder`, e adicionar `'manual'` às fontes válidas do `AssinanteAddon`.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/database/seeders/ProdutoSeeder.php` → criar
- `backend/database/seeders/DatabaseSeeder.php` → modificar
- `backend/app/Models/AssinanteAddon.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/database/seeders/DatabaseSeeder.php` → ver como seeders são registrados
- `backend/app/Models/AssinanteAddon.php` → localizar constante FONTES existente

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**ProdutoSeeder** — usar `updateOrCreate(['chave' => $chave], [...dados])` para ser idempotente:
```php
$produtos = [
    [
        'chave'         => 'elections',
        'nome'          => 'Monitor Eleitoral',
        'descricao'     => null,
        'preco_label'   => null,
        'link_compra'   => env('LASTLINK_URL_ELECTIONS', null),
        'link_reativar' => env('LASTLINK_REATIVAR_ELECTIONS', null),
        'ativo'         => true,
        'ordem'         => 1,
    ],
    [
        'chave'         => 'war',
        'nome'          => 'Monitor de Guerra',
        'descricao'     => null,
        'preco_label'   => null,
        'link_compra'   => env('LASTLINK_URL_WAR', null),
        'link_reativar' => env('LASTLINK_REATIVAR_WAR', null),
        'ativo'         => true,
        'ordem'         => 2,
    ],
];
```

**DatabaseSeeder** — chamar `$this->call(ProdutoSeeder::class)` após os seeders existentes.

**AssinanteAddon** — localizar `const FONTES = [...]` e adicionar `'manual'` ao array.

### Regras de negócio aplicáveis:
- Seeder deve ser idempotente (`updateOrCreate`) — pode rodar múltiplas vezes
- Links lidos de env vars (podem ser null se não configuradas)
- `'manual'` deve ser uma fonte válida para addons criados pelo admin

### Critérios de aceitação:
- [ ] `php artisan db:seed --class=ProdutoSeeder` executa sem erros
- [ ] Executar o seeder duas vezes não duplica registros
- [ ] `AssinanteAddon::FONTES` inclui `'manual'`
- [ ] `php artisan db:seed` chama `ProdutoSeeder`

## Teste esperado
- Comando: `php artisan db:seed --class=ProdutoSeeder`
- Verificação: `php artisan tinker` → `App\Models\Produto::count()` retorna 2
