# Task 26.4 – FormRequests CriarProdutoRequest e AtualizarProdutoRequest

## Objetivo
Criar os dois FormRequests de validação para o CRUD admin de produtos, com regra de unicidade na criação e campo `chave` read-only na atualização.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `backend/app/Http/Requests/Admin/CriarProdutoRequest.php` → criar
- `backend/app/Http/Requests/Admin/AtualizarProdutoRequest.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Requests/` → ver padrão de FormRequest existente no projeto
- `backend/app/Models/Produto.php` → ver campos do modelo

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
**CriarProdutoRequest:**
- `authorize()` → verificar se usuário tem role `admin`
- Regras:
  - `chave`: `required|string|regex:/^[a-z0-9_-]+$/|unique:produtos,chave`
  - `nome`: `required|string|max:255`
  - `descricao`: `nullable|string`
  - `preco_label`: `nullable|string|max:100`
  - `link_compra`: `nullable|url|max:500`
  - `link_reativar`: `nullable|url|max:500`
  - `ativo`: `boolean`
  - `ordem`: `integer|min:0`

**AtualizarProdutoRequest:**
- `authorize()` → verificar se usuário tem role `admin`
- Regras iguais, exceto:
  - `chave`: **ausente** (não permitir alteração)
  - `nome`: `sometimes|required|string|max:255`
  - Demais campos com `sometimes`

### Regras de negócio aplicáveis:
- `chave` segue o padrão `^[a-z0-9_-]+$` (apenas minúsculas, números, hífens e underscores)
- `chave` é único no create; não pode ser alterado no update
- Links devem ser URLs válidas quando fornecidos

### Critérios de aceitação:
- [ ] `chave` com caracteres maiúsculos é rejeitado com 422
- [ ] `chave` duplicado no create é rejeitado com 422
- [ ] Request de update sem `chave` é aceito normalmente
- [ ] `link_compra` inválido (não URL) é rejeitado com 422
- [ ] Usuário não-admin recebe 403
