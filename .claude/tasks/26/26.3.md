# Task 26.3 – MeusProdutosController + Rota GET /api/meus-produtos

## Objetivo
Criar o endpoint autenticado que retorna a lista de produtos com `status_usuario` calculado individualmente para o usuário logado.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Api/MeusProdutosController.php` → criar
- `backend/routes/api.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/Produto.php` → entender scopes disponíveis
- `backend/app/Models/AssinanteAddon.php` → entender método `temAddon()` e estrutura
- `backend/app/Models/Assinante.php` → ver método `temAddon()` existente
- `backend/routes/api.php` → ver estrutura de grupos de rota existentes

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Controller `MeusProdutosController` com único método `index()`:

Lógica de `status_usuario` para cada produto:
1. Se o usuário é **admin** → `'ativo'`
2. Se `$assinante->temAddon($produto->chave)` → `'ativo'`
3. Se existe registro em `AssinanteAddon` para o user+chave → retornar `status` do último registro (`cancelado`, `expirado`, `reembolsado`)
4. Sem registro → `null`

Resposta JSON por produto:
```json
{
  "chave": "elections",
  "nome": "Monitor Eleitoral",
  "descricao": null,
  "preco_label": null,
  "link_compra": "https://...",
  "link_reativar": "https://...",
  "ativo": true,
  "ordem": 1,
  "status_usuario": "ativo"
}
```

Retornar apenas produtos `ativo=true`, ordenados por `ordem`.

**Rota:** `GET /api/meus-produtos` dentro do grupo `auth:sanctum`.

### Regras de negócio aplicáveis:
- Somente produtos `ativo=true` são retornados
- Admin sempre recebe `status_usuario: 'ativo'` em todos os produtos
- `status_usuario: null` significa que o usuário nunca adquiriu o produto
- Usar `Produto::ativo()->ordenado()->get()` para a query

### Critérios de aceitação:
- [ ] Usuário não autenticado recebe 401
- [ ] Admin recebe `status_usuario: 'ativo'` em todos os produtos
- [ ] Assinante com addon ativo recebe `status_usuario: 'ativo'` no produto correspondente
- [ ] Assinante com addon cancelado recebe `status_usuario: 'cancelado'`
- [ ] Assinante sem registro recebe `status_usuario: null`
- [ ] Produtos `ativo=false` não aparecem na resposta
