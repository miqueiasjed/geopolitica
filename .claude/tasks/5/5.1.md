# Task 5.1 – Types, hook useFeed e serviço api.ts

## Objetivo
Criar as interfaces TypeScript do feed, o hook `useFeed` com paginação cursor e refetch automático a cada 5 minutos, e garantir que o `api.ts` tem o interceptor Bearer configurado.

## Tipo

- `frontend/src/hooks/useFeed.ts` → criar
- `frontend/src/lib/axios.ts` → verificar/complementar (interceptor Bearer)

- `frontend/src/types/auth.ts` → ver padrão de tipos adotado

## Especificação

### O que fazer:
1. Criar `src/types/feed.ts`:
   ```ts
   export type ImpactLabel = 'CRÍTICO' | 'ALTO' | 'MÉDIO' | 'MONITORAR'
   export interface Event {
     id: number
     titulo: string
     resumo: string
     analise_ia: string | null
     fonte: string
     regiao: string | null
     impact_score: number
     impact_label: ImpactLabel
     categorias: string[]
     publicado_em: string
   }
   export interface FeedFilters {
     categoria?: string
     regiao?: string
     label?: ImpactLabel
   }
   export interface FeedPage {
     data: Event[]
     next_cursor: string | null
     prev_cursor: string | null
   }
   ```
2. Criar `src/hooks/useFeed.ts`:
   - Usar `useInfiniteQuery` do TanStack Query v5
   - `queryKey: ['feed', filters]`
   - `queryFn`: `GET /api/feed?cursor={cursor}&...filtros`
   - `getNextPageParam`: extrair `next_cursor` da resposta
   - `refetchInterval: 5 * 60 * 1000` (5 minutos)
   - Exportar `{ events, fetchNextPage, hasNextPage, isFetchingNextPage, isLoading }`
3. Verificar `src/lib/axios.ts`:
   - Se o interceptor Bearer não foi criado na task 2.5, adicionar agora
   - Interceptor de request: `config.headers.Authorization = Bearer ${token}`

### Regras de negócio aplicáveis:
- `refetchInterval` ativo mesmo quando a janela está em foco
- Filtros mudam o `queryKey` para forçar nova busca quando o filtro muda
- `next_cursor: null` indica que não há mais páginas (desabilitar "carregar mais")

### Critérios de aceitação:
- [ ] `useFeed()` retorna `events` como array flat de todos os eventos carregados
- [ ] `useFeed({ categoria: 'energia' })` busca com filtro na URL
- [ ] `fetchNextPage()` carrega próxima página com o cursor correto
- [ ] Após 5 minutos, `useFeed` faz refetch automático (testar com network tab)
- [ ] `isLoading: true` durante primeira carga
