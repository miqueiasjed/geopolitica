# Task 5.3 – Componentes FilterBar e EventList com scroll infinito

## Objetivo
Criar a `FilterBar` com os 8 filtros de categoria e o `EventList` com scroll infinito via IntersectionObserver que carrega a próxima página do cursor ao atingir o fim da lista.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/components/FilterBar.tsx` → criar
- `frontend/src/components/EventList.tsx` → criar
- `frontend/src/components/LoadingSpinner.tsx` → criar
- `frontend/src/components/EmptyState.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useFeed.ts` → ver interface do hook (task 5.1)
- `frontend/src/components/EventCard.tsx` → ver props do EventCard (task 5.2)
- `frontend/src/types/feed.ts` → ver FeedFilters

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:
1. Criar `FilterBar.tsx`:
   - Props: `filters: FeedFilters`, `onChange: (filters: FeedFilters) => void`
   - Botões: `Todos | Energia | Alimentos | Câmbio | Conflitos | Sanções | Eleições | Commodities`
   - Estilo ativo: `bg-[#C9B882]/10 text-[#C9B882] border border-[#C9B882]/30`
   - Estilo inativo: `text-gray-500 hover:text-gray-300 border border-transparent`
   - Layout horizontal scrollável em mobile
2. Criar `LoadingSpinner.tsx`:
   - Spinner animado simples (border-t dourado)
   - Tamanho: `size?: 'sm' | 'md' | 'lg'`
3. Criar `EmptyState.tsx`:
   - Props: `message?: string`
   - Ícone + texto centralizado, cor `text-gray-600`
4. Criar `EventList.tsx`:
   - Props: nenhuma (consome `useFeed` internamente via props `events`, `hasNextPage`, etc.)
   - Renderizar lista de `EventCard`
   - Ref sentinel no último elemento (div vazia)
   - `IntersectionObserver` no sentinel: quando visível e `hasNextPage`, chamar `fetchNextPage()`
   - Mostrar `LoadingSpinner` durante `isFetchingNextPage`
   - Mostrar `EmptyState` quando `events.length === 0` e não `isLoading`

### Regras de negócio aplicáveis:
- Scroll infinito via IntersectionObserver — não usar botão "carregar mais"
- O sentinel deve ter `threshold: 0.1` para disparar antes de atingir o fundo exato
- `FilterBar` não faz chamada de API — apenas propaga mudanças para o componente pai

### Critérios de aceitação:
- [ ] `FilterBar` renderiza os 8 botões de categoria
- [ ] Clicar em "Energia" atualiza `filters.categoria = 'energia'`
- [ ] Clicar em "Todos" limpa o filtro
- [ ] Ao rolar até o fim da lista, próxima página é carregada automaticamente
- [ ] `LoadingSpinner` aparece durante carregamento de nova página
- [ ] `EmptyState` aparece quando não há eventos

## Teste esperado
