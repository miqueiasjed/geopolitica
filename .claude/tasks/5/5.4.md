# Task 5.4 – Layout DashboardLayout, TopNav e página Feed

## Objetivo
Criar o layout base do dashboard com `DashboardLayout` e `TopNav`, e a página `Feed.tsx` que compõe todos os componentes numa página funcional acessível em `/dashboard/feed`.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/components/DashboardLayout.tsx` → criar
- `frontend/src/components/TopNav.tsx` → criar
- `frontend/src/pages/Feed.tsx` → criar
- `frontend/src/App.tsx` → modificar (adicionar rota /dashboard/feed)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/contexts/AuthContext.tsx` → ver useAuth para logout e user info (task 2.5)

## Especificação

### O que fazer:
1. Criar `TopNav.tsx`:
   - Logo/nome "Geopolítica para Investidores" em dourado `#C9B882`
   - Links de navegação (apenas "Feed" por agora — outros serão adicionados nos planos seguintes)
   - Badge do plano do usuário (ex: `PRO`)
   - Botão de logout
   - Indicador de "última atualização" (timestamp da última busca do feed)
   - Fundo: `bg-[#0a0a0b] border-b border-[#1e1e20]`
2. Criar `DashboardLayout.tsx`:
   - Wrapper com `min-h-screen bg-[#0a0a0b]`
   - `TopNav` fixo no topo
   - `<main>` com padding abaixo do nav
   - Exportar via `<Outlet />` ou `children`
3. Criar `Feed.tsx`:
   - Estado local: `filters: FeedFilters`
   - Passar `filters` ao `useFeed` e ao `FilterBar`
   - Layout: `FilterBar` no topo + `EventList` abaixo
   - Indicador de próxima atualização (countdown 5 min)
4. Atualizar `App.tsx`:
   ```tsx
   <Route element={<RotaProtegida />}>
     <Route element={<DashboardLayout />}>
       <Route path="/dashboard/feed" element={<Feed />} />
     </Route>
   </Route>
   ```
   - Redirecionar `/` para `/dashboard/feed`

### Regras de negócio aplicáveis:
- Fundo `#0a0a0b` em toda a aplicação — nunca branco ou cinza claro
- O countdown de próxima atualização deve ser puramente visual (baseado em `dataUpdatedAt` do React Query)
- TopNav deve exibir o plano do usuário para orientar o que ele pode ver

### Critérios de aceitação:
- [ ] Acessar `/dashboard/feed` autenticado renderiza a página corretamente
- [ ] `FilterBar` muda o filtro e `EventList` recarrega com novos dados
- [ ] Logout no `TopNav` redireciona para `/login`
- [ ] Layout responsivo: TopNav não quebra em mobile
- [ ] Redirecionar `/` → `/dashboard/feed`

## Teste esperado
