# Task 5.5 – Indicador de atualização e polimento visual do dashboard

## Objetivo
Adicionar o indicador de "Atualizando..." com animação quando o feed está sendo refetchado, polir o visual geral do dashboard (espaçamentos, tipografia, responsive) e garantir acessibilidade básica (prefers-reduced-motion).

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/components/RefetchIndicator.tsx` → criar
- `frontend/src/components/EventCard.tsx` → modificar (polimento)
- `frontend/src/pages/Feed.tsx` → modificar (integrar RefetchIndicator)
- `frontend/src/index.css` → modificar (prefers-reduced-motion)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useFeed.ts` → ver `isFetching` e `dataUpdatedAt` (task 5.1)
- `frontend/src/components/EventCard.tsx` → ver estado atual (task 5.2)
- `frontend/src/pages/Feed.tsx` → ver estado atual (task 5.4)

## Skills necessárias
- frontend-design-system
- animacoes-motion

## Especificação

### O que fazer:
1. Criar `RefetchIndicator.tsx`:
   - Props: `isFetching: boolean`, `dataUpdatedAt: number`
   - Quando `isFetching`: mostrar pulsing dot dourado + texto "Atualizando..."
   - Quando idle: mostrar "Atualizado há X min" (tempo relativo a `dataUpdatedAt`)
   - Animação: `framer-motion` `AnimatePresence` para entrada/saída suave
   - Posição: canto superior direito do feed, abaixo do TopNav
2. Polir `EventCard.tsx`:
   - Verificar espaçamentos em mobile (padding menor em `sm:`)
   - Truncar título em 2 linhas (`line-clamp-2`)
   - Truncar análise IA em 3 linhas (`line-clamp-3`)
   - Adicionar `title` attr no truncado para acessibilidade
3. Atualizar `Feed.tsx`:
   - Integrar `RefetchIndicator` passando `isFetching` e `dataUpdatedAt` do `useFeed`
4. Atualizar `index.css`:
   ```css
   @media (prefers-reduced-motion: reduce) {
     *, *::before, *::after {
       animation-duration: 0.01ms !important;
       transition-duration: 0.01ms !important;
     }
   }
   ```

### Regras de negócio aplicáveis:
- `prefers-reduced-motion` é obrigatório — respeitar preferência de acessibilidade do usuário
- O indicador de atualização deve ser sutil — não distrai do conteúdo principal
- Line-clamp mantém o layout consistente independente do tamanho do texto

### Critérios de aceitação:
- [ ] `RefetchIndicator` mostra pulsing dot durante refetch automático dos 5 min
- [ ] "Atualizado há X min" atualiza corretamente após cada refetch
- [ ] Títulos longos são truncados em 2 linhas sem quebrar o layout
- [ ] Análise IA truncada em 3 linhas
- [ ] Com `prefers-reduced-motion`, animações do EventCard são desabilitadas
- [ ] Dashboard visual aprovado em mobile (375px) e desktop (1280px)

## Teste esperado
- Verificação manual: aguardar ou forçar refetch e observar o indicador animar
- Testar com DevTools em modo `prefers-reduced-motion: reduce`
