# Task 1.2 – Configurar Redis, filas e arquivos de configuração customizados

## Objetivo
Configurar Redis como driver de cache, sessões e filas no Laravel, e criar os arquivos `config/claude.php` e `config/feed.php` com as configurações específicas do projeto.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `config/cache.php` → modificar (driver para redis)
- `config/session.php` → modificar (driver para redis)
- `config/queue.php` → modificar (driver para redis)
- `config/claude.php` → criar
- `config/feed.php` → criar

## Contexto necessário (arquivos para ler antes)
- `config/cache.php` → entender estrutura padrão do Laravel
- `config/queue.php` → entender configuração de connections

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Atualizar `.env` (já criado na 1.1):
   - `CACHE_DRIVER=redis`
   - `SESSION_DRIVER=redis`
   - `QUEUE_CONNECTION=redis`
2. Verificar que `config/cache.php`, `config/session.php` e `config/queue.php` apontam para Redis via env vars
3. Criar `config/claude.php`:
```php
return [
    'api_key' => env('CLAUDE_API_KEY'),
    'model'   => env('CLAUDE_MODEL', 'claude-sonnet-4-5'),
    'max_tokens' => env('CLAUDE_MAX_TOKENS', 1024),
];
```
4. Criar `config/feed.php`:
```php
return [
    'sources' => [
        // Lista de URLs RSS a serem configuradas
    ],
    'refresh_interval_minutes' => env('FEED_REFRESH_INTERVAL', 30),
    'max_items_per_source'     => env('FEED_MAX_ITEMS', 50),
];
```
5. Verificar que `php artisan queue:work --once` funciona sem erros de conexão Redis

### Regras de negócio aplicáveis:
- Redis deve ser o único driver de cache/sessão/fila — não usar file driver em nenhuma das três
- As variáveis de configuração do Claude e do Feed devem ter valores padrão sensatos

### Critérios de aceitação:
- [ ] `php artisan config:show cache` mostra `driver: redis`
- [ ] `php artisan config:show session` mostra `driver: redis`
- [ ] `php artisan config:show queue` mostra `default: redis`
- [ ] `config/claude.php` existe e lê `CLAUDE_API_KEY` do `.env`
- [ ] `config/feed.php` existe com estrutura de sources e intervalo
- [ ] `php artisan tinker` → `Cache::put('test', 1)` e `Cache::get('test')` retornam sem erro

## Teste esperado
- Comando: `php artisan config:show cache`
- Verificação manual: `php artisan tinker` → `Cache::put('test', 1, 10); Cache::get('test');` → retorna `1`

baixa
~40 linhas de configuração | ~5 arquivos
