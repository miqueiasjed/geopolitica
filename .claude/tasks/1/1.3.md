# Task 1.3 – Configurar CORS, Sanctum e executar migration base

## Objetivo
Configurar CORS no Laravel para aceitar requisições do frontend Vite (localhost:5173), configurar Sanctum para SPA e executar a migration base de `users` + tabelas do Sanctum e Spatie Permission.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `config/cors.php` → modificar
- `config/sanctum.php` → modificar
- `bootstrap/app.php` → modificar (registrar middleware Sanctum)
- `database/migrations/` → verificar migrations existentes e executar

## Contexto necessário (arquivos para ler antes)
- `config/cors.php` → entender a estrutura e ajustar `allowed_origins`
- `config/sanctum.php` → entender `stateful_domains`

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Configurar `config/cors.php`:
   - `allowed_origins` → `['http://localhost:5173', env('FRONTEND_URL', 'http://localhost:5173')]`
   - `allowed_methods` → `['*']`
   - `allowed_headers` → `['*']`
   - `supports_credentials` → `true`
2. Configurar `config/sanctum.php`:
   - `stateful` → incluir `localhost:5173` e o env `SANCTUM_STATEFUL_DOMAINS`
3. Registrar middleware do Sanctum em `bootstrap/app.php` (ou `app/Http/Kernel.php` se Laravel < 11 style):
   - Adicionar `\Laravel\Sanctum\Http\Middleware\EnsureFrontendRequestsAreStateful::class` no grupo `api`
4. Executar migrations:
   - `php artisan migrate` (cria `users`, `password_reset_tokens`, `sessions`, tabelas Sanctum e Spatie)
5. Verificar que o banco de dados tem todas as tabelas esperadas

### Regras de negócio aplicáveis:
- O token Sanctum é armazenado em `localStorage` no SPA (não usa cookies) — confirmar que a config não força cookie-only
- CORS deve ser restritivo por padrão e expandir apenas via variável de ambiente

### Critérios de aceitação:
- [ ] `php artisan migrate` roda sem erros
- [ ] Tabelas `personal_access_tokens`, `roles`, `permissions`, `model_has_roles` existem no banco
- [ ] `config/cors.php` tem `localhost:5173` em `allowed_origins`
- [ ] Uma requisição OPTIONS para `/api/auth/login` retorna headers CORS corretos
- [ ] `php artisan route:list` mostra as rotas do Sanctum

## Teste esperado
- Comando: `php artisan migrate:status`
- Verificação manual: acessar `http://localhost:8000/api/sanctum/csrf-cookie` e verificar resposta 204
