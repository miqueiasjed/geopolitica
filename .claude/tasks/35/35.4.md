# Task 35.4 – Toasts Mobile: posição e tamanho adequados em 375px

## Objetivo
Garantir que os toasts/notificações do sistema não bloqueiem botões de ação em mobile, usando posição `bottom-center` e largura `calc(100vw-2rem)` em telas pequenas.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- Arquivo de configuração do Sonner (ou Radix Toast) → modificar (tipicamente `App.tsx` ou `main.tsx` ou `ToasterProvider.tsx`)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/App.tsx` → verificar onde o `<Toaster>` é configurado (Sonner ou Radix Toast)
- `frontend/src/main.tsx` → se não estiver no App.tsx
- Buscar: `grep -r "Toaster\|toast" frontend/src/ --include="*.tsx" -l | head -5` para localizar o provider

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**Se usar Sonner (`sonner`):**
```tsx
<Toaster
  position="bottom-right"
  toastOptions={{
    className: "w-[calc(100vw-2rem)] sm:w-auto",
  }}
  // Em mobile, Sonner aceita position="bottom-center"
  // ou usar CSS para sobrescrever em mobile
/>
```
Alternativa com CSS global:
```css
@media (max-width: 640px) {
  [data-sonner-toaster] {
    left: 1rem !important;
    right: 1rem !important;
    bottom: 1rem !important;
    width: calc(100vw - 2rem) !important;
  }
}
```

**Se usar Radix UI Toast:**
- Verificar o `<Toast.Viewport>` — adicionar `className="fixed bottom-4 left-4 right-4 sm:left-auto sm:right-4 sm:w-96"`.

**Verificação adicional:**
- Confirmar que os toasts não ficam posicionados sobre a navegação mobile (header h-14 do AdminLayout ou TopNav).
- Se houver botões de ação fixos no bottom (ex: "Salvar" sticky), garantir que os toasts ficam **acima** deles com z-index maior, ou que a posição muda para `top`.

### Regras de negócio aplicáveis:
- Toasts de sucesso/erro são críticos para feedback do usuário — não podem ser bloqueados por outros elementos.
- Em mobile, `bottom-right` padrão fica fora da área de toque visível — `bottom-center` com largura full é mais acessível.

### Critérios de aceitação:
- [ ] Toasts centralizados horizontalmente em 375px (não cortados à direita).
- [ ] Largura do toast: `calc(100vw-2rem)` ou equivalente em mobile.
- [ ] Toasts não bloqueiam botões de ação fixos ou input de mensagem (ChatPage).
- [ ] Em sm+: posição original (bottom-right) preservada.

## Teste esperado
- Em 375px: disparar um toast de sucesso e um de erro — ambos aparecem centralizados, legíveis e não cobrem conteúdo interativo.
