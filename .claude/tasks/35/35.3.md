# Task 35.3 – Audit de Dialog.Content (Radix UI): max-w correto em mobile

## Objetivo
Auditar todos os `<Dialog.Content>` do projeto para garantir largura máxima adequada em mobile (`max-w-[calc(100vw-2rem)]`), scroll interno quando conteúdo é maior que a tela, e overlay fechando ao clicar fora em touch.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- Arquivos com `<Dialog.Content>` do Radix UI encontrados durante a auditoria → modificar (tipicamente 3-8 arquivos)

## Contexto necessário (arquivos para ler antes)
- Buscar em `frontend/src/` por: `Dialog.Content`, `DialogContent`, `<Dialog` para listar todos os usos
- Ver se há um componente `<Modal>` ou `<Dialog>` wrapper customizado no projeto

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. **Busca inicial:** `grep -r "Dialog.Content\|DialogContent" frontend/src/ --include="*.tsx" -l` para listar todos os arquivos.
2. **Para cada Dialog.Content encontrado**, verificar as classes de largura:
   - Se não tiver `max-w` mobile: adicionar `max-w-[calc(100vw-2rem)] sm:max-w-[500px]` (ou manter o `max-w-sm` existente se adequado).
   - Padrão ideal: `w-full max-w-[calc(100vw-2rem)] sm:max-w-lg`.
3. **Scroll interno:** verificar se o conteúdo longo tem `overflow-y-auto max-h-[80vh]`.
4. **Formulários em modais:** campos com `w-full` dentro do Dialog.
5. Se houver um componente wrapper `<Modal>` ou `<Dialog>` customizado: corrigir no wrapper e todos os usos herdam automaticamente.
6. **Overlay:** o Radix UI fecha ao clicar fora nativamente — verificar se não há `e.preventDefault()` no overlay que bloqueie o comportamento em touch.

### Regras de negócio aplicáveis:
- Modais de confirmação (deletar, confirmar ação): devem caber em 375px sem scroll.
- Modais de formulário (editar, criar): precisam de scroll interno.
- Não alterar o comportamento de fechamento ao clicar fora — é padrão Radix.

### Critérios de aceitação:
- [ ] Todos os Dialog.Content com `max-w` que respeita 375px (não excede `100vw`).
- [ ] Modais de formulário longo com scroll interno (`overflow-y-auto max-h-[80vh]`).
- [ ] Campos de formulário dentro de modais com `w-full`.
- [ ] Clicar no overlay fecha o modal em touch (comportamento Radix padrão preservado).

## Teste esperado
- Em 375px: abrir 3-4 modais diferentes do projeto — nenhum deve ultrapassar a largura da tela.

média
~40 linhas de código | ~3-8 arquivos (depende da quantidade encontrada)
