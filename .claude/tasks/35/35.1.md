# Task 35.1 – AlertaPanel Mobile: w-full, header fixo e scroll interno

## Objetivo
Garantir que `AlertaPanel` ocupe `w-full` em mobile (< sm), com header sticky e corpo com scroll interno, preservando o layout desktop de painel lateral fixo.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/components/alertas/AlertaPanel.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/components/alertas/AlertaPanel.tsx` → ver classes de largura (`sm:w-80 md:w-96`), posicionamento (`fixed`), estrutura interna

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. Verificar as classes de largura: provavelmente `w-80 md:w-96` ou similar. Mudar para `w-full sm:w-80 md:w-96`.
2. Se o painel for `fixed` à direita: em mobile deve usar `fixed inset-x-0 bottom-0` ou `fixed inset-0` (full-screen) — depende do design atual.
   - Preferência: `fixed inset-0 sm:inset-auto sm:right-0 sm:top-0 sm:h-full sm:w-80` (full-screen em mobile, lateral em desktop).
3. Header do painel: adicionar `sticky top-0 bg-background z-10` para ficar fixo durante o scroll interno.
4. Corpo do painel: adicionar `overflow-y-auto` para scroll interno.
5. Botão de fechar (X): deve estar no header, acessível sem scroll em mobile (44x44px min).
6. Opcional: suporte a swipe para fechar em mobile (usando evento `onTouchStart/onTouchEnd` com threshold de 100px — apenas se já houver padrão de swipe no projeto).

### Regras de negócio aplicáveis:
- AlertaPanel contém alertas de mercado — o conteúdo não pode ser truncado.
- O botão de fechar deve ser sempre visível (header sticky).
- Em desktop (sm+): comportamento de painel lateral preservado.

### Critérios de aceitação:
- [ ] AlertaPanel com `w-full` em 375px, sem overflow horizontal.
- [ ] Header do painel sticky com botão fechar sempre visível.
- [ ] Corpo com scroll interno quando conteúdo excede a altura da tela.
- [ ] Em sm+: painel lateral preservado.

## Teste esperado
- Em 375px: abrir AlertaPanel → ocupa tela toda → scroll dentro do painel → fechar via botão X visível no header.
