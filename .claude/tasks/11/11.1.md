# Task 11.1 – Migrations e Models: PerfilPais e PaisUsuario

## Objetivo
Criar as duas migrations e os dois Models que sustentam o módulo de Perfil de País — `perfis_paises` (dados estáticos + conteúdo gerado pela IA) e `paises_usuarios` (pivot user-país).

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_perfis_paises_table.php` → criar
- `database/migrations/xxxx_create_paises_usuarios_table.php` → criar
- `app/Models/PerfilPais.php` → criar
- `app/Models/PaisUsuario.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/User.php` → ver relacionamentos e campos existentes
- `database/migrations/` → verificar convenções de nomenclatura do projeto

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Migration `perfis_paises`:**
```
codigo_pais (char 2, unique) — ex: "BR", "US"
nome_pt (string)
bandeira_emoji (string, nullable)
regiao_geopolitica (string) — ex: "América do Sul"
contexto_geopolitico (text, nullable) — gerado pela IA
analise_lideranca (text, nullable) — gerado pela IA
indicadores_relevantes (json, default []) — array de strings ex: ["PIB", "Dívida/PIB"]
termos_busca (json, default []) — array de strings para busca de eventos
gerado_em (timestamp, nullable)
timestamps
```

**Migration `paises_usuarios`:**
```
user_id (foreignId → users, cascadeOnDelete)
codigo_pais (char 2)
adicionado_em (timestamp, default now)
UNIQUE KEY (user_id, codigo_pais)
```
Sem `timestamps` padrão (usar `adicionado_em`).

**Model `PerfilPais`:**
- `$primaryKey = 'codigo_pais'`; `$keyType = 'string'`; `$incrementing = false`
- `$fillable`: todos os campos
- Cast `indicadores_relevantes` e `termos_busca` → `array`
- Cast `gerado_em` → `datetime`
- Relacionamento `usuarios()`: hasMany via `PaisUsuario`
- Scope `scopePorRegiao($query, $regiao)`

**Model `PaisUsuario`:**
- `$fillable`: `user_id`, `codigo_pais`, `adicionado_em`
- `$timestamps = false`
- Cast `adicionado_em` → `datetime`
- Relacionamento `perfil()`: belongsTo `PerfilPais` via `codigo_pais`
- Relacionamento `usuario()`: belongsTo `User`

### Regras de negócio aplicáveis:
- `codigo_pais` segue ISO 3166-1 alpha-2 (2 letras maiúsculas)
- A tabela `paises_usuarios` não deve ter colunas `created_at`/`updated_at` padrão

### Critérios de aceitação:
- [ ] `php artisan migrate` executa sem erros
- [ ] `PerfilPais::create([...])` salva corretamente com casts de array
- [ ] `PaisUsuario` respeita UNIQUE KEY (erro ao inserir duplicata)
- [ ] Relacionamentos `perfilPais->usuarios()` e `paisUsuario->perfil()` funcionam

## Teste esperado

- Verificação manual: `php artisan tinker` — instanciar `PerfilPais` e `PaisUsuario`
