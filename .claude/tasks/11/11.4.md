## Especificação

### O que fazer:

**`LimitePaisesAtingidoException`:**
```php
class LimitePaisesAtingidoException extends \RuntimeException
{
    public function __construct(int $limite, string $plano)
    {
        parent::__construct("Limite de {$limite} países atingido no plano {$plano}. Faça upgrade para adicionar mais.");
    }
}
```
Registrar em `app/Exceptions/Handler.php` para retornar HTTP 422 com `message`.

**`PaisController`:**

`GET /api/paises` — lista todos os países:
- Parâmetro opcional `?q=` para busca por `nome_pt` ILIKE
- Cache Redis 30 min: chave `paises_lista_{md5(q)}`
- Retornar: `codigo_pais`, `nome_pt`, `bandeira_emoji`, `regiao_geopolitica`, `gerado_em`
- Ordenar por `nome_pt`

`GET /api/paises/{codigo}` — perfil completo:
- Cache Redis 30 min: chave `perfil_pais_{codigo}`
- Se `gerado_em` for null, despachar `GerarPerfilPaisJob` (gerar assíncrono)
- Retornar todos os campos do `PerfilPais`

`GET /api/paises/{codigo}/eventos` — eventos recentes:
- Chamar `EventosPaisService::buscarEventosRecentes($pais)`
- Sem cache (dado em tempo real)
- Retornar array de eventos (campos: `id`, `titulo`, `descricao`, `nivel_tensao`, `created_at`)

**`AdicionarPaisRequest`:**
```php
'codigo_pais' => 'required|string|size:2|exists:perfis_paises,codigo_pais'
```

**`PaisUsuarioController`:**

Limites por plano:
```php
private const LIMITES = ['essencial' => 3, 'pro' => 10];
// reservado/admin → ilimitado (sem verificação)
```

`GET /api/meus-paises`:
- Retornar países seguidos pelo usuário autenticado com perfil
- `PaisUsuario::where('user_id', auth()->id())->with('perfil')->get()`

`POST /api/meus-paises`:
- Validar com `AdicionarPaisRequest`
- Verificar limite: contar `paisesUsuario` do user, comparar com `LIMITES[$plano]`
- Se `role` for `reservado` ou `admin` → sem limite
- Se atingiu → lançar `LimitePaisesAtingidoException`
- Criar `PaisUsuario`
- Retornar 201 com o registro criado + perfil

`DELETE /api/meus-paises/{codigo}`:
- Verificar se o país pertence ao usuário autenticado (403 se não)
- Deletar e retornar 204

**Rotas em `routes/api.php`:**
```php
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/paises', [PaisController::class, 'index']);
    Route::get('/paises/{codigo}', [PaisController::class, 'show']);
    Route::get('/paises/{codigo}/eventos', [PaisController::class, 'eventos']);

    Route::get('/meus-paises', [PaisUsuarioController::class, 'index']);
    Route::post('/meus-paises', [PaisUsuarioController::class, 'store']);
    Route::delete('/meus-paises/{codigo}', [PaisUsuarioController::class, 'destroy']);
});
```

### Regras de negócio aplicáveis:
- Limite: `essencial` → 3 países, `pro` → 10 países, `reservado`/`admin` → ilimitado
- Cache Redis 30 min para perfil individual e lista
- Cache invalidado na regeneração do perfil (feito no Service)
- `DELETE` deve verificar ownership (não permitir deletar país de outro user)

### Critérios de aceitação:
- [ ] `GET /api/paises` retorna lista com suporte a `?q=`
- [ ] `GET /api/paises/{codigo}` retorna perfil completo (ou despacha Job se sem conteúdo)
- [ ] `GET /api/paises/{codigo}/eventos` retorna eventos recentes
- [ ] `POST /api/meus-paises` com usuário `essencial` falha com 422 ao tentar adicionar o 4º país
- [ ] `DELETE /api/meus-paises/{codigo}` retorna 403 para países de outro usuário
- [ ] Cache Redis funcionando (segunda requisição mais rápida)
