# Task 33.3 – Tabelas Mobile: CrisesPage, BibliotecaPage e FontesPage (admin)

## Objetivo
Aplicar overflow-x-auto nas tabelas de CrisesPage (admin), BibliotecaPage (admin) e FontesPage (admin), com filtros responsivos e botões abreviados em mobile.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/CrisesPage.tsx` → modificar
- `frontend/src/pages/admin/BibliotecaPage.tsx` → modificar (versão admin)
- `frontend/src/pages/admin/FontesPage.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/CrisesPage.tsx` → ver estrutura de colunas e filtros
- `frontend/src/pages/admin/BibliotecaPage.tsx` → ver se é lista/tabela ou grid de cards
- `frontend/src/pages/admin/FontesPage.tsx` → ver estrutura (feeds RSS)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**Para cada uma das 3 páginas:**
1. Envolver `<table>` em `<div className="overflow-x-auto">` + `min-w-[560px]` na tabela.
2. BibliotecaPage (admin): se usar grid de cards de conteúdo, trocar para `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`.
3. Filtros: `flex flex-wrap gap-2` ou `flex flex-col sm:flex-row gap-2`.
4. Inputs de busca: `w-full sm:w-64` para não transbordar.
5. Botões com texto longo: `<span className="hidden sm:inline">` para rótulos.

### Regras de negócio aplicáveis:
- FontesPage (feeds RSS) pode ter URLs longas — adicionar `max-w-[200px] truncate` nas células de URL.
- BibliotecaPage admin pode ter filtros de categoria/tipo — todos devem estar em `flex-wrap`.

### Critérios de aceitação:
- [ ] CrisesPage (admin): tabela scrollável em 375px.
- [ ] BibliotecaPage (admin): lista/grid usável em 375px (scroll ou grid 1-col).
- [ ] FontesPage (admin): tabela scrollável em 375px, URLs truncadas na célula.
- [ ] Filtros das 3 páginas não transbordam em mobile.

## Teste esperado
- Verificação visual em 375px: sem overflow horizontal em nenhuma das 3 páginas.
