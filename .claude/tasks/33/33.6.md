# Task 33.6 – Formulários Mobile: ImportarAddonsPage + Botões Abreviados

## Objetivo
Tornar ImportarAddonsPage responsiva em mobile (área de upload + preview) e garantir que botões de ação com rótulos longos usem `hidden sm:inline` em todas as páginas admin já corrigidas neste plano.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/ImportarAddonsPage.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/ImportarAddonsPage.tsx` → ver layout atual (área de upload, preview de CSV/tabela, botão importar)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
**ImportarAddonsPage:**
1. Área de upload (dropzone ou input file): garantir `w-full` e altura mínima de 120px em mobile.
2. Se houver preview de dados (tabela de preview de CSV): envolver em `overflow-x-auto` + `min-w-[400px]`.
3. Botão de importação: `w-full sm:w-auto`, posicionado após o preview.
4. Steps/instruções de importação: empilhar verticalmente se houver layout side-by-side.
5. Mensagens de erro/validação: `w-full`, quebra de linha adequada.

**Revisão de botões abreviados (auditoria transversal):**
- Em **todas** as páginas modificadas nas Tasks 33.1–33.5, verificar se botões primários com rótulos longos (ex: "Remover Assinante", "Exportar CSV", "Adicionar Produto") usam `hidden sm:inline` no texto.
- Padrão: `<Button><PlusIcon /><span className="hidden sm:inline ml-1">Adicionar Addon</span></Button>`.
- Esta auditoria não requer abrir todos os arquivos — apenas registrar o padrão para o agente que executar as tasks anteriores verificar se aplicaram.

### Regras de negócio aplicáveis:
- Importação de addons processa CSV — a preview pode ter muitas colunas, então `overflow-x-auto` é obrigatório.
- O botão de confirmação de importação deve ter estado de loading visível em mobile.

### Critérios de aceitação:
- [ ] ImportarAddonsPage: dropzone com `w-full` em 375px, sem overflow.
- [ ] Tabela de preview de CSV com scroll horizontal em mobile.
- [ ] Botão "Importar" acessível abaixo da preview em mobile.
- [ ] Padrão de `hidden sm:inline` documentado para aplicação nas tasks anteriores.

## Teste esperado
- Em 375px: fazer upload de um CSV de teste, ver preview com scroll lateral, botão importar visível.
