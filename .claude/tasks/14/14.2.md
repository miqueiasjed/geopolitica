# Task 14.2 – ChatRecuperacaoService: busca FULLTEXT nas 4 fontes

## Objetivo
Implementar o serviço que recebe a pergunta do usuário e recupera contexto relevante das 4 fontes (conteudos, events, perfis_paises, crises_historicas) via FULLTEXT MySQL para compor o prompt da Claude API.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/ChatRecuperacaoService.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/Conteudo.php` → campos disponíveis (do Plano 7: `titulo`, `corpo`, `tags`)
- `app/Models/Event.php` → campos disponíveis (do Plano 4: `titulo`, `descricao`, `nivel_tensao`)
- `app/Models/PerfilPais.php` → campos disponíveis (11.1: `nome_pt`, `contexto_geopolitico`, `analise_lideranca`)
- `app/Models/CriseHistorica.php` → campos disponíveis (12.1: `titulo`, `contexto_geopolitico`, `impacto_global`)

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**`ChatRecuperacaoService`:**

```php
public function recuperarContexto(string $pergunta): string
```

Executar 4 queries FULLTEXT em paralelo (usando `DB::select` ou FULLTEXT index nativo do MySQL):

**Fonte 1 — Conteúdos (M03):**
```sql
SELECT titulo, corpo, tags FROM conteudos
WHERE MATCH(titulo, corpo) AGAINST(? IN BOOLEAN MODE)
ORDER BY publicado_em DESC LIMIT 3
```
Formatar cada resultado como: `[Biblioteca] {titulo}: {primeiros 300 chars do corpo}`

**Fonte 2 — Eventos (M01):**
```sql
SELECT titulo, descricao, nivel_tensao, created_at FROM events
WHERE MATCH(titulo, descricao) AGAINST(? IN BOOLEAN MODE)
ORDER BY created_at DESC LIMIT 3
```
Formatar: `[Evento - {nivel_tensao}] {titulo}: {descricao truncada em 200 chars}`

**Fonte 3 — Perfis de Países (M06):**
```sql
SELECT nome_pt, contexto_geopolitico, analise_lideranca FROM perfis_paises
WHERE MATCH(contexto_geopolitico, analise_lideranca) AGAINST(? IN BOOLEAN MODE)
AND contexto_geopolitico IS NOT NULL LIMIT 2
```
Formatar: `[País: {nome_pt}] {primeiros 200 chars do contexto}`

**Fonte 4 — Crises Históricas (M07):**
```sql
SELECT titulo, contexto_geopolitico, impacto_brasil FROM crises_historicas
WHERE MATCH(titulo, contexto_geopolitico) AGAINST(? IN BOOLEAN MODE)
LIMIT 2
```
Formatar: `[Crise Histórica] {titulo}: {primeiros 200 chars do contexto}`

**Fallback se FULLTEXT falhar ou retornar vazio:**
- Se total de resultados < 3: adicionar busca LIKE simples nas palavras-chave extraídas da pergunta
- Extrair palavras-chave: `preg_split` por espaço, filtrar stopwords PT (`de`, `da`, `do`, `que`, `em`, etc.), pegar as 3 primeiras palavras com 4+ chars

**Montagem do contexto:**
```php
$contextoFormatado = implode("\n\n", $todosResultados);
return "CONTEXTO DO CANAL:\n{$contextoFormatado}\n\nTOTAL: " . count($todosResultados) . " fontes encontradas.";
```
- Truncar contexto total em 4000 chars se necessário

### Regras de negócio aplicáveis:
- Máximo de 10 resultados totais (3+3+2+2)
- Truncar textos longos para não exceder limite de tokens do prompt
- Se nenhuma fonte retornar resultados: retornar string vazia (chat funciona sem contexto)

### Critérios de aceitação:
- [ ] Busca FULLTEXT nas 4 tabelas sem erros
- [ ] Resultados formatados com prefixo de fonte `[Biblioteca]`, `[Evento]`, etc.
- [ ] Contexto truncado em 4000 chars
- [ ] Fallback LIKE funciona quando FULLTEXT retorna vazio
- [ ] Retorna string vazia (não lança exceção) se nenhuma fonte tem resultado
