## Especificação

### O que fazer:

**`ChatService`:**

```php
public function verificarLimite(User $user): void
```
- Limites: `essencial` → 5/dia, `pro` → 20/dia, `reservado`/`admin` → sem limite
- Chave Redis: `chat_limite_{userId}_{date}` onde `date` = `now()->timezone('America/Sao_Paulo')->format('Y-m-d')`
- TTL calculado: segundos até meia-noite de Brasília (`Carbon::tomorrow()->timezone('America/Sao_Paulo')->startOfDay()->diffInSeconds(now())`)
- Se limite atingido: lançar `\Symfony\Component\HttpKernel\Exception\TooManyRequestsHttpException` com mensagem de upgrade

```php
public function perguntar(User $user, string $pergunta, callable $onToken): void
```
- Chamar `verificarLimite($user)`
- Obter sessão: `ChatSessao::obterOuCriarHoje($user->id)`
- Buscar histórico da sessão (últimas 10 mensagens para contexto)
- Chamar `ChatRecuperacaoService::recuperarContexto($pergunta)`
- Salvar mensagem `role=user` na sessão
- Montar mensagens para Claude API:
  ```
  System: "Você é um analista geopolítico especializado. Responda em português brasileiro de forma objetiva e fundamentada. Use apenas o contexto fornecido quando relevante.\n\n{contexto}"
  + histórico de mensagens anteriores
  + mensagem atual do usuário
  ```
- Chamar Claude API com streaming (modelo `claude-sonnet-4-5`):
  ```php
  $stream = $this->anthropic->messages->stream([
      'model' => 'claude-sonnet-4-5',
      'max_tokens' => 1024,
      'system' => $systemPrompt,
      'messages' => $mensagensFormatadas,
  ]);
  ```
- Para cada token recebido: chamar `$onToken($token)`
- Após stream completo: salvar mensagem `role=assistant` com conteúdo completo
- Incrementar `pergunta_count` na sessão + Redis

**`ChatController`:**

`POST /api/chat/perguntar`:
```php
public function perguntar(Request $request)
{
    $request->validate(['pergunta' => 'required|string|max:500']);
    
    return response()->stream(function () use ($request) {
        $this->chatService->perguntar(
            auth()->user(),
            $request->input('pergunta'),
            function (string $token) {
                echo "data: " . json_encode(['token' => $token]) . "\n\n";
                ob_flush();
                flush();
            }
        );
        echo "data: [DONE]\n\n";
        ob_flush();
        flush();
    }, 200, [
        'Content-Type' => 'text/event-stream',
        'Cache-Control' => 'no-cache',
        'X-Accel-Buffering' => 'no',
    ]);
}
```

Tratar `TooManyRequestsHttpException` no handler para retornar HTTP 429 JSON com:
```json
{"message": "Limite de X perguntas/dia atingido. Faça upgrade para continuar.", "upgrade": true}
```

**`ChatHistoricoController`:**

`GET /api/chat/historico`:
- Buscar sessão do dia do usuário autenticado
- Retornar mensagens ordenadas por `created_at ASC`
- Se não existir sessão: retornar `{ "mensagens": [], "pergunta_count": 0, "limite": N }`
- Incluir `limite` do plano e `pergunta_count` na resposta

**Rotas:**
```php
Route::middleware('auth:sanctum')->group(function () {
    Route::post('/chat/perguntar', [ChatController::class, 'perguntar']);
    Route::get('/chat/historico', [ChatHistoricoController::class, 'index']);
});
```

### Regras de negócio aplicáveis:
- Limite reinicia à meia-noite de Brasília (`America/Sao_Paulo`)
- `reservado` e `admin` sem limite de perguntas
- Mensagem de erro 429 deve incluir `"upgrade": true` para o frontend exibir CTA
- Usar `claude-sonnet-4-5` (não opus — controle de custo)
- Histórico: apenas as últimas 10 mensagens para manter contexto dentro do limite de tokens

### Critérios de aceitação:
- [ ] Streaming SSE funciona (tokens chegam progressivamente no browser)
- [ ] Usuário `essencial` recebe 429 na 6ª pergunta do dia
- [ ] Limite reinicia à meia-noite de Brasília
- [ ] Histórico retorna mensagens da sessão do dia
- [ ] Mensagens persistidas no banco após stream
- [ ] `pergunta_count` incrementado após cada pergunta
