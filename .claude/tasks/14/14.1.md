# Task 14.1 – Migrations e Models: ChatSessao e ChatMensagem

## Objetivo
Criar as duas migrations e Models do sistema de chat — `chat_sessoes` (sessão diária por usuário) e `chat_mensagens` (histórico de mensagens da sessão).

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_chat_sessoes_table.php` → criar
- `database/migrations/xxxx_create_chat_mensagens_table.php` → criar
- `app/Models/ChatSessao.php` → criar
- `app/Models/ChatMensagem.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/User.php` → ver relacionamentos existentes
- `database/migrations/` → verificar convenções de nomenclatura

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Migration `chat_sessoes`:**
```
id (bigIncrements)
user_id (foreignId → users, cascadeOnDelete)
data_sessao (date) — ex: "2026-04-14"
pergunta_count (unsignedInteger, default 0)
UNIQUE KEY (user_id, data_sessao)
timestamps
```

**Migration `chat_mensagens`:**
```
id (bigIncrements)
sessao_id (foreignId → chat_sessoes, cascadeOnDelete)
role (enum: 'user', 'assistant')
conteudo (text)
created_at (timestamp, useCurrent) — sem updated_at
```
Usar `$table->timestamp('created_at')->useCurrent();` sem `timestamps()`.

**Model `ChatSessao`:**
- `$fillable`: `user_id`, `data_sessao`, `pergunta_count`
- Cast `data_sessao` → `date`
- Relacionamento `mensagens()`: hasMany `ChatMensagem`
- Relacionamento `usuario()`: belongsTo `User`
- Scope `scopeHoje($query)`: `whereDate('data_sessao', today())`
- Método estático `obterOuCriarHoje(int $userId): self` — retorna ou cria sessão do dia com `firstOrCreate(['user_id' => $userId, 'data_sessao' => today()])`

**Model `ChatMensagem`:**
- `$fillable`: `sessao_id`, `role`, `conteudo`
- `$timestamps = false`; adicionar `created_at` manualmente
- Cast `created_at` → `datetime`
- Relacionamento `sessao()`: belongsTo `ChatSessao`

### Regras de negócio aplicáveis:
- UNIQUE KEY em `(user_id, data_sessao)` garante uma sessão por dia por usuário
- `chat_mensagens` não precisa de `updated_at` — mensagens são imutáveis
- Sessão é criada automaticamente na primeira pergunta do dia

### Critérios de aceitação:
- [ ] `php artisan migrate` executa sem erros
- [ ] `ChatSessao::obterOuCriarHoje($userId)` retorna sessão existente ou cria nova
- [ ] Re-chamar `obterOuCriarHoje` no mesmo dia não duplica sessão
- [ ] `ChatMensagem::create([...])` salva com `created_at` automático
