# Task 24.1 – Migrations: mapa_risco_ativos e carteiras

## Objetivo
Criar as duas migrations que sustentam o Risk Score de Portfólio: a tabela de mapeamento de ativos para categorias de risco e a tabela de portfólios dos assinantes.

## Especificação

### O que fazer:

#### `create_mapa_risco_ativos_table`
```php
Schema::create('mapa_risco_ativos', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->string('ticker')->unique();       // ex: "PETR4", "VALE3", "SPY"
    $table->string('name');
    $table->string('asset_type')->nullable(); // stock_br | stock_us | etf | fund | commodity | unknown
    $table->json('risk_weights');
    // ex: {"energy": 0.85, "food": 0.1, "fx": 0.35, "military": 0.2}
    $table->json('regions')->nullable();      // regiões geopolíticas relevantes
    $table->text('notes')->nullable();        // justificativa do mapeamento
    $table->timestamp('updated_at')->useCurrent()->useCurrentOnUpdate();
    // sem created_at — só updated_at para controle de cache
});
```

#### `create_carteiras_table`
```php
Schema::create('carteiras', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->foreignId('user_id')->unique()->constrained()->cascadeOnDelete();
    // unique: um portfólio por usuário
    $table->string('nome')->default('Meu Portfólio');
    $table->json('ativos')->default('[]');
    // ex: [{"ticker":"PETR4","peso":0.3},{"ticker":"VALE3","peso":0.2}]
    $table->json('ultimo_score')->nullable();
    // ex: {"total":72,"energia":85,"alimentos":45,"cambio":68,"militar":60,"calculado_em":"..."}
    $table->timestamps();

    $table->index('user_id');
});
```

### Regras de negócio aplicáveis:
- `carteiras.user_id` tem `unique` — um portfólio por usuário (upsert via `updateOrCreate`).
- `mapa_risco_ativos` usa apenas `updated_at` (sem `created_at`) para rastrear quando o mapeamento foi atualizado pela IA.
- `risk_weights` em `mapa_risco_ativos` usa chaves em inglês (`energy`, `food`, `fx`, `military`) para consistência com o motor de cálculo.

### Critérios de aceitação:
- [ ] Migration `mapa_risco_ativos` cria tabela com `ticker` único e `risk_weights` JSON
- [ ] Migration `carteiras` cria tabela com `user_id` único (um portfólio por user)
- [ ] `php artisan migrate` executa sem erros
- [ ] `php artisan migrate:rollback` reverte ambas as tabelas
