# Task 24.8 – Rota, navegação condicional e AddonGate para Risk Score

## Objetivo
Registrar a rota `/dashboard/risk-score`, adicionar "Risk Score" à navegação lateral visível apenas para planos Pro/Reservado/B2B, e garantir que assinantes Essencial vejam o `AddonGate` ao tentar acessar a página.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/router/index.tsx` → modificar (adicionar rota)
- `frontend/src/components/TopNav.tsx` → modificar (item condicional na navegação)
- `frontend/src/pages/dashboard/RiskScore.tsx` → modificar (guardar com AddonGate no topo)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/router/index.tsx` → onde adicionar a nova rota (após `/dashboard/relatorios`)
- `frontend/src/components/TopNav.tsx` → como o Monitor Eleitoral e Monitor de Guerra condicionam exibição por plano/addon
- `frontend/src/components/AddonGate.tsx` → props necessárias para configurar a tela de bloqueio
- `frontend/src/hooks/useAuth.ts` → como verificar plano do usuário no frontend

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

#### `router/index.tsx`
Importar `RiskScore` e adicionar rota após `/dashboard/relatorios/novo`:
```tsx
import { RiskScore } from '../pages/dashboard/RiskScore'
// ...
<Route path="/dashboard/risk-score" element={<RiskScore />} />
```

#### `TopNav.tsx`
Adicionar item condicional em `navItems` após "Relatórios":
```typescript
// Verificar se o usuário tem plano Pro, Reservado ou é admin
const temAcessoRiskScore = ['pro', 'reservado', 'admin'].includes(user?.assinante?.plano ?? user?.role ?? '')

// Na lista navItems:
...(temAcessoRiskScore ? [
  { to: '/dashboard/risk-score', label: 'Risk Score', isActive: (p: string) => p === '/dashboard/risk-score' }
] : []),
```

#### `RiskScore.tsx`
No início do componente, antes de renderizar o conteúdo da página, verificar plano:
```tsx
const { user } = useAuth()
const plano = user?.assinante?.plano ?? user?.role

if (!['pro', 'reservado', 'admin'].includes(plano ?? '')) {
  return (
    <AddonGate
      addonKey="risk_score"
      titulo="Risk Score de Portfólio"
      descricao="Calcule a exposição geopolítica da sua carteira com base nos eventos ativos. Disponível para assinantes Pro e Reservado."
      preco="Disponível no plano Pro"
      ctaLabel="Fazer upgrade para Pro"
      ctaUrl={import.meta.env.VITE_UPGRADE_PRO_URL || '#'}
      upgradePlano="Pro"
    />
  )
}
```
Adicionar `VITE_UPGRADE_PRO_URL` ao `.env.example` do frontend.

### Regras de negócio aplicáveis:
- Planos com acesso: `pro`, `reservado`, `admin` — plano `essencial` vê `AddonGate`.
- "Risk Score" não aparece na navegação para Essencial (evitar frustração ao ver item bloqueado).
- O `AddonGate` é exibido dentro do layout do dashboard (não redirect) — usuário vê o bloqueio com CTA.
- `VITE_UPGRADE_PRO_URL` deve ser configurado no `.env.example` mas pode ser `#` por padrão.

### Critérios de aceitação:
- [ ] Rota `/dashboard/risk-score` registrada no router
- [ ] "Risk Score" aparece na navegação para Pro, Reservado e Admin
- [ ] "Risk Score" NÃO aparece na navegação para Essencial
- [ ] Acessar `/dashboard/risk-score` como Essencial exibe `AddonGate` com CTA de upgrade
- [ ] `VITE_UPGRADE_PRO_URL` no `.env.example` do frontend

## Teste esperado
- Verificação manual: login com conta Essencial → navegar para `/dashboard/risk-score` → deve ver AddonGate
- Login com conta Pro → navegar para `/dashboard/risk-score` → deve ver a página normalmente
- Verificar que "Risk Score" aparece no menu Pro mas não no menu Essencial
