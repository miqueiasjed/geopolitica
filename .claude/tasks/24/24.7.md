## Especificação

### O que fazer:

#### Estado da página
```typescript
const [ativos, setAtivos] = useState([{ ticker: '', peso: 100 }])
const [score, setScore] = useState<RiskScoreResult | null>(null)
const [calculando, setCalculando] = useState(false)
const [erroPeso, setErroPeso] = useState(false)
```

Tipo `RiskScoreResult`:
```typescript
interface RiskScoreResult {
  total: number
  breakdown: { energia: number; alimentos: number; cambio: number; militar: number }
  alertas: Array<{ title: string; level: string }>
  top_riscos: string[]
  calculado_em: string
}
```

#### Formulário de ativos
- Lista dinâmica de linhas: input de ticker (uppercase, placeholder "PETR4") + input numérico de peso (1-100, em %) + botão "×" para remover (se houver mais de 1 linha)
- Botão "+ Adicionar ativo" abaixo da lista
- Contador de total de pesos: se soma ≠ 100%, exibir aviso em texto vermelho ("Os pesos devem somar 100%. Atualmente: X%")
- Máximo de 20 ativos

#### Botão Calcular
- "Calcular risk score →" — desabilitado se `calculando` ou se soma dos pesos ≠ 100% (±5%)
- Ao clicar: normalizar pesos para decimal (peso/100), fazer `POST /api/carteira`, atualizar `score`

#### Score total (exibido após cálculo)
Caixa central com borda dourada fraca:
- Label "Score geopolítico do portfólio" dourado/60
- Número grande (6xl mono bold) com cor: `< 40` verde, `40-69` amarelo, `>= 70` vermelho
- `/100` em cinza claro
- Texto: "Maior risco nas categorias: {topRiscos.join(' e ')}"

#### Breakdown por categoria
Grid 2×2 com `ScoreGauge` para cada categoria:
- Energia, Alimentos, Câmbio, Geopolítica Militar

#### Alertas relacionados
Se `score.alertas.length > 0`:
- Container com label "Alertas preditivos ativos relacionados"
- Cada alerta: badge com nível em cor + título

#### Exportar PDF
`<ExportPdfButton tipo="risk_score" id="current" label="Exportar relatório PDF" />`

#### Cabeçalho
Ao carregar a página, `GET /api/carteira` via `useQuery` para pré-popular o formulário com ativos salvos (se existirem) e exibir o último score calculado.

### Regras de negócio aplicáveis:
- Tickers convertidos para uppercase ao digitar (sem esperar submit).
- Soma dos pesos exibida em tempo real conforme o usuário edita.
- O formulário pré-populado com dados do `useQuery /api/carteira` na montagem da página.
- Se `ultimo_score` existir na carteira salva, exibir automaticamente o breakdown.

### Critérios de aceitação:
- [ ] Formulário permite adicionar/remover ativos dinamicamente
- [ ] Tickers exibidos em uppercase ao digitar
- [ ] Validação de soma dos pesos com feedback visual em tempo real
- [ ] Score total com cor dinâmica (verde/amarelo/vermelho)
- [ ] 4 `ScoreGauge` com valores corretos do breakdown
- [ ] Alertas preditivos exibidos quando presentes
- [ ] `ExportPdfButton` visível após cálculo
