# Task 24.9 – Template PDF do Risk Score e suporte no PdfTemplateService

## Objetivo
Criar o template Blade `pdf/risk_score.blade.php` para exportação do relatório de portfólio e adicionar o suporte ao tipo `risk_score` no `PdfTemplateService` e no `ExportPdfController`.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/resources/views/pdf/risk_score.blade.php` → criar
- `backend/app/Services/PdfTemplateService.php` → modificar (adicionar tipo risk_score)

## Contexto necessário (arquivos para ler antes)
- `backend/resources/views/pdf/layout.blade.php` → como estender o layout base (Task 22.2)
- `backend/app/Services/PdfTemplateService.php` → onde adicionar o novo tipo (Task 22.4)
- `backend/app/Models/Carteira.php` → campos disponíveis no model (Task 24.2)

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

#### `pdf/risk_score.blade.php`
Recebe variável `$carteira` (array com `ativos`, `ultimo_score`). Exibe:
- Título: "Risk Score do Portfólio"
- Data de cálculo: `$carteira['ultimo_score']['calculado_em']` formatada em pt-BR
- **Score Total:** número grande com indicador de nível (BAIXO/MÉDIO/ALTO) e faixa de cor — exibir como `{$score['total']}/100` com label de nível ao lado
- **Tabela de Composição da Carteira:** colunas Ativo | Peso — lista todos os `$carteira['ativos']` com ticker e peso em %
- **Breakdown por Categoria:** mini-tabela com Categoria | Score | Nível para as 4 categorias (Energia, Alimentos, Câmbio, Geopolítica Militar)
- **Alertas Preditivos Relacionados:** se houver alertas em `$score['alertas']`, listar com badge de nível e título

CSS interno: seguir o mesmo padrão do layout (Georgia, #C9B882, cores de nível: verde/amarelo/vermelho).

#### `PdfTemplateService.php`
No método `buscarDados()`, adicionar o caso `risk_score`:
```php
case 'risk_score':
    $carteira = Carteira::where('user_id', $userId)->firstOrFail();
    // Recalcular o score ao exportar para garantir dados atualizados
    $score = app(RiskScoreService::class)->calcularRiscoPortfolio($carteira->ativos);
    return ['carteira' => array_merge($carteira->toArray(), ['ultimo_score' => $score])];
```

No método `selecionarView()`, adicionar o case: `'risk_score' => 'pdf.risk_score'`.

No `ExportPdfController`, atualizar a validação do campo `tipo` para incluir `risk_score`:
```php
'tipo' => 'required|string|in:briefing,alerta,pais,chat,risk_score',
```

### Regras de negócio aplicáveis:
- Para exportar o risk_score, o usuário precisa ter uma carteira salva (404 se não tiver).
- O score é recalculado no momento da exportação (não usa o `ultimo_score` cached) para garantir atualidade.
- Validação do controller atualizada para aceitar o novo tipo.

### Critérios de aceitação:
- [ ] `pdf/risk_score.blade.php` renderiza score total, composição da carteira e breakdown
- [ ] `POST /api/export-pdf` com `tipo="risk_score"` gera PDF sem erros
- [ ] PDF exportado contém dados reais da carteira do usuário
- [ ] Controller aceita `tipo="risk_score"` na validação
- [ ] 404 retornado quando usuário não tem carteira salva

## Teste esperado
