## Especificação

### O que fazer:

#### `CarteiraRiscoController.php`

**`buscar(Request $request): JsonResponse`**
- Verificar se o plano do assinante permite acesso: `pro`, `reservado`, `admin` — caso contrário, retornar 403
- `$carteira = Carteira::where('user_id', auth()->id())->first()`
- Retornar `['carteira' => $carteira]` (null se não existir)

**`calcular(Request $request): JsonResponse`**
- Verificar plano (mesma lógica) — retornar 403 se Essencial
- Validar request:
  ```php
  $request->validate([
      'ativos'         => 'required|array|min:1|max:20',
      'ativos.*.ticker'=> 'required|string|max:20',
      'ativos.*.peso'  => 'required|numeric|min:0.01|max:1',
  ]);
  ```
- `$ativos = collect($request->input('ativos'))->map(fn($a) => ['ticker' => strtoupper($a['ticker']), 'peso' => (float) $a['peso']])->toArray()`
- `$carteira = $this->riskScoreService->salvarCarteira(auth()->id(), $ativos)`
- Retornar `['carteira' => $carteira, 'score' => $carteira->score]`

Lógica de verificação de plano (método privado):
```php
private function verificarAcessoRiskScore(): void
{
    $plano = auth()->user()->assinante?->plano;
    if (!in_array($plano, ['pro', 'reservado', 'admin'], true)) {
        abort(403, 'Risk Score disponível apenas para planos Pro e Reservado.');
    }
}
```

#### `routes/api.php`
Dentro do grupo `auth:sanctum`:
```php
Route::get('/carteira',  [CarteiraRiscoController::class, 'buscar']);
Route::post('/carteira', [CarteiraRiscoController::class, 'calcular']);
```

### Regras de negócio aplicáveis:
- Plano Essencial recebe 403 — o frontend deve renderizar `AddonGate` com CTA de upgrade.
- Tickers são normalizados para uppercase antes de processar.
- Máximo de 20 ativos por portfólio.
- Pesos em decimal (0.3 = 30%) — validação de soma (0.95–1.05) feita no service.

### Critérios de aceitação:
- [ ] `GET /api/carteira` retorna carteira salva ou `{"carteira": null}` para novo usuário
- [ ] `POST /api/carteira` calcula score e retorna dados completos
- [ ] Plano Essencial recebe 403 em ambos os endpoints
- [ ] Tickers convertidos para uppercase automaticamente
- [ ] Request inválida (ex: ativo sem ticker) retorna 422 com erros de validação
- [ ] Sem autenticação retorna 401

## Teste esperado
- `curl -X GET /api/carteira -H "Authorization: Bearer {token}"` — retorna carteira
- `curl -X POST /api/carteira -d '{"ativos":[{"ticker":"petr4","peso":0.6},{"ticker":"vale3","peso":0.4}]}'` — retorna score
