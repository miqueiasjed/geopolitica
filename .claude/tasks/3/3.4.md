# Task 3.4 – Templates Blade de e-mail (4 templates)

## Objetivo
Criar os 4 templates Blade de e-mail transacionais: boas-vindas, acesso liberado, cancelamento e reembolso — com design responsivo e dados dinâmicos via Mailable.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `resources/views/emails/boas-vindas.blade.php` → criar
- `resources/views/emails/acesso-liberado.blade.php` → criar
- `resources/views/emails/cancelamento.blade.php` → criar
- `resources/views/emails/reembolso.blade.php` → criar
- `app/Mail/AcessoLiberadoMail.php` → criar
- `app/Mail/CancelamentoMail.php` → criar
- `app/Mail/ReembolsoMail.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Mail/BoasVindasMail.php` → ver padrão do Mailable já criado na task 3.2
- `app/Models/Assinante.php` → ver campos `plano` e `expira_em`

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar `AcessoLiberadoMail` (Mailable):
   - Enviado quando `ativo` volta a `true` após reativação
   - Template: confirmação de reativação com link de acesso
2. Criar `CancelamentoMail` (Mailable):
   - Recebe `$assinante` com `expira_em`
   - Template: confirmação de cancelamento, data de acesso final
3. Criar `ReembolsoMail` (Mailable):
   - Template: confirmação de reembolso processado
4. Para cada template Blade (`boas-vindas.blade.php` já tem base na 3.2):
   - HTML responsivo inline (sem CSS externo — compatível com clientes de e-mail)
   - Cores neutras (fundo branco, texto escuro) — e-mail não segue o tema dark do site
   - Variáveis: `$userName`, `$plano`, datas relevantes, CTAs com links
5. Atualizar `HotmartHandlerService` (task 3.2) para disparar os e-mails corretos em cada handler

### Regras de negócio aplicáveis:
- E-mails transacionais devem usar `Resend` como driver (`MAIL_MAILER=resend`)
- Inline CSS obrigatório nos templates Blade — clientes de e-mail não carregam folhas de estilo externas
- Não incluir imagens externas — apenas texto e bordas/cores CSS

### Critérios de aceitação:
- [ ] 4 Mailables criados e funcionais
- [ ] 4 templates Blade com HTML responsivo e inline CSS
- [ ] `CancelamentoMail` exibe corretamente a data `expira_em` formatada
- [ ] `php artisan tinker` → `Mail::to('test@test.com')->send(new BoasVindasMail($user))` sem erros
- [ ] Templates renderizam sem variáveis undefined

## Teste esperado

- Configurar `MAIL_MAILER=log` no `.env` de teste e verificar `storage/logs/laravel.log`
