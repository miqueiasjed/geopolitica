# Task 3.3 – WebhookHotmartController e rota pública

## Objetivo
Implementar o controller que recebe eventos da Hotmart, valida o token de segurança via header, loga o evento e delega ao `HotmartHandlerService`. O endpoint sempre retorna HTTP 200.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/WebhookHotmartController.php` → criar
- `routes/api.php` → modificar (adicionar rota pública do webhook)

## Contexto necessário (arquivos para ler antes)
- `app/Services/HotmartHandlerService.php` → entender a interface do service (task 3.2)
- `app/Models/WebhookEvento.php` → ver campos para log (task 3.1)
- `routes/api.php` → ver estrutura atual de rotas

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar `WebhookHotmartController` com método `receber(Request $request)`:
   - Validar header `x-hotmart-webhook-token` contra `config('services.hotmart.webhook_token')`
   - Se inválido: logar tentativa e retornar **HTTP 200** (não 401 — evitar reenvios)
   - Se válido:
     1. Criar `WebhookEvento` com `event_type`, `payload` (json completo), `email`, `hotmart_subscriber_code`
     2. Tentar processar via `HotmartHandlerService::handle($evento)`
     3. Se sucesso: atualizar `processado = true`, `processado_em = now()`
     4. Se falha: salvar mensagem de erro em `$evento->erro`
     5. **Sempre retornar HTTP 200**
2. Adicionar variável ao `.env`: `HOTMART_WEBHOOK_TOKEN=`
3. Configurar em `config/services.php`:
   ```php
   'hotmart' => ['webhook_token' => env('HOTMART_WEBHOOK_TOKEN')],
   ```
4. Registrar rota **sem middleware de autenticação**:
   ```php
   Route::post('/webhook/hotmart', [WebhookHotmartController::class, 'receber']);
   ```

### Regras de negócio aplicáveis:
- **Sempre retornar HTTP 200** independente do resultado — a Hotmart reenvia se receber qualquer outro status
- Logar TODOS os eventos antes de processar — mesmo os inválidos
- O processamento pode falhar silenciosamente (erro salvo no banco) sem impactar o response

### Critérios de aceitação:
- [ ] `POST /api/webhook/hotmart` com token correto retorna 200 e loga o evento
- [ ] `POST /api/webhook/hotmart` com token incorreto retorna 200 (não 401)
- [ ] Evento é salvo na tabela `webhook_eventos` antes do processamento
- [ ] Falha no handler salva mensagem em `webhook_eventos.erro` mas ainda retorna 200
- [ ] Rota acessível sem autenticação Sanctum
