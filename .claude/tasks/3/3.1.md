# Task 3.1 – Migration e Model WebhookEvento

## Objetivo
Criar a migration da tabela `webhook_eventos` para log de todos os eventos recebidos da Hotmart e o Model correspondente.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_webhook_eventos_table.php` → criar
- `app/Models/WebhookEvento.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Models/Assinante.php` → ver padrão de Model adotado no projeto
- `database/migrations/` → verificar ordem de migrations existentes

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar migration `create_webhook_eventos_table` com colunas:
   - `id` (bigIncrements)
   - `event_type` (string — ex: `PURCHASE_APPROVED`, `SWITCH_PLAN`)
   - `hotmart_subscriber_code` (string, nullable)
   - `email` (string, nullable — email do comprador)
   - `payload` (json — payload completo do evento)
   - `processado` (boolean, default: false)
   - `processado_em` (timestamp, nullable)
   - `erro` (text, nullable — mensagem de erro se falhar)
   - `timestamps`
2. Criar `app/Models/WebhookEvento.php`:
   - `$fillable` com todos os campos
   - Cast `payload` para `array`
   - Cast `processado` para `boolean`
   - Cast `processado_em` para `datetime`
   - Scope `pendentes()`: `where('processado', false)`
   - Scope `porTipo(string $tipo)`: `where('event_type', $tipo)`

### Regras de negócio aplicáveis:
- Todo evento recebido da Hotmart deve ser logado, mesmo que falhe o processamento
- O campo `erro` deve registrar stack trace ou mensagem quando o handler falhar
- O campo `payload` deve guardar o JSON completo para reprocessamento futuro

### Critérios de aceitação:
- [ ] `php artisan migrate` cria a tabela `webhook_eventos` sem erros
- [ ] `WebhookEvento::create([...])` funciona com todos os campos
- [ ] Cast de `payload` funciona: ao recuperar, retorna array PHP
- [ ] Scope `pendentes()` filtra `processado = false`
- [ ] `php artisan migrate:status` mostra migration como executada

## Teste esperado
- Comando: `php artisan migrate:status`
- Verificação manual: `php artisan tinker` → `Schema::hasTable('webhook_eventos')` retorna `true`
