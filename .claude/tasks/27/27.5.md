# Task 27.5 – Endpoint Importar Addons (POST /admin/assinantes/addons/importar)

## Objetivo
Implementar o endpoint de importação de CSV/XLSX com processamento síncrono (≤500 linhas) ou assíncrono via job (>500 linhas), retornando relatório de erros por linha.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Api/Admin/AdminAssinanteAddonController.php` → modificar
- `backend/app/Jobs/ImportarAddonsJob.php` → modificar

## Especificação

### O que fazer:
Adicionar método `importar(Request $request)` ao controller:

**Validação da request:**
- `arquivo`: `required|file|mimes:csv,xlsx,xls|max:10240`

**Lógica de importação (função privada compartilhada `processarLinhas()`):**
Para cada linha do arquivo:
1. Buscar usuário por email (`User::where('email', $linha['email'])`)
2. Se não encontrado: registrar erro `{ linha: N, motivo: 'Usuário não encontrado: email@x.com' }`
3. Se encontrado: `updateOrCreate(['user_id' => $user->id, 'addon_key' => $linha['addon_key']], [...dados])`
4. Se status == ativo: chamar `AddonService::ativar()`

Formato esperado do CSV: `email, addon_key, status, fonte, iniciado_em, expira_em`

**Regra de despacho:**
- ≤ 500 linhas: processar no mesmo request, retornar `{ importados: N, erros: [...] }`
- > 500 linhas: salvar arquivo em storage, despachar `ImportarAddonsJob`, retornar `{ mensagem: 'Importação em andamento', job_id: '...' }`

**Completar ImportarAddonsJob** com a lógica de `processarLinhas()`.

**Rota:** `POST /admin/assinantes/addons/importar` no grupo admin
(Atenção: esta rota deve ser registrada ANTES das rotas com `{user}` para evitar conflito)

### Regras de negócio aplicáveis:
- Erros por linha não interrompem o processamento das demais linhas
- `updateOrCreate` evita duplicatas
- Sincronizar `AddonService` apenas para addons com `status=ativo`

### Critérios de aceitação:
- [ ] CSV com 3 linhas válidas retorna `{ importados: 3, erros: [] }`
- [ ] XLSX também é aceito
- [ ] Linha com email inexistente gera erro com número da linha e motivo
- [ ] Arquivo > 500 linhas retorna resposta de "em andamento" com job_id
- [ ] `updateOrCreate` não duplica registros em reimportação
