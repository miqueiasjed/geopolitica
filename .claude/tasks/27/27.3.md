## Especificação

### O que fazer:
**AdminAssinanteAddonController:**

```php
// GET /admin/assinantes/{user}/addons
public function index(User $user): JsonResponse
{
    $addons = AssinanteAddon::where('user_id', $user->id)
        ->orderBy('criado_em', 'desc')
        ->get();

    return response()->json($addons);
}

// POST /admin/assinantes/{user}/addons
public function store(AdicionarAddonRequest $request, User $user): JsonResponse
{
    $dados = $request->validated();
    $dados['user_id'] = $user->id;
    $dados['fonte']   = $dados['fonte'] ?? 'manual';

    $addon = AssinanteAddon::create($dados);

    // Se status == ativo, sincronizar via AddonService
    if ($addon->status === 'ativo') {
        app(AddonService::class)->ativar($user, $addon->addon_key);
    }

    return response()->json($addon, 201);
}
```

**Rotas** no grupo admin:
```
GET    /admin/assinantes/{user}/addons         → AdminAssinanteAddonController@index
POST   /admin/assinantes/{user}/addons         → AdminAssinanteAddonController@store
PUT    /admin/assinantes/{user}/addons/{addon} → AdminAssinanteAddonController@update
DELETE /admin/assinantes/{user}/addons/{addon} → AdminAssinanteAddonController@destroy
```

### Regras de negócio aplicáveis:
- Ao adicionar addon com `status=ativo`, chamar `AddonService::ativar()` para manter `assinantes.addons[]` sincronizado
- `fonte` padrão é `'manual'` quando não informado
- Listar todos os addons do assinante (ativos e histórico)

### Critérios de aceitação:
- [ ] `GET /admin/assinantes/{user}/addons` retorna lista de addons do assinante
- [ ] `POST /admin/assinantes/{user}/addons` com dados válidos cria addon e retorna 201
- [ ] Após `POST` com `status=ativo`, `temAddon()` retorna true para o assinante
- [ ] `addon_key` inexistente retorna 422 (validado pelo FormRequest)
- [ ] Usuário não-admin recebe 403
