# Task 21.5 – Página MonitorGuerra

## Objetivo
Criar a página `MonitorGuerra` com o feed de eventos militares filtrado (endpoint `GET /api/war-feed`) e briefings de guerra da Biblioteca, protegida pelo `AddonGate`.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/dashboard/MonitorGuerra.tsx` → criar
- `frontend/src/hooks/useWarFeed.ts` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/Feed.tsx` → padrão de feed com scroll infinito/paginação por cursor
- `frontend/src/hooks/useAddonAccess.ts` → após Task 21.3
- `frontend/src/components/AddonGate.tsx` → após Task 21.3

## Skills necessárias
- frontend-design-system
- animacoes-motion

## Especificação

### O que fazer:

**`useWarFeed.ts`:**
- Hook TanStack Query com `useInfiniteQuery`
- Endpoint: `GET /api/war-feed?cursor={cursor}&limit=20`
- Retorna páginas de eventos com `nextCursor` para scroll infinito
- Igual ao padrão de paginação já usado no `Feed.tsx`

**`MonitorGuerra.tsx`:**

1. No topo: verificar acesso com `useAddonAccess('war')`
   - Se `false`: renderizar `<AddonGate addonKey="war" />`

2. Se tem acesso — layout:
   - Header padrão: label muted uppercase + título `"Monitor de Guerra"` em serif
   - Subtítulo: `"Conflitos e tensões militares com impacto em mercados"`

3. Seção Feed Militar:
   - Título de seção: `"Feed de Tensões Militares"` em tracking-widest uppercase muted
   - Usar `useWarFeed` para carregar eventos
   - Reutilizar o card de evento do `Feed.tsx` (ou componente similar existente)
   - Botão "Carregar mais" ou scroll infinito (usar o mesmo padrão do Feed)
   - Estado de loading com skeleton, empty state com mensagem

4. Seção Briefings de Guerra:
   - Título: `"Briefings de Guerra Recentes"`
   - Buscar: `GET /api/conteudos?vertical=war&publicado=true&limit=10`
   - Mesmo padrão de lista da página MonitorEleitoral

### Regras de negócio aplicáveis:
- Usuário Reservado tem acesso sem add-on
- Usuário Essencial ou Pro só acessa com add-on `war`
- Feed mostra eventos com `categories=military` ou impacto CRÍTICO/ALTO

### Critérios de aceitação:
- [ ] Usuário Reservado vê página completa
- [ ] Usuário Pro sem add-on `war` vê `AddonGate` com CTA correto
- [ ] Feed militar carrega via API com paginação por cursor
- [ ] Briefings de guerra listados abaixo do feed
- [ ] Loading e empty state implementados

## Teste esperado
- Testar com usuário Reservado e usuário Pro (sem add-on) no browser
