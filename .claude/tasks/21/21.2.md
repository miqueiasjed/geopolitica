# Task 21.2 – Admin: campo vertical_conteudo no formulário de conteúdo

## Objetivo
Adicionar o campo `vertical_conteudo` ao formulário de criação/edição de conteúdo no painel admin, permitindo que o editor classifique um briefing como GPI Core, Monitor Eleitoral ou Monitor de Guerra.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/pages/admin/AdminNovoConteudo.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/AdminNovoConteudo.tsx` → ver estrutura atual do formulário e campos existentes

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

Adicionar campo `vertical_conteudo` ao formulário:
- Tipo: `<select>` ou RadioGroup
- Opções:
  - `""` (vazio/null) → "GPI Core (sem vertical)"
  - `"elections"` → "Monitor Eleitoral"
  - `"war"` → "Monitor de Guerra"
- Posicionamento: após o campo `plano_minimo`
- Valor default: `""` (null no backend)
- Incluir na lógica de submit: enviar `vertical_conteudo: valor || null`
- Incluir no estado inicial ao editar conteúdo existente

Usar os componentes de design system existentes (Radix Select ou input nativo com classes Tailwind do projeto).

### Regras de negócio aplicáveis:
- `vertical_conteudo = null` significa conteúdo do GPI Core (visível para todos com acesso ao plano mínimo)
- `vertical_conteudo = elections` significa que o conteúdo só aparece no Monitor Eleitoral
- `vertical_conteudo = war` significa que o conteúdo só aparece no Monitor de Guerra

### Critérios de aceitação:
- [ ] Campo exibido no formulário com as três opções
- [ ] Valor `""` é enviado como `null` para o backend
- [ ] Ao editar um conteúdo existente, o valor atual é pré-selecionado
- [ ] Submit inclui `vertical_conteudo` no payload enviado à API

## Teste esperado
- Abrir `/admin/conteudos/novo` e verificar campo presente
- Criar briefing com `vertical = elections` e confirmar salvo no banco
