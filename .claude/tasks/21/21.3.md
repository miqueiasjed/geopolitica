# Task 21.3 – Hook useAddonAccess e componente AddonGate

## Objetivo
Criar o hook `useAddonAccess` que verifica se o usuário tem acesso a uma vertical, e o componente `AddonGate` que exibe o bloqueio com CTA de compra quando o acesso é negado.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/hooks/useAddonAccess.ts` → criar
- `frontend/src/components/AddonGate.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useAuth.ts` (ou equivalente) → ver como plano e dados do usuário são acessados no contexto de auth
- `frontend/src/pages/Feed.tsx` → ver padrão de layout/estilo das páginas do dashboard para manter consistência

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

**`useAddonAccess(addonKey: 'elections' | 'war'): boolean`:**
- Ler `plano` e `addons` do contexto de autenticação existente
- Lógica:
  - `elections`: retorna `true` se `plano === 'pro' || plano === 'reservado' || addons.includes('elections')`
  - `war`: retorna `true` se `plano === 'reservado' || addons.includes('war')`
- Retornar `false` se não autenticado ou sem dados

**`AddonGate` — props: `addonKey: 'elections' | 'war'`:**

Configuração por addon_key:
```ts
const ADDON_INFO = {
  elections: {
    titulo: 'Monitor Eleitoral',
    descricao: 'Acompanhe eleições globais que movem mercados. Briefings eleitorais diários + radar visual dos próximos 12 meses.',
    preco: 'R$ 297/ano',
    ctaLabel: 'Adicionar Monitor Eleitoral',
    ctaUrl: import.meta.env.VITE_LASTLINK_ELECTIONS_URL || '#',
    upgradePlano: 'Pro',
  },
  war: {
    titulo: 'Monitor de Guerra',
    descricao: 'Feed em tempo real de conflitos armados e movimentações militares com impacto em mercados de energia e commodities.',
    preco: 'R$ 497/ano',
    ctaLabel: 'Adicionar Monitor de Guerra',
    ctaUrl: import.meta.env.VITE_LASTLINK_WAR_URL || '#',
    upgradePlano: 'Reservado',
  },
}
```

Layout do componente (manter estilo das páginas existentes):
- Label superior: `"Conteúdo exclusivo"` em tracking-widest uppercase muted
- Título da vertical em serif bold
- Descrição do que está por trás do bloqueio
- Box de preço com fundo sutil (estilo cartão, borda dourada fina)
- Botão CTA primário abrindo link externo em `_blank`
- Texto secundário sugerindo upgrade de plano

### Regras de negócio aplicáveis:
- `VITE_LASTLINK_ELECTIONS_URL` e `VITE_LASTLINK_WAR_URL` devem ser adicionados ao `.env.example` do frontend

### Critérios de aceitação:
- [ ] `useAddonAccess('elections')` retorna `true` para plano Pro sem add-on
- [ ] `useAddonAccess('war')` retorna `false` para plano Pro sem add-on war
- [ ] `AddonGate` renderiza informações corretas para cada addon_key
- [ ] Botão CTA abre URL do Lastlink em nova aba
- [ ] Variáveis de ambiente no `.env.example`
