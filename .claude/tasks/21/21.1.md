# Task 21.1 – Backend: endpoint GET /api/war-feed e filtro vertical em conteudos

## Objetivo
Criar o endpoint de feed militar filtrado (`GET /api/war-feed`) e atualizar o endpoint de conteúdos para suportar filtro e exposição de `vertical_conteudo`.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/WarFeedController.php` → criar
- `backend/app/Http/Controllers/ConteudoController.php` → modificar (adicionar filtro vertical)
- `backend/routes/api.php` → modificar (nova rota)

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/ConteudoController.php` → ver estrutura atual e como filtros são aplicados
- `backend/database/migrations/2026_04_14_180000_create_gdelt_cache_table.php` → ver campos da tabela eventos
- `backend/routes/api.php` → onde inserir a nova rota

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**`WarFeedController`:**

Método `index(Request $request)`:
- Parâmetros: `cursor` (timestamp ISO8601, opcional) e `limit` (default 20, max 50)
- Query na tabela `eventos`:
  - `is_active = true`
  - Filtro: `categories` contém `military` (usando `LIKE '%military%'` ou JSON_CONTAINS conforme tipo da coluna) **OU** `impact_label` IN `('CRÍTICO', 'ALTO')`
  - Ordenação: `impact_score DESC`, `published_at DESC`
  - Se `cursor` fornecido: `published_at < $cursor`
  - Limit + 1 (para detectar próxima página)
- Retornar JSON:
  ```json
  {
    "events": [...],
    "nextCursor": "<published_at do último item>" | null
  }
  ```
- Proteger com middleware `auth:sanctum` + Gate `acessar-vertical` com `war`

**`ConteudoController` — atualizar:**
- No método de listagem (index/listar), aceitar query param `vertical` (opcional)
- Se `vertical` informado: `->where('vertical_conteudo', $vertical)`
- Se `vertical` não informado: comportamento atual mantido
- Incluir `vertical_conteudo` no resource/array de retorno

**Rota nova:**
```php
Route::middleware('auth:sanctum')->get('/war-feed', [WarFeedController::class, 'index']);
```

### Regras de negócio aplicáveis:
- O gate `acessar-vertical` deve ser verificado antes de retornar dados — retornar 403 se não autorizado
- Paginação por cursor (não por página) para consistência com o Feed de Tensões existente

### Critérios de aceitação:
- [ ] `GET /api/war-feed` retorna apenas eventos com `categories` contendo `military` ou impacto crítico/alto
- [ ] Paginação por cursor funciona corretamente
- [ ] Usuário sem add-on `war` (e sem plano Reservado) recebe 403
- [ ] `GET /api/conteudos?vertical=elections` retorna apenas conteúdos com `vertical_conteudo = elections`
- [ ] `vertical_conteudo` aparece no payload de resposta dos conteúdos
