# Task 21.6 – Navegação condicional e rotas protegidas

## Objetivo
Atualizar a navegação lateral do dashboard para exibir Monitor Eleitoral e Monitor de Guerra condicionalmente, e registrar as rotas protegidas que renderizam `AddonGate` para acesso negado.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/components/layout/Sidebar.tsx` (ou equivalente de navegação) → modificar
- `frontend/src/router.tsx` (ou `App.tsx`) → modificar (adicionar rotas)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/components/layout/Sidebar.tsx` (ou arquivo de navegação equivalente) → ver como os itens de nav são definidos e renderizados
- `frontend/src/router.tsx` ou `frontend/src/App.tsx` → ver como as rotas são definidas
- `frontend/src/hooks/useAddonAccess.ts` → após Task 21.3

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

**Atualizar a navegação lateral:**
- Importar `useAddonAccess`
- Adicionar itens condicionalmente:
  ```ts
  const hasElections = useAddonAccess('elections')
  const hasWar = useAddonAccess('war')
  ```
- Inserir após o item "Eleições" (RadarEleicoes):
  - `{ href: '/dashboard/monitor-eleitoral', label: 'Monitor Eleitoral', show: hasElections }`
  - `{ href: '/dashboard/monitor-guerra', label: 'Monitor de Guerra', show: hasWar }`
- Quem não tem acesso não vê o item na nav — mas se acessar a URL, cai no `AddonGate` (comportamento das páginas)

**Atualizar o router:**
- Adicionar rota `/dashboard/monitor-eleitoral` → `<MonitorEleitoral />`
- Adicionar rota `/dashboard/monitor-guerra` → `<MonitorGuerra />`
- As páginas já fazem o controle interno via `useAddonAccess` + `AddonGate` — não precisa de route guard separado

### Regras de negócio aplicáveis:
- A navegação condicional é UX (não segurança) — a proteção real é feita nas páginas e no backend
- Itens ocultos na nav não impedem acesso via URL direta — o `AddonGate` cuida disso

### Critérios de aceitação:
- [ ] Monitor Eleitoral aparece na nav para usuários Pro e Reservado
- [ ] Monitor Eleitoral NÃO aparece para usuário Essencial sem add-on `elections`
- [ ] Monitor de Guerra aparece na nav apenas para usuário Reservado
- [ ] Acesso via URL direta sem permissão exibe `AddonGate` (não 404)
- [ ] Rotas registradas corretamente no router

## Teste esperado
- Logar com usuário de cada plano e verificar itens de nav visíveis
- Acessar `/dashboard/monitor-guerra` com usuário Essencial e confirmar AddonGate
