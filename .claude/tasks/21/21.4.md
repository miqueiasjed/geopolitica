# Task 21.4 – Página MonitorEleitoral

## Objetivo
Criar a página `MonitorEleitoral` que combina o componente `RadarEleicoes` existente (M8) com uma lista de briefings eleitorais da Biblioteca, protegida pelo `AddonGate`.

## Especificação

### O que fazer:

**`MonitorEleitoral.tsx`:**

1. No topo: verificar acesso com `useAddonAccess('elections')`
   - Se `false`: renderizar `<AddonGate addonKey="elections" />`

2. Se tem acesso — layout da página:
   - Header padrão das páginas (label muted uppercase + título em serif)
   - Título: `"Monitor Eleitoral"`
   - Subtítulo: `"Eleições globais com impacto para investidores"`

3. Seção Radar de Eleições:
   - Reutilizar diretamente o conteúdo de `RadarEleicoes.tsx` (importar os mesmos componentes: `ElectionsCalendar`, `RadarGrid`, etc.) OU redirecionar para a rota existente — preferir reutilizar componente

4. Seção Briefings Eleitorais:
   - Título de seção: `"Briefings Eleitorais Recentes"` em tracking-widest uppercase muted
   - Buscar conteúdos via `GET /api/conteudos?vertical=elections&publicado=true&limit=10`
   - Usar TanStack Query (`useQuery`)
   - Exibir lista de briefings (título, resumo, data de publicação) com link para leitura
   - Estado de loading com skeleton, estado vazio com mensagem neutra

### Regras de negócio aplicáveis:
- Página só renderiza conteúdo real se `useAddonAccess('elections')` retornar `true`
- Briefings são filtrados por `vertical_conteudo = elections` via query param

### Critérios de aceitação:
- [ ] Usuário Pro vê página completa (radar + briefings)
- [ ] Usuário Essencial sem add-on vê `AddonGate` com CTA correto
- [ ] Usuário Essencial com add-on `elections` vê página completa
- [ ] Seção de briefings carrega via API com loading e empty state
