# Task 34.8 – PaisPage Mobile: seções de gráficos empilhadas

## Objetivo
Tornar a PaisPage (perfil de país) utilizável em mobile empilhando seções de gráficos verticalmente e garantindo que gráficos side-by-side sejam responsivos.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/paises/PaisPage.tsx` → modificar (confirmar caminho exato)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/paises/PaisPage.tsx` → ver layout de seções (gráficos side-by-side, tabs de dados, header do país)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. Seções com `flex flex-row` ou `grid grid-cols-2` para gráficos: trocar por `flex flex-col md:flex-row` ou `grid grid-cols-1 md:grid-cols-2`.
2. Gráficos Recharts: verificar `<ResponsiveContainer width="100%" height={...}>` (igual à Task 34.3).
3. Header do país (nome, bandeira, indicadores principais): se usar `flex-row` fixo → `flex flex-wrap gap-2`.
4. Tabs de dados (histórico, indicadores, riscos): verificar que as tabs não transbordam horizontalmente em mobile. Se sim, adicionar `overflow-x-auto` no container das tabs ou usar `flex-wrap`.
5. Cards de indicadores numéricos: `grid grid-cols-2 sm:grid-cols-4` (não colapsar para 1 coluna — 2 colunas é suficiente em mobile para indicadores curtos).

### Regras de negócio aplicáveis:
- Cada seção da página representa um conjunto de dados do país — nenhuma seção deve ser ocultada em mobile.
- Gráficos de histórico podem ser altos em mobile — garantir scroll vertical (não horizontal).

### Critérios de aceitação:
- [ ] PaisPage em 375px: sem overflow horizontal.
- [ ] Seções de gráficos empilhadas verticalmente em mobile.
- [ ] Gráficos com 100% de largura e altura adequada em mobile.
- [ ] Tabs do perfil de país acessíveis (sem overflow horizontal das tabs).
- [ ] Cards de indicadores em grid 2-col em mobile.

## Teste esperado
- Em 375px: navegar para o perfil de um país, ver todas as seções empilhadas, gráficos legíveis, tabs clicáveis.
