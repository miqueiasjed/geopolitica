# Task 34.6 – ChatPage Mobile: uma tela por vez (lista OU chat ativo)

## Objetivo
Tornar a ChatPage utilizável em mobile implementando navegação de uma tela por vez: ou a lista de conversas ou o chat ativo, com botão "Voltar" para retornar à lista.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/ChatPage.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/ChatPage.tsx` → ver estrutura de layout (dois painéis: lista de conversas + área de chat), estado de conversa selecionada

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. Identificar o estado de conversa selecionada (provavelmente `conversaSelecionada` ou similar).
2. Em mobile (`< sm`), usar lógica condicional:
   ```tsx
   {/* Mobile: uma tela por vez */}
   <div className="flex flex-col sm:flex-row h-[calc(100vh-56px)]">
     {/* Lista de conversas */}
     <div className={cn(
       "w-full sm:w-80 border-r overflow-y-auto",
       conversaSelecionada ? "hidden sm:flex flex-col" : "flex flex-col"
     )}>
       {/* lista */}
     </div>

     {/* Área de chat */}
     <div className={cn(
       "flex-1 flex flex-col",
       !conversaSelecionada ? "hidden sm:flex" : "flex"
     )}>
       {/* Botão voltar mobile */}
       <div className="sm:hidden flex items-center gap-2 p-3 border-b">
         <button onClick={() => setConversaSelecionada(null)}>
           <ChevronLeftIcon /> Conversas
         </button>
       </div>
       {/* mensagens */}
     </div>
   </div>
   ```
3. O botão de input de mensagem deve estar sempre visível (não coberto por teclado virtual) — usar `sticky bottom-0` no input.
4. A altura do container deve ser `h-[calc(100vh-56px)]` considerando o header mobile do AdminLayout.

### Regras de negócio aplicáveis:
- Em desktop (sm+): layout dois painéis preservado.
- Em mobile: navegar para uma conversa mostra apenas o chat; pressionar voltar mostra apenas a lista.
- Nenhuma mensagem deve ser perdida ao alternar entre lista e chat.

### Critérios de aceitação:
- [ ] Em 375px: lista de conversas visível ao abrir ChatPage.
- [ ] Selecionar uma conversa: lista some, chat aparece com botão voltar.
- [ ] Botão voltar: chat some, lista reaparece.
- [ ] Input de mensagem visível sem scroll em 375px.
- [ ] Em sm+: dois painéis preservados.

## Teste esperado
- Em 375px: navegar entre lista e conversas individuais via botão voltar; enviar mensagem com input visível.
