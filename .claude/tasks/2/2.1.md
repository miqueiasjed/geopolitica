# Task 2.1 – Migration e Model Assinante

## Objetivo
Criar a migration da tabela `assinantes` e o Model `Assinante` com relacionamentos, castings e escopos necessários. Atualizar o Model `User` com o relacionamento `hasOne Assinante`.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/migrations/xxxx_create_assinantes_table.php` → criar
- `app/Models/Assinante.php` → criar
- `app/Models/User.php` → modificar (adicionar relacionamento)

## Contexto necessário (arquivos para ler antes)
- `app/Models/User.php` → entender estrutura base para adicionar `hasOne`
- `database/migrations/` → verificar ordem das migrations existentes

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar migration `create_assinantes_table` com colunas:
   - `id` (bigIncrements)
   - `user_id` (foreignId → users, onDelete cascade)
   - `plano` (string: `essencial` | `pro` | `reservado`)
   - `ativo` (boolean, default: false)
   - `status` (string: `ativo` | `cancelado` | `reembolsado` | `expirado` | `chargeback`)
   - `hotmart_subscriber_code` (string, nullable)
   - `assinado_em` (timestamp, nullable)
   - `expira_em` (timestamp, nullable)
   - `timestamps`
2. Criar `app/Models/Assinante.php`:
   - `$fillable` com todos os campos
   - Cast `ativo` para `boolean`
   - Cast `assinado_em` e `expira_em` para `datetime`
   - Relacionamento `belongsTo(User::class)`
   - Scope `ativo()`: `where('ativo', true)`
3. Atualizar `app/Models/User.php`:
   - Adicionar `hasOne(Assinante::class)`
   - Usar trait `HasRoles` do Spatie

### Regras de negócio aplicáveis:
- `ativo = false` por padrão — só fica `true` após evento `PURCHASE_APPROVED` ou `PURCHASE_COMPLETE` da Hotmart
- A tabela `assinantes` nunca é preenchida manualmente — sempre via webhook (Plano 3)

### Critérios de aceitação:
- [ ] `php artisan migrate` cria a tabela `assinantes` sem erros
- [ ] `Assinante::create([...])` funciona com os campos definidos
- [ ] `$user->assinante` retorna o relacionamento corretamente
- [ ] `$user->hasRole('assinante_pro')` funciona (trait HasRoles)
- [ ] Scope `Assinante::ativo()->get()` filtra apenas `ativo = true`

## Teste esperado
- Comando: `php artisan migrate:status`
- Verificação manual: `php artisan tinker` → `Schema::hasTable('assinantes')` retorna `true`
