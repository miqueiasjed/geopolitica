# Task 2.3 – AuthController (login, logout, me) com Sanctum

## Objetivo
Implementar o `AuthController` com os endpoints de login, logout e me, incluindo o `LoginRequest` com validação, e registrar as rotas de autenticação.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Auth/AuthController.php` → criar
- `app/Http/Requests/Auth/LoginRequest.php` → criar
- `routes/api.php` → modificar (adicionar rotas auth)

## Contexto necessário (arquivos para ler antes)
- `app/Models/User.php` → entender relacionamento com Assinante e HasRoles
- `app/Models/Assinante.php` → ver campos retornados no `me`
- `routes/api.php` → ver estrutura atual de rotas

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar `LoginRequest` com validação:
   - `email`: required, email
   - `password`: required, string
2. Criar `AuthController` com métodos:
   - `login(LoginRequest $request)`:
     - Verificar credenciais com `Auth::attempt`
     - Se falhar: retornar 401 com mensagem
     - Se sucesso: retornar token Sanctum via `$user->createToken('spa')->plainTextToken`
     - Retornar `{ token, user: { id, name, email, role, assinante } }`
   - `logout(Request $request)`:
     - Revogar token atual: `$request->user()->currentAccessToken()->delete()`
     - Retornar 204
   - `me(Request $request)`:
     - Retornar usuário autenticado com relacionamento `assinante` e role atual
3. Registrar rotas em `routes/api.php`:
   ```php
   Route::prefix('auth')->group(function () {
       Route::post('/login', [AuthController::class, 'login']);
       Route::middleware('auth:sanctum')->group(function () {
           Route::post('/logout', [AuthController::class, 'logout']);
           Route::get('/me', [AuthController::class, 'me']);
       });
   });
   ```

### Regras de negócio aplicáveis:
- Retornar sempre `{ token, user }` no login para o frontend armazenar no localStorage
- O endpoint `me` deve carregar `$user->load('assinante')` e retornar o role atual
- Nunca retornar o campo `password` no response

### Critérios de aceitação:
- [ ] `POST /api/auth/login` com credenciais válidas retorna `{ token, user }` com status 200
- [ ] `POST /api/auth/login` com credenciais inválidas retorna status 401
- [ ] `POST /api/auth/logout` com token válido revoga o token e retorna 204
- [ ] `GET /api/auth/me` retorna dados do usuário autenticado com role
- [ ] `GET /api/auth/me` sem token retorna 401
