# Task 2.2 – Seeder de Roles Spatie e usuário admin inicial

## Objetivo
Criar o seeder que registra os 4 roles Spatie Permission (`assinante_essencial`, `assinante_pro`, `assinante_reservado`, `admin`) e um usuário admin inicial para uso em desenvolvimento.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `database/seeders/RolesSeeder.php` → criar
- `database/seeders/AdminSeeder.php` → criar
- `database/seeders/DatabaseSeeder.php` → modificar (chamar os seeders)

## Contexto necessário (arquivos para ler antes)
- `database/seeders/DatabaseSeeder.php` → entender como chamar os seeders
- `app/Models/User.php` → verificar fillable e campos necessários

## Skills necessárias
- laravel-arquitetura
- permissoes-e-sync

## Especificação

### O que fazer:
1. Criar `RolesSeeder`:
   ```php
   Role::firstOrCreate(['name' => 'assinante_essencial', 'guard_name' => 'sanctum']);
   Role::firstOrCreate(['name' => 'assinante_pro', 'guard_name' => 'sanctum']);
   Role::firstOrCreate(['name' => 'assinante_reservado', 'guard_name' => 'sanctum']);
   Role::firstOrCreate(['name' => 'admin', 'guard_name' => 'sanctum']);
   ```
2. Criar `AdminSeeder`:
   - Criar `User` com `name`, `email` (de env `ADMIN_EMAIL`), `password` (bcrypt do env `ADMIN_PASSWORD`)
   - Atribuir role `admin` ao usuário
   - Usar `firstOrCreate` para ser idempotente
3. Atualizar `DatabaseSeeder` para chamar ambos em ordem:
   ```php
   $this->call([RolesSeeder::class, AdminSeeder::class]);
   ```
4. Adicionar `ADMIN_EMAIL` e `ADMIN_PASSWORD` ao `.env` e `.env.example`

### Regras de negócio aplicáveis:
- `guard_name` deve ser `sanctum` (não `web`) para todas as roles — o projeto usa Sanctum como guard
- Usar `firstOrCreate` em ambos os seeders para serem seguros em produção (idempotentes)

### Critérios de aceitação:
- [ ] `php artisan db:seed` executa sem erros
- [ ] 4 roles existem na tabela `roles` com `guard_name = sanctum`
- [ ] Usuário admin existe com role `admin` atribuída
- [ ] Executar o seeder duas vezes não duplica dados
- [ ] `$adminUser->hasRole('admin')` retorna `true`

## Teste esperado
- Comando: `php artisan db:seed --class=RolesSeeder`
- Verificação manual: `php artisan tinker` → `Role::all()->pluck('name')` lista os 4 roles
