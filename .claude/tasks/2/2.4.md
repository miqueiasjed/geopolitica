# Task 2.4 – SenhaController, PerfilController e middleware EnsureAssinanteAtivo

## Objetivo
Implementar o fluxo de redefinição de senha via e-mail, o endpoint de perfil (GET/PATCH), e o middleware que bloqueia assinantes inativos com HTTP 403.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Auth/SenhaController.php` → criar
- `app/Http/Controllers/PerfilController.php` → criar
- `app/Http/Middleware/EnsureAssinanteAtivo.php` → criar
- `app/Http/Requests/AtualizarPerfilRequest.php` → criar
- `routes/api.php` → modificar (adicionar rotas senha e perfil)
- `bootstrap/app.php` → modificar (registrar middleware alias)

## Contexto necessário (arquivos para ler antes)
- `app/Http/Controllers/Auth/AuthController.php` → ver padrão adotado
- `app/Models/Assinante.php` → ver campo `ativo`

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:
1. Criar `SenhaController`:
   - `esqueci(Request $request)`: valida email, dispara `Password::sendResetLink(['email' => $request->email])`, retorna 200
   - `redefinir(Request $request)`: valida token + email + password, chama `Password::reset(...)`, retorna 200 ou 422 se token inválido
2. Criar `AtualizarPerfilRequest` com validação:
   - `name`: sometimes, string, max:255
   - `email`: sometimes, email, unique:users,email,{user_id}
   - `password`: sometimes, confirmed, min:8
3. Criar `PerfilController`:
   - `show(Request $request)`: retorna `$request->user()->load('assinante')`
   - `update(AtualizarPerfilRequest $request)`: atualiza nome/email/senha, retorna usuário atualizado
4. Criar `EnsureAssinanteAtivo`:
   ```php
   if (!$request->user()->assinante?->ativo) {
       return response()->json(['message' => 'Assinatura inativa.'], 403);
   }
   return $next($request);
   ```
5. Registrar rotas:
   ```php
   Route::prefix('auth/senha')->group(function () {
       Route::post('/esqueci', [SenhaController::class, 'esqueci']);
       Route::post('/redefinir', [SenhaController::class, 'redefinir']);
   });
   Route::middleware(['auth:sanctum', 'assinante.ativo'])->group(function () {
       Route::get('/perfil', [PerfilController::class, 'show']);
       Route::patch('/perfil', [PerfilController::class, 'update']);
   });
   ```
6. Registrar alias `assinante.ativo` → `EnsureAssinanteAtivo` no `bootstrap/app.php`

### Regras de negócio aplicáveis:
- O middleware `EnsureAssinanteAtivo` só se aplica a rotas protegidas (não em login/reset de senha)
- Usuários com role `admin` devem passar pelo middleware mesmo sem `assinante` — verificar `$user->hasRole('admin')` e dar bypass se for admin

### Critérios de aceitação:
- [ ] `POST /api/auth/senha/esqueci` com email válido envia e-mail e retorna 200
- [ ] `POST /api/auth/senha/redefinir` com token válido redefine a senha
- [ ] `GET /api/perfil` retorna dados do usuário autenticado
- [ ] `PATCH /api/perfil` atualiza nome/email/senha corretamente
- [ ] Usuário com `assinante.ativo = false` recebe 403 nas rotas protegidas com o middleware
- [ ] Usuário com role `admin` passa pelo middleware sem bloqueio
