# Task 2.6 – Páginas React: Login, EsqueciSenha, RedefinirSenha e Perfil

## Objetivo
Criar as 4 páginas de autenticação com design escuro (#0a0a0b), formulários funcionais integrados com `useAuth` e TanStack Query, e configurar as rotas públicas/privadas no React Router.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/Login.tsx` → criar
- `frontend/src/pages/EsqueciSenha.tsx` → criar
- `frontend/src/pages/RedefinirSenha.tsx` → criar
- `frontend/src/pages/Perfil.tsx` → criar
- `frontend/src/App.tsx` → criar/modificar (configurar rotas)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/contexts/AuthContext.tsx` → ver interface do AuthContext (task 2.5)
- `frontend/src/components/RotaProtegida.tsx` → ver como usar nas rotas privadas
- `frontend/src/types/auth.ts` → ver tipos disponíveis

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:
1. Criar `Login.tsx`:
   - Fundo `bg-[#0a0a0b]`, card centralizado
   - Campos: email, password
   - Botão submit com loading state
   - Usar `useAuth().login()` ao submeter
   - Link para `/esqueci-senha`
   - Redirecionar para `/` após login bem-sucedido
2. Criar `EsqueciSenha.tsx`:
   - Campo email + botão enviar
   - Usar `useMutation` do TanStack Query para `POST /api/auth/senha/esqueci`
   - Mostrar mensagem de sucesso após envio
3. Criar `RedefinirSenha.tsx`:
   - Ler params `token` e `email` da URL (query string)
   - Campos: password, password_confirmation
   - Usar `useMutation` para `POST /api/auth/senha/redefinir`
   - Redirecionar para `/login` após sucesso
4. Criar `Perfil.tsx`:
   - Usar `useQuery` para `GET /api/perfil`
   - Formulário de edição: nome, email, senha (opcional)
   - Usar `useMutation` para `PATCH /api/perfil`
   - Exibir badge com plano/role do usuário
5. Configurar rotas em `App.tsx`:
   ```tsx
   <Routes>
     <Route path="/login" element={<Login />} />
     <Route path="/esqueci-senha" element={<EsqueciSenha />} />
     <Route path="/redefinir-senha" element={<RedefinirSenha />} />
     <Route element={<RotaProtegida />}>
       <Route path="/perfil" element={<Perfil />} />
       {/* demais rotas protegidas virão aqui */}
     </Route>
   </Routes>
   ```

### Regras de negócio aplicáveis:
- Design: fundo `#0a0a0b`, texto branco/cinza, sem tema claro
- Não há registro público — omitir qualquer link de "criar conta"
- Feedback de erro deve ser mostrado inline no formulário (não apenas toast)

### Critérios de aceitação:
- [ ] Página de login funcional: login correto redireciona, login errado mostra erro
- [ ] Página "Esqueci Senha" envia e-mail e mostra confirmação
- [ ] Página "Redefinir Senha" lê token da URL e redireciona após sucesso
- [ ] Página de Perfil exibe dados do usuário e permite editar nome/email/senha
- [ ] Rotas públicas acessíveis sem autenticação
- [ ] Rotas protegidas redirecionam para login quando não autenticado
- [ ] Design escuro consistente em todas as páginas

## Teste esperado
- Verificação manual: fluxo completo — login → perfil → logout → tentar acessar /perfil → redirecionado para /login
