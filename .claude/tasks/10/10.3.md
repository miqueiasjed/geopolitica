# Task 10.3 – AnalisadorConvergenciaService, EntregaAlertaService e Jobs

Implementar o serviço de análise de convergência de sinais (que gera alertas preditivos), o serviço de entrega de e-mails via Resend, os Jobs correspondentes e o scheduler.

## Especificação

### O que fazer:

1. **`AnalisadorConvergenciaService`**:

   - `analisar(): void`
     - Busca sinais das últimas 72h (`CONVERGENCIA_JANELA_HORAS` do `.env`, default 72)
     - Agrupa por `regiao`
     - Para cada região com sinais: chama `verificarConvergencia($regiao, $sinais)`

   - `verificarConvergencia(string $regiao, Collection $sinais): void`
     - Conta tipos únicos presentes: `$tiposUnicos = $sinais->pluck('tipo_padrao')->unique()`
     - Se menos de 2 tipos únicos: sem convergência, retorna
     - Soma pesos: `$pesoTotal = $sinais->sum('peso')`
     - Determina nível:
       - `>= ALERTA_THRESHOLD_CRITICAL (env, default 10)` → `critical`
       - `>= ALERTA_THRESHOLD_HIGH (env, default 7)` → `high`
       - `>= ALERTA_THRESHOLD_MEDIUM (env, default 4)` → `medium`
       - abaixo → sem alerta
     - Anti-duplicata: verifica se já existe alerta para a mesma região nas últimas 48h
     - Se novo alerta necessário: chama `gerarAlerta()`

   - `gerarAlerta(string $regiao, string $nivel, Collection $sinais, int $pesoTotal): void`
     - Monta prompt para Claude com os sinais da região
     - Chama `claude-sonnet-4-6` para gerar análise (mais completa que Haiku)
     - Cria `AlertaPreditivo` com título, análise, resumo_sinais, tipos_padrao, peso_total, nivel
     - Despacha `EnviarEmailAlertaJob::dispatch($alerta->id)` se nível `high` ou `critical`

2. **`EntregaAlertaService`**:

   - `enviarEmails(AlertaPreditivo $alerta): void`
     - Busca todos os `User` com role `assinante_reservado` ou `admin`
     - Envia e-mails em lotes de 50 via Resend
     - Usa `Mail::to($usuarios)->send(new AlertaPreditivoMail($alerta))`
     - Após envio bem-sucedido: `$alerta->update(['notificado_em' => now()])`

3. **`AnalisarConvergenciaJob`**:
   - `$queue = 'alertas'`, `$tries = 3`, `$timeout = 180`, `$backoff = 60`
   - `handle()`: chama `AnalisadorConvergenciaService::analisar()`

4. **`EnviarEmailAlertaJob`** (recebe `$alertaId`):
   - `$queue = 'emails'`, `$tries = 3`, `$timeout = 300`
   - `handle()`: busca alerta por id e chama `EntregaAlertaService::enviarEmails($alerta)`

5. **`Console/Kernel.php`** — adicionar:
   ```php
   $schedule->job(new DetectarSinaisJob)->hourly()->withoutOverlapping();
   $schedule->job(new AnalisarConvergenciaJob)->hourlyAt(5)->withoutOverlapping();
   ```

### Regras de negócio aplicáveis:
- Convergência exige 2+ tipos únicos na mesma região na janela de tempo
- Anti-duplicata: uma região não gera dois alertas em 48h
- `claude-sonnet-4-6` para análise narrativa; `claude-haiku-4-5-20251001` para classificação
- E-mails apenas para `assinante_reservado` e `admin`
- `notificado_em` só atualizado após envio confirmado

### Critérios de aceitação:
- [ ] `AnalisadorConvergenciaService::analisar()` gera alertas quando convergência detectada
- [ ] Mesma região não gera dois alertas em menos de 48h
- [ ] Nível calculado corretamente conforme variáveis de ambiente
- [ ] `EntregaAlertaService::enviarEmails()` envia apenas para roles corretos
- [ ] `notificado_em` atualizado após envio
- [ ] Scheduler registrado para detecção (hourly) e convergência (hourlyAt 5)
