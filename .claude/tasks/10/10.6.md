## Especificação

### O que fazer:

1. **`src/types/alertas.ts`**:
   ```ts
   export type NivelAlerta = 'medium' | 'high' | 'critical'

   export interface ResumoSinal {
     titulo: string
     tipo: 'military' | 'diplomatic' | 'supply'
   }

   export interface Alerta {
     id: number
     nivel: NivelAlerta
     regiao: string
     titulo: string
     analise: string
     resumo_sinais: ResumoSinal[]
     tipos_padrao: string[]
     peso_total: number
     created_at: string
   }

   export interface AlertasResponse {
     alertas: Alerta[]
     total_nao_lidos: number
   }
   ```

2. **`useAlertas.ts`**:
   - `useQuery` com chave `['alertas']`
   - Query function: `GET /api/alertas`
   - `refetchInterval: 5 * 60 * 1000` (5 minutos — polling)
   - Expõe: `alertas: Alerta[]`, `totalNaoLidos: number`, `isLoading`

3. **`useMarcarAlertaLido.ts`**:
   - `useMutation` para `POST /api/alertas/:id/lido`
   - `onSuccess`: invalida query `['alertas']` (atualiza contagem do badge)
   - Trata 409 silenciosamente (alerta já lido não é erro visível)
   - Expõe: `marcarLido(id: number): void`, `isPending`

4. **`AlertaBadge.tsx`** (sem props — usa hook internamente):
   - Ícone de sino (heroicons ou similar)
   - Badge vermelho com contagem quando `totalNaoLidos > 0`:
     - Número exibido: até 99; se > 99 exibe "99+"
     - Animação de pulso (`animate-pulse` ou Framer Motion `scale` loop)
   - Quando `totalNaoLidos === 0`: badge oculto (não renderiza o número)
   - Ao clicar: abre/fecha o `AlertaPanel` (via estado local ou contexto)
   - Acessibilidade: `aria-label="Alertas: X não lidos"`

### Regras de negócio aplicáveis:
- Polling de 5 minutos no frontend sincroniza com atualização do backend
- Badge some quando não há alertas não lidos (não exibe "0")
- Contagem > 99 exibe "99+" para não quebrar layout

### Critérios de aceitação:
- [ ] `useAlertas` retorna lista e contagem corretamente
- [ ] `useMarcarAlertaLido` invalida query `['alertas']` após sucesso
- [ ] Badge vermelho visível apenas quando `totalNaoLidos > 0`
- [ ] "99+" exibido quando contagem passa de 99
- [ ] Animação de pulso ativa no badge
- [ ] TypeScript compila sem erros
