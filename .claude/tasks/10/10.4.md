# Task 10.4 – AlertaController, AlertaPreditivoService e Rotas

## Objetivo
Implementar o serviço de listagem de alertas filtrados por role, o controller com os endpoints de listagem e marcação como lido, e as rotas correspondentes.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Services/AlertaPreditivoService.php` → criar
- `app/Http/Controllers/Api/AlertaController.php` → criar
- `routes/api.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `app/Models/AlertaPreditivo.php` → ver scopes `visivelPara()` e `naoLidosPor()`
- `app/Models/AlertaLeitura.php` → ver estrutura para criação de leitura
- `routes/api.php` → ver estrutura de grupos de rotas existentes

## Skills necessárias
- laravel-arquitetura
- permissoes-e-sync

## Especificação

### O que fazer:

1. **`AlertaPreditivoService`**:

   - `alertasNaoLidos(int $userId, string $role): array`
     - Aplica `AlertaPreditivo::visivelPara($role)->naoLidosPor($userId)->orderByDesc('created_at')->get()`
     - Retorna array com `['alertas' => [...], 'total_nao_lidos' => int]`

   - `marcarLido(int $alertaId, int $userId): array`
     - Verifica se já existe `AlertaLeitura` para o par `(user_id, alerta_id)`
     - Se já existe: retorna `['success' => false, 'message' => 'Alerta já marcado como lido']` (Controller retornará 409)
     - Se não existe: cria `AlertaLeitura` com `lido_em = now()`
     - Retorna `['success' => true]`

2. **`AlertaController`**:

   - `index(Request $request): JsonResponse`
     - Obtém `$userId = $request->user()->id` e `$role = $request->user()->getRoleNames()->first()`
     - Chama `AlertaPreditivoService::alertasNaoLidos($userId, $role)`
     - Retorna JSON com `alertas` e `total_nao_lidos`

   - `marcarLido(Request $request, int $id): JsonResponse`
     - Verifica que o alerta existe (404 se não)
     - Chama `AlertaPreditivoService::marcarLido($id, $request->user()->id)`
     - Se `success = false` → retorna HTTP 409 com mensagem
     - Se sucesso → retorna HTTP 200

   - Middleware no construtor: `auth:sanctum`

3. **Rotas em `routes/api.php`** (grupo `auth:sanctum`):
   ```php
   Route::get('/alertas', [AlertaController::class, 'index']);
   Route::post('/alertas/{id}/lido', [AlertaController::class, 'marcarLido']);
   ```

### Regras de negócio aplicáveis:
- `essencial` → apenas `medium`; `pro` → `medium`+`high`; `reservado`/`admin` → todos
- 409 (não 422) quando alerta já foi marcado como lido
- Controller não contém lógica de filtro — tudo no Service

### Critérios de aceitação:
- [ ] `GET /api/alertas` com token `assinante_essencial` retorna apenas alertas `medium`
- [ ] `GET /api/alertas` com token `assinante_reservado` retorna todos os níveis
- [ ] `POST /api/alertas/{id}/lido` retorna 200 na primeira chamada
- [ ] `POST /api/alertas/{id}/lido` retorna 409 na segunda chamada (mesmo alerta)
- [ ] `GET /api/alertas` sem token retorna 401
