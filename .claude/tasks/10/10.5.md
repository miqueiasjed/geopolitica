## Especificação

### O que fazer:

1. **`AlertaPreditivoMail`**:
   - `public function __construct(public AlertaPreditivo $alerta)`
   - `envelope()`: assunto dinâmico por nível:
     - `critical` → "🚨 Alerta Crítico: {titulo}"
     - `high` → "⚠️ Alerta Alto: {titulo}"
     - `medium` → "📊 Alerta: {titulo}"
   - `content()`: usa a view `emails.alerta-preditivo`
   - `attachments()`: retorna `[]`

2. **`resources/views/emails/alerta-preditivo.blade.php`**:
   - Template HTML responsivo simples (sem framework CSS no e-mail)
   - Cabeçalho com badge de nível (cores: critical vermelho, high laranja, medium amarelo)
   - Região e data de criação
   - Título do alerta em destaque
   - Análise IA como parágrafo principal
   - Lista dos sinais detectados (`resumo_sinais`)
   - Rodapé: "Geopolítica para Investidores — Você recebe este e-mail por ser assinante Reservado."
   - Link "Ver no dashboard" apontando para `APP_URL/dashboard/alertas`

3. **`CronSecretMiddleware`**:
   - Verifica header `X-Cron-Secret` ou query param `cron_secret`
   - Compara com `config('app.cron_secret')` usando `hash_equals` (timing-safe)
   - Se inválido: retorna HTTP 403
   - Usar para rotas internas do scheduler (não implementadas neste plano, mas middleware pronto)

4. **Registrar middleware** em `bootstrap/app.php`:
   ```php
   ->withMiddleware(function (Middleware $middleware) {
       $middleware->alias(['cron.secret' => CronSecretMiddleware::class]);
   })
   ```

5. **`.env.example`** — adicionar:
   ```
   RESEND_API_KEY=
   RESEND_FROM_EMAIL=alertas@geopoliticaparainvestidores.com.br
   CRON_SECRET=
   ALERTA_THRESHOLD_CRITICAL=10
   ALERTA_THRESHOLD_HIGH=7
   ALERTA_THRESHOLD_MEDIUM=4
   CONVERGENCIA_JANELA_HORAS=72
   ```

### Regras de negócio aplicáveis:
- `hash_equals` obrigatório no middleware para prevenir timing attacks
- E-mail de alerta enviado apenas para `assinante_reservado` e `admin` (controlado no `EntregaAlertaService`)
- Template de e-mail sem dependência de CSS externo (inline styles apenas)

### Critérios de aceitação:
- [ ] `Mail::to($user)->send(new AlertaPreditivoMail($alerta))` não lança exceção
- [ ] Assunto do e-mail reflete corretamente o nível do alerta
- [ ] Middleware `cron.secret` rejeita request sem o header correto com HTTP 403
- [ ] `.env.example` documenta todas as variáveis novas
- [ ] Template Blade renderiza sem erros com alerta de teste
