# Task 32.1 – Estado menuAberto + Header Mobile no AdminLayout

## Objetivo
Adicionar o estado `menuAberto` (useState) e o header mobile sticky (h-14) com logo/título à esquerda e ícone hamburger/fechar à direita, visível apenas em telas `< lg`.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/layouts/AdminLayout.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/layouts/AdminLayout.tsx` → entender estrutura atual (sidebar, links, classes desktop)
- `frontend/src/layouts/DashboardLayout.tsx` → ver como o hamburger já foi feito no layout assinante (referência de padrão)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. Importar `useState` do React.
2. Adicionar `const [menuAberto, setMenuAberto] = useState(false)` no topo do componente.
3. Adicionar header mobile logo abaixo do elemento raiz, **antes** do sidebar desktop:
   ```tsx
   <header className="lg:hidden sticky top-0 z-30 h-14 bg-background border-b flex items-center justify-between px-4">
     <span className="font-semibold text-sm">Geopolítica – Admin</span>
     <button onClick={() => setMenuAberto(v => !v)} aria-label="Menu">
       {menuAberto ? <Cross2Icon className="h-5 w-5" /> : <HamburgerMenuIcon className="h-5 w-5" />}
     </button>
   </header>
   ```
4. Importar `HamburgerMenuIcon` e `Cross2Icon` de `@radix-ui/react-icons`.
5. O sidebar desktop existente deve ganhar `hidden lg:flex` (se ainda não tiver) para sumir em mobile.
6. O conteúdo principal deve ter `pt-0 lg:pl-56` (sem padding-top extra em desktop, sem pl em mobile).

### Regras de negócio aplicáveis:
- O header mobile **nunca** aparece em `lg` (1024px+).
- O sidebar desktop permanece **exatamente** como está em `lg+`.
- Não remover nenhuma classe ou lógica existente do sidebar desktop.

### Critérios de aceitação:
- [ ] `menuAberto` é um estado React controlado por `useState`.
- [ ] Header mobile aparece em 375px e desaparece em 1024px.
- [ ] Ícone alterna entre HamburgerMenuIcon (fechado) e Cross2Icon (aberto).
- [ ] Sidebar desktop continua visível e funcional em `lg+`.
- [ ] Sem overflow horizontal em 375px após as mudanças.

## Teste esperado
- Redimensionar para 375px: header com hamburger aparece, sidebar some.
- Redimensionar para 1024px+: header desaparece, sidebar desktop aparece.
