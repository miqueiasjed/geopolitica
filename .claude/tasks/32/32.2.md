# Task 32.2 – Drawer Mobile com Framer Motion no AdminLayout

## Objetivo
Implementar o drawer sidebar overlay em mobile usando Framer Motion (`AnimatePresence` + `motion.aside`) que desliza da esquerda quando `menuAberto` é `true`, com backdrop semitransparente que fecha o drawer ao clicar fora.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/layouts/AdminLayout.tsx` → modificar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/layouts/AdminLayout.tsx` → estado após Task 32.1 (precisa de menuAberto + header já adicionados)
- `frontend/src/layouts/DashboardLayout.tsx` → ver padrão de AnimatePresence já usado no projeto

## Skills necessárias
- `frontend-design-system`
- `animacoes-motion`

## Especificação

### O que fazer:
1. Importar `AnimatePresence`, `motion` do `framer-motion`.
2. Adicionar o drawer mobile **logo após** o header mobile (antes do sidebar desktop):
   ```tsx
   <AnimatePresence>
     {menuAberto && (
       <>
         {/* Backdrop */}
         <motion.div
           className="lg:hidden fixed inset-0 z-40 bg-black/50"
           initial={{ opacity: 0 }}
           animate={{ opacity: 1 }}
           exit={{ opacity: 0 }}
           onClick={() => setMenuAberto(false)}
         />
         {/* Drawer */}
         <motion.aside
           className="lg:hidden fixed top-0 left-0 z-50 h-full w-56 bg-background border-r overflow-y-auto"
           initial={{ x: -224 }}
           animate={{ x: 0 }}
           exit={{ x: -224 }}
           transition={{ type: "tween", duration: 0.25 }}
         >
           {/* Conteúdo: copiar links/itens do sidebar desktop aqui */}
         </motion.aside>
       </>
     )}
   </AnimatePresence>
   ```
3. O conteúdo do drawer (logo, links de navegação) deve ser **idêntico** ao sidebar desktop — copiar os elementos, não referenciar via componente separado.
4. Adicionar `overflow-y-auto` no drawer para scroll interno.
5. O backdrop usa `z-40`, o drawer usa `z-50`.

### Regras de negócio aplicáveis:
- Drawer só aparece em mobile (`lg:hidden` garante isso, mas também condicionar pelo estado `menuAberto`).
- A animação de entrada é `x: -224 → 0` (desliza da esquerda).
- A animação de saída é `x: 0 → -224`.
- Clicar no backdrop fecha o drawer (chama `setMenuAberto(false)`).
- O sidebar desktop **não é modificado** por esta task.

### Critérios de aceitação:
- [ ] Drawer aparece com animação suave ao abrir.
- [ ] Backdrop semitransparente cobre o restante da tela.
- [ ] Clicar no backdrop fecha o drawer com animação de saída.
- [ ] Drawer tem scroll interno quando conteúdo excede a altura da tela.
- [ ] Sem interferência com o sidebar desktop em `lg+`.

## Teste esperado
- Em 375px: clicar no hamburger → drawer desliza da esquerda. Clicar fora → fecha com animação.
