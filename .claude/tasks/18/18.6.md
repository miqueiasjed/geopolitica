# Task 18.6 – Página /admin/ia/uso: cards de resumo + tabela de breakdown por serviço

## Objetivo
Criar a página React `/admin/ia/uso` com os cards de resumo (hoje/mês) e a tabela de breakdown por serviço, usando o hook `useAiUso`.

frontend-pagina

## Especificação

### Layout da página:
```
/admin/ia/uso
├── Título: "Painel de Uso da IA"
├── [Aviso threshold — renderizado apenas se custo_mes >= threshold_alerta_usd]
├── Grid 2x2: cards de resumo (Hoje)
│   ├── Chamadas hoje
│   ├── Tokens hoje (entrada + saída)
│   ├── Custo estimado hoje (USD)
│   └── Duração média hoje (ms)
├── Grid 2x2: cards de resumo (Mês atual)
│   ├── Chamadas mês
│   ├── Tokens mês
│   ├── Custo estimado mês (USD)
│   └── Provider ativo + modelo
├── [Gráfico 7 dias — componente separado, Task 18.7]
└── Tabela de breakdown por serviço
    ├── Colunas: Serviço | Chamadas | Custo Estimado
    └── Ordenada por chamadas desc
```

### Cards de resumo:
Cada card exibe: label, valor principal formatado, subtítulo (ex: "estimado").
- Custo em USD: formatar como `$0.001234` (6 casas decimais quando < $0.01, 4 quando >= $0.01 e < $1, 2 quando >= $1).
- Tokens: formatar com separador de milhar (ex: "18.000").
- Duração: `1.240 ms` arredondado para inteiro.

### Aviso de threshold:
Se `data.mes.custo_estimado_usd >= data.threshold_alerta_usd`, renderizar um banner amarelo/laranja com o texto: "⚠ Custo estimado do mês (${X}) atingiu o limite de alerta (${Y})."

### Tabela de serviços:
Usar `<table>` HTML padrão com classes Tailwind. Linhas de dados de `data.por_servico`. Se array vazio, mostrar mensagem "Nenhuma chamada registrada este mês."

### Estados:
- Loading: skeleton ou spinner centralizado.
- Error: mensagem de erro.
- Empty (sem logs): cards com zeros.

### Rota:
Adicionar no arquivo de rotas a rota `/admin/ia/uso` apontando para `UsoPage`. Verificar o padrão de rotas admin existente (lazy import, proteção de autenticação).

### Regras de negócio aplicáveis:
- Página acessível apenas para admins (usar o mesmo guard das outras páginas admin).
- Não há filtros de data — exibe sempre hoje / mês atual / últimos 7 dias.
- Custo é sempre rotulado como "estimado".

### Critérios de aceitação:
- [ ] Rota `/admin/ia/uso` acessível e renderiza a página.
- [ ] 4 cards de "Hoje" e 4 cards de "Mês" exibidos.
- [ ] Tabela de breakdown por serviço renderizada.
- [ ] Aviso de threshold aparece quando custo >= limite.
- [ ] Loading state visível durante fetch.
- [ ] TypeScript compila sem erros.
