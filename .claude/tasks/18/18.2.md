# Task 18.2 – Integrar logging fire-and-forget nos providers ClaudeProvider e OpenAiProvider

## Objetivo
Adicionar registro automático em `ai_logs` nos providers `ClaudeProvider` e `OpenAiProvider` após cada chamada `complete()`, de forma fire-and-forget (falha de log nunca bloqueia a chamada à IA).

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/Ai/ClaudeProvider.php` → modificar
- `app/Services/Ai/OpenAiProvider.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `app/Services/Ai/ClaudeProvider.php` → ler método `complete()` completo para entender fluxo atual
- `app/Services/Ai/OpenAiProvider.php` → idem
- `app/Models/AiLog.php` → interface do model (Task 18.1)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Em ambos os providers, envolver o bloco de chamada à API com medição de tempo e capturar tokens reais quando disponíveis na resposta.

**Padrão a aplicar em cada provider (método `complete`):**

```php
$inicio = microtime(true);
$sucesso = true;
$erro = null;
$tokensEntrada = 0;
$tokensSaida = 0;

try {
    // ... chamada original à API ...
    // capturar tokens da resposta se disponíveis (ver abaixo)
    return $texto;
} catch (\Exception $e) {
    $sucesso = false;
    $erro = $e->getMessage();
    throw $e; // relançar — o log não suprime a exceção
} finally {
    $duracaoMs = (int) round((microtime(true) - $inicio) * 1000);
    try {
        AiLog::create([
            'provider'          => 'claude', // ou 'openai'
            'modelo'            => config('claude.model'), // ou config('ai.openai.model')
            'servico'           => 'geral', // será sobrescrito quando passado via contexto (ver nota)
            'tokens_entrada'    => $tokensEntrada,
            'tokens_saida'      => $tokensSaida,
            'custo_estimado_usd'=> AiLog::calcularCusto('claude', config('claude.model'), $tokensEntrada, $tokensSaida),
            'duracao_ms'        => $duracaoMs,
            'sucesso'           => $sucesso,
            'erro'              => $erro,
        ]);
    } catch (\Exception) {
        // fire-and-forget: falha silenciosa de log
    }
}
```

**Captura de tokens reais:**
- **ClaudeProvider:** A resposta da SDK Anthropic tem `$response->usage->inputTokens` e `$response->usage->outputTokens`. Capturar após a chamada bem-sucedida.
- **OpenAiProvider:** `$response->usage->promptTokens` e `$response->usage->completionTokens`. Capturar após a chamada bem-sucedida.

**Nota sobre `servico`:** Por ora registrar como `'geral'`. O Plano 18 prevê que no futuro os services individuais (AiAnalyzerService, ChatService, etc.) possam passar o nome do serviço via contexto — mas isso não está no escopo desta task. A coluna `servico` aceita `'geral'` como placeholder válido.

Não modificar o método `stream()` — logging de streaming fica fora de escopo.

### Regras de negócio aplicáveis:
- O `try/catch` no bloco `finally` garante que falha de log nunca propaga exceção para o caller.
- Tokens estimados por `strlen/4` só se a SDK não fornecer tokens reais.
- Não alterar a assinatura dos métodos `complete()`.

### Critérios de aceitação:
- [ ] Após uma chamada bem-sucedida a `ClaudeProvider::complete()`, existe um registro em `ai_logs` com `sucesso=true`.
- [ ] Após uma chamada falhada (ex: API key inválida), existe um registro com `sucesso=false` e `erro` preenchido.
- [ ] Falha ao salvar o log (ex: banco desconectado) não lança exceção — a chamada à IA retorna normalmente.
- [ ] `tokens_entrada` e `tokens_saida` refletem os valores reais da SDK quando disponíveis.
- [ ] `duracao_ms` > 0 em todos os casos.
