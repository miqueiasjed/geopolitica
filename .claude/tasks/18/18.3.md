# Task 18.3 – Command LimparAiLogsAntigos + agendamento no Kernel

## Objetivo
Criar um Artisan command que deleta registros de `ai_logs` com mais de 90 dias e agendá-lo para rodar diariamente, mantendo o banco enxuto.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Console/Commands/LimparAiLogsAntigos.php` → criar
- `app/Console/Kernel.php` → modificar (adicionar schedule)

## Contexto necessário (arquivos para ler antes)
- `app/Console/Kernel.php` → ver o padrão de agendamento existente no projeto
- `app/Models/AiLog.php` → ver campos disponíveis (Task 18.1)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### Command `LimparAiLogsAntigos`:
```php
protected $signature = 'ai:limpar-logs {--dias=90 : Número de dias para reter logs}';
protected $description = 'Remove registros de ai_logs com mais de N dias';

public function handle(): int
{
    $dias = (int) $this->option('dias');
    $corte = now()->subDays($dias);
    $deletados = AiLog::where('created_at', '<', $corte)->delete();
    $this->info("ai_logs: {$deletados} registros removidos (mais de {$dias} dias).");
    return Command::SUCCESS;
}
```

### Agendamento em Kernel.php:
```php
$schedule->command('ai:limpar-logs')->dailyAt('03:00');
```

### Regras de negócio aplicáveis:
- TTL padrão é 90 dias; pode ser sobrescrito com `--dias=30` para testes.
- O horário `03:00` evita janela de pico.
- Usar `delete()` direto (sem soft delete — logs não precisam de soft delete).

### Critérios de aceitação:
- [ ] `php artisan ai:limpar-logs` executa sem erros e exibe contagem de deletados.
- [ ] `php artisan ai:limpar-logs --dias=0` deleta todos os registros (útil para testes).
- [ ] `php artisan schedule:list` exibe `ai:limpar-logs` agendado para `03:00`.
- [ ] Registros com `created_at` dentro dos últimos 90 dias não são deletados.
