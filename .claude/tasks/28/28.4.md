# Task 28.4 – Atualizar useAddonAccess.ts para delegar ao useProdutos

## Objetivo
Refatorar `useAddonAccess.ts` para buscar dados do hook `useProdutos` em vez de fonte hardcoded, mantendo a assinatura pública `(addonKey: string): boolean` para não quebrar call-sites.

## Especificação

### O que fazer:
Reescrever `useAddonAccess` para:
1. Chamar `useProdutos()` internamente
2. Filtrar o produto pela `chave` fornecida
3. Retornar `true` apenas se `produto.status_usuario === 'ativo'`
4. Manter a assinatura pública: `useAddonAccess(addonKey: string): boolean`

```typescript
import { useProdutos } from './useProdutos';

export function useAddonAccess(addonKey: string): boolean {
  const { data: produtos } = useProdutos();

  if (!produtos) return false;

  const produto = produtos.find(p => p.chave === addonKey);
  return produto?.status_usuario === 'ativo';
}
```

Verificar todos os call-sites de `useAddonAccess` para garantir que a assinatura permanece compatível.

### Regras de negócio aplicáveis:
- Retornar `false` durante loading (dados não disponíveis) — evitar acesso prematuramente liberado
- A assinatura `(addonKey: string): boolean` não pode mudar — há call-sites existentes
- Após refatoração, `useAddonAccess` é um wrapper fino sobre `useProdutos`

### Critérios de aceitação:
- [ ] `useAddonAccess('elections')` retorna `true` quando `status_usuario === 'ativo'`
- [ ] `useAddonAccess('elections')` retorna `false` para status cancelado/expirado/null
- [ ] Todos os call-sites existentes continuam funcionando sem alteração
- [ ] Arquivo compila sem erros TypeScript
- [ ] Sem regressão nas telas que usam o hook
