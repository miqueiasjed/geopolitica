# Task 16.2 – Backend: Endpoint GET /api/admin/configuracoes/defaults

## Objetivo
Criar o endpoint `GET /api/admin/configuracoes/defaults` que retorna os textos padrão de todos os 6 prompts lendo diretamente de `config/ai.php`, nunca do banco de dados.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Api/Admin/AdminConfiguracaoController.php` → modificar
- `backend/routes/api.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/Api/Admin/AdminConfiguracaoController.php` → ver estrutura atual e middlewares aplicados
- `backend/routes/api.php` → ver bloco de rotas admin para adicionar a nova rota no lugar correto
- `backend/config/ai.php` → confirmar as chaves dos prompts padrão (`ai.prompts.*`)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

**No `AdminConfiguracaoController`**, adicionar o método `defaults()`:

```php
public function defaults(): JsonResponse
{
    $chaves = [
        'prompt_analise_sistema'     => 'analise_sistema',
        'prompt_chat_sistema'        => 'chat_sistema',
        'prompt_detector_sistema'    => 'detector_sistema',
        'prompt_convergencia_sistema'=> 'convergencia_sistema',
        'prompt_perfil_contexto'     => 'perfil_contexto',
        'prompt_perfil_lideranca'    => 'perfil_lideranca',
    ];

    $defaults = [];
    foreach ($chaves as $chaveConfig => $chaveAi) {
        $defaults[$chaveConfig] = config("ai.prompts.{$chaveAi}");
    }

    return response()->json(['data' => $defaults]);
}
```

**No `api.php`**, adicionar a rota dentro do bloco admin existente (junto com as outras rotas de configuração):
```php
Route::get('/configuracoes/defaults', [AdminConfiguracaoController::class, 'defaults']);
```

### Regras de negócio aplicáveis:
- Lê **somente** de `config/ai.php` — nunca do banco de dados
- Protegido pelo mesmo middleware `auth:sanctum` + `role:admin` já aplicado ao grupo admin
- Não expõe dados sensíveis (prompts não são sensíveis)
- Resposta segue o padrão `{ "data": { ... } }` do projeto

### Critérios de aceitação:
- [ ] `GET /api/admin/configuracoes/defaults` retorna HTTP 200 com os textos padrão dos 6 prompts
- [ ] Rota bloqueada para usuários não-admin (retorna 401/403)
- [ ] Resposta não contém valores do banco — mesmo que um prompt tenha sido customizado, retorna sempre o padrão do config
- [ ] Todas as 6 chaves de prompt estão presentes na resposta

## Teste esperado
- Verificação manual (Tinker ou curl): `GET /api/admin/configuracoes/defaults` com token admin retorna os textos padrão
- Verificação de segurança: sem token retorna 401; com token de usuário comum retorna 403
