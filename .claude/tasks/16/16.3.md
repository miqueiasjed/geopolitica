# Task 16.3 – Frontend: Tipos TypeScript e UI de metadados estruturados nos prompts

## Objetivo
Atualizar o tipo `Configuracao` para incluir os novos campos de metadados, e renderizar essas informações na interface do admin: labels de tela/trigger/saída e badges para variáveis disponíveis.

## Tipo
frontend-componente

## Arquivos a criar/modificar
- `frontend/src/types/configuracao.ts` → modificar
- `frontend/src/pages/admin/AdminConfiguracoes.tsx` → modificar (apenas seção de renderização do componente `Campo`)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/types/configuracao.ts` → ver a interface `Configuracao` atual para adicionar os novos campos
- `frontend/src/pages/admin/AdminConfiguracoes.tsx` → ver o componente `Campo` (linhas ~48-144) para entender onde e como inserir a exibição dos metadados

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:

**Em `configuracao.ts`**, adicionar os campos opcionais à interface `Configuracao`:
```typescript
interface Configuracao {
  // ... campos existentes ...
  tela?: string | null
  trigger?: string | null
  saida?: string | null
  variaveis?: string[]
}
```

**Em `AdminConfiguracoes.tsx`**, no componente `Campo`, quando `tipo === 'textarea'` e o campo pertencer ao grupo `prompts`, renderizar um bloco de metadados logo **abaixo da `descricao`** e **acima do textarea**:

```
┌─────────────────────────────────────────────┐
│ [Label]                                      │
│ [Descrição existente]                        │
│                                              │
│  Tela: Feed de Tensões  Trigger: Cron        │  ← 3 pills de info
│  Saída: JSON array                           │
│                                              │
│  Variáveis:  [{{pais}}]                      │  ← badges (apenas se variaveis.length > 0)
│                                              │
│ ┌─────────────────────────────────────────┐ │
│ │ (textarea)                               │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**Estilo dos pills de metadados:**
- Container: `flex flex-wrap gap-2 mb-3 text-xs`
- Cada pill: `inline-flex items-center gap-1 px-2 py-1 rounded bg-slate-800 text-slate-400`
- Label do pill (ex: "Tela:"): `text-slate-500`
- Valor do pill: `text-slate-300`

**Badges de variáveis:**
- Visíveis apenas quando `variaveis && variaveis.length > 0`
- Label "Variáveis:" no mesmo estilo pill
- Cada variável: `inline-flex px-2 py-0.5 rounded font-mono text-xs bg-blue-950 text-blue-300 border border-blue-800`

**Condição de exibição:** só mostrar o bloco se pelo menos um dos campos (`tela`, `trigger`, `saida`) estiver presente. Campos null/undefined são simplesmente omitidos do bloco.

### Regras de negócio aplicáveis:
- Apenas prompts (grupo `prompts`, tipo `textarea`) exibem esses metadados
- A interface de outros grupos (`ia`, `email`, etc.) não é afetada
- Nenhuma chamada de API adicional — os dados vêm do endpoint existente `GET /api/admin/configuracoes`

### Critérios de aceitação:
- [ ] Tipo `Configuracao` em `configuracao.ts` inclui `tela`, `trigger`, `saida` e `variaveis` como opcionais
- [ ] Os 6 prompts exibem pills de tela/trigger/saída na interface
- [ ] `prompt_perfil_contexto` e `prompt_perfil_lideranca` exibem badge `{{pais}}`
- [ ] Os outros 4 prompts não exibem seção de variáveis
- [ ] Campos ausentes/null no metadado não geram pills vazios

## Teste esperado

- `prompt_perfil_contexto` deve exibir o badge `{{pais}}`
