# Task 16.1 – Backend: Enriquecer metadados dos 6 prompts no ConfiguracaoService

## Objetivo
Adicionar campos estruturados (`tela`, `trigger`, `saida`, `variaveis`) às definições de cada prompt no `ConfiguracaoService`, e garantir que esses campos extras sejam incluídos na resposta do endpoint `GET /api/admin/configuracoes`.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `backend/app/Services/ConfiguracaoService.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Services/ConfiguracaoService.php` → ver estrutura completa de `definicoes()` e `todos()` para entender como os dados chegam ao controller

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

No método `definicoes()` do `ConfiguracaoService`, adicionar 4 novos campos a cada um dos 6 prompts:

```php
'tela'      => string,   // nome da tela do usuário onde o resultado aparece
'trigger'   => string,   // quando o prompt é chamado
'saida'     => string,   // formato de saída obrigatório
'variaveis' => string[], // variáveis disponíveis para interpolação, ex: ['{{pais}}']
```

**Mapeamento dos 6 prompts:**

| Chave | tela | trigger | saida | variaveis |
|-------|------|---------|-------|-----------|
| `prompt_analise_sistema` | Timeline / Feed de Tensões | Cron de ingestão de feed | JSON array | [] |
| `prompt_chat_sistema` | Chat com os Briefings | Ação do usuário (pergunta) | Texto livre (streaming) | [] |
| `prompt_detector_sistema` | Alertas Preditivos (background) | Cron de detecção de sinais | JSON array | [] |
| `prompt_convergencia_sistema` | Alertas Preditivos (card de alerta) | Cron de convergência | JSON `{titulo, analise}` | [] |
| `prompt_perfil_contexto` | Perfil de País (aba Contexto) | Ação admin (gerar perfil) | Texto livre | ['{{pais}}'] |
| `prompt_perfil_lideranca` | Perfil de País (aba Liderança) | Ação admin (gerar perfil) | Texto livre | ['{{pais}}'] |

No método `todos()`, garantir que os campos extras (`tela`, `trigger`, `saida`, `variaveis`) sejam incluídos no array retornado para cada configuração do grupo `prompts`. Os demais grupos não recebem esses campos (podem ser null/ausentes).

**Importante:** esses campos são metadados estáticos — não são armazenados no banco, não modificam o model `Configuracao`. Eles existem apenas na camada de serviço e na resposta da API.

### Regras de negócio aplicáveis:
- Metadados são somente leitura — nunca persistidos no banco
- O campo `variaveis` deve ser um array vazio `[]` para prompts sem variáveis (não null)
- Nenhuma outra funcionalidade do sistema é afetada (logging, chat, análises continuam iguais)

### Critérios de aceitação:
- [ ] Os 6 prompts no método `definicoes()` têm os campos `tela`, `trigger`, `saida`, `variaveis`
- [ ] O endpoint `GET /api/admin/configuracoes` inclui esses 4 campos na resposta para cada prompt
- [ ] Prompts com variáveis retornam o array correto (ex: `['{{pais}}']`)
- [ ] Grupos não-prompt (`ia`, `email`, etc.) não são afetados na resposta

## Teste esperado
- Verificação manual: chamar `GET /api/admin/configuracoes` e confirmar que a resposta do grupo `prompts` inclui os campos `tela`, `trigger`, `saida` e `variaveis` para cada item
- `php artisan tinker` → `app(App\Services\ConfiguracaoService::class)->todos()` e inspecionar chaves de `prompts`
