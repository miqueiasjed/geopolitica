# Task 16.4 – Frontend: Botão "↺ Restaurar padrão" com fetch e feedback visual

## Objetivo
Adicionar o botão "↺ Restaurar padrão" a cada campo textarea de prompt: ao clicar, busca o texto padrão via API, preenche o textarea localmente (sem salvar), e exibe feedback visual (borda âmbar + tooltip) indicando que o campo foi restaurado mas não salvo.

## Especificação

### O que fazer:

**1. Query para buscar defaults**

No componente `AdminConfiguracoes` (componente principal), adicionar uma query TanStack Query para buscar o endpoint de defaults **uma única vez** (não por prompt):

```typescript
const { data: defaults } = useQuery({
  queryKey: ['configuracoes-defaults'],
  queryFn: () => api.get('/admin/configuracoes/defaults').then(r => r.data.data),
  staleTime: Infinity, // defaults nunca mudam em runtime
})
```

O objeto `defaults` tem formato `{ prompt_analise_sistema: "...", prompt_chat_sistema: "...", ... }`.

**2. Passar o default para o componente Campo**

Passar `defaultValor?: string` como prop para o componente `Campo`. Para prompts, passar `defaults?.[campo.chave]`.

**3. Estado de "restaurado" no Campo**

No componente `Campo`, adicionar estado local `restaurado: boolean` (inicia como `false`). Quando o usuário clica no botão de restaurar, seta `restaurado = true`. Quando edita o textarea manualmente ou quando o valor é salvo com sucesso, resetar para `false`.

**4. Botão "↺ Restaurar padrão"**

Adicionar o botão no canto superior direito do label do campo (ao lado do label), visível apenas quando:
- `tipo === 'textarea'`
- `campo.configurado === true` (foi personalizado no banco)
- `defaults` foi carregado com sucesso

Ao clicar no botão:
- Chama o callback `onChange` passando o valor de `defaultValor`
- Seta `restaurado = true`

**Estilo do botão:**
```
className="text-xs text-slate-500 hover:text-amber-400 flex items-center gap-1 transition-colors"
```
Texto: `↺ Restaurar padrão`

**5. Feedback visual quando restaurado**

Quando `restaurado === true`:
- O textarea ganha classe de borda âmbar: `ring-1 ring-amber-500/50 border-amber-500/50`
- Abaixo do textarea, mostrar tooltip inline: `text-xs text-amber-400 mt-1 → "↑ Restaurado. Clique em Salvar para confirmar."`

**6. Salvar null ao confirmar**

Quando o admin salva após restaurar, o valor enviado deve ser o texto do default (não null). O sistema de "restaurar para null" funciona se o valor enviado for **idêntico ao padrão** — neste caso o backend pode salvar null automaticamente.

**Alternativa mais simples (preferida):** ao clicar em "Restaurar padrão", colocar o valor como string vazia `""` em vez do texto default. Assim, ao salvar, o backend recebe `""` e salva null no banco (verificar como o `atualizar()` do service trata valores vazios). Se o service já trata `""` como null → usar string vazia. Caso contrário, usar o texto do default e documentar que "restaurar" significa "salvar o texto padrão no banco", sendo equivalente funcionalmente.

> **Instrução para o executor:** ler o método `atualizar()` do `ConfiguracaoService` para determinar como valores vazios são tratados antes de decidir entre as duas abordagens. Usar a mais simples que satisfaça o comportamento descrito.

### Regras de negócio aplicáveis:
- O botão só aparece quando o prompt foi personalizado (`configurado === true`)
- Clicar em "Restaurar" NÃO salva automaticamente — apenas preenche o campo
- O feedback âmbar indica "campo alterado mas não salvo"
- O feedback âmbar desaparece quando o usuário edita manualmente o textarea

### Critérios de aceitação:
- [ ] Botão "↺ Restaurar padrão" aparece apenas para prompts customizados
- [ ] Ao clicar, o textarea é preenchido com o texto padrão
- [ ] Borda âmbar aparece no textarea restaurado
- [ ] Tooltip "Restaurado. Clique em Salvar para confirmar." aparece abaixo do textarea
- [ ] Após salvar (clique no botão Salvar do grupo), o feedback âmbar desaparece
- [ ] Se o usuário editar o textarea após restaurar, o feedback âmbar desaparece
- [ ] Prompts não-customizados não exibem o botão

## Teste esperado
- Verificação visual: customizar um prompt, salvar, e confirmar que o botão "↺ Restaurar padrão" aparece
- Clicar no botão: textarea deve receber o texto original, borda âmbar deve aparecer
- Salvar: borda âmbar deve desaparecer; o prompt deve voltar ao comportamento padrão
