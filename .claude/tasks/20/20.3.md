# Task 20.3 – LastlinkHandlerService: handleCancelamento + AddonBoasVindasMail

## Objetivo
Completar o `LastlinkHandlerService` com o handler de cancelamento/reembolso e criar o Mailable `AddonBoasVindasMail` com dois fluxos (conta nova e conta existente).

## Tipo
backend-logica

- `backend/app/Mail/AddonBoasVindasMail.php` → criar
- `backend/resources/views/emails/addon-boas-vindas.blade.php` → criar

## Contexto necessário (arquivos para ler antes)

## Especificação

### O que fazer:

**Adicionar ao `LastlinkHandlerService`:**

Método privado `handleCancelamento(array $payload, string $motivo): void`:
1. Extrair email do payload
2. Buscar User pelo email — se não encontrar, retornar sem erro
3. Extrair product_id e resolver addon_key — se null, retornar
4. Chamar `AddonService::cancelar($user->id, $addonKey, $motivo)`

Conectar ao `handle()`: já mapeado nos eventos `CANCELLED`, `REFUNDED`, `CHARGEBACK`.

**`AddonBoasVindasMail`:**
- Construtor: `string $nome, string $addonKey, string $linkAcesso, bool $contaNova`
- `build()`: usa view `emails.addon-boas-vindas`
- Assunto: `"[GPI] Monitor {$addonLabel} ativado"`

**Template `addon-boas-vindas.blade.php`:**
Manter o estilo visual dos e-mails existentes (fundo escuro #0a0a0b, dourado #C9B882). Conteúdo:
- Título: `{Monitor Eleitoral|Monitor de Guerra} ativado.`
- Texto conta nova: `"Sua conta foi criada. Clique abaixo para definir sua senha e acessar o dashboard."`
- Texto conta existente: `"O {addon} foi adicionado à sua conta. Acesse o dashboard para começar."`
- Botão CTA com `$linkAcesso`
- Labels por addon_key: `elections → Monitor Eleitoral`, `war → Monitor de Guerra`

### Regras de negócio aplicáveis:
- Link para conta nova: URL de redefinição de senha (gerado no handler)
- Link para conta existente: URL base do dashboard (`config('app.url') . '/dashboard'`)
- Motivos de cancelamento aceitos: `cancelado`, `reembolsado`

### Critérios de aceitação:
- [ ] `handleCancelamento` remove addon via AddonService
- [ ] `AddonBoasVindasMail` renderiza sem erros para conta nova e existente
- [ ] Label correto exibido conforme addon_key
- [ ] Template mantém estilo visual dos e-mails existentes
