## Especificação

### O que fazer:

**`LastlinkHandlerService`:**

Método `registrarEvento(array $payload): WebhookEvento`:
- Extrair `event_type` do payload Lastlink (caminho: `event`, `status`, ou `type`)
- Extrair `email` do payload (caminho: `customer.email`)
- Criar registro em `webhook_eventos` com `event_type` prefixado por `LASTLINK_` e `email`
- O campo `hotmart_subscriber_code` fica null (não se aplica ao Lastlink)

Método `handle(WebhookEvento $evento): void`:
- Match no `event_type` (normalizado para uppercase):
  - `LASTLINK_APPROVED`, `LASTLINK_COMPLETE`, `LASTLINK_PAID` → `handleCompra($evento->payload)`
  - `LASTLINK_CANCELLED`, `LASTLINK_REFUNDED`, `LASTLINK_CHARGEBACK` → na Task 20.3
  - `default` → null
- Marcar evento como `processado = true, processado_em = now()` em caso de sucesso
- Em `\Throwable`: salvar `erro` no evento (não relançar)

Método privado `handleCompra(array $payload): void`:
1. Extrair email e nome do comprador
2. Extrair `product_id` e `order_id` do payload Lastlink (caminho: `product.id`, `order.id`)
3. Resolver addon_key: `AddonService::resolverAddonKey($productId, 'lastlink')` — se null, retornar sem erro
4. Localizar ou criar User:
   - `User::firstOrCreate(['email' => $email], ['name' => $nome, 'password' => Str::password(24)])`
5. Se User foi criado (`wasRecentlyCreated`):
   - Criar `Assinante` com `plano = 'essencial', ativo = true, status = 'ativo'`
   - Gerar token de redefinição de senha e enviar `AddonBoasVindasMail` com `conta_nova = true`
6. Se User já existia:
   - Enviar `AddonBoasVindasMail` com `conta_nova = false`
7. Chamar `AddonService::ativar($user->id, $addonKey, 'lastlink', $orderId, $productId)`

### Regras de negócio aplicáveis:
- Conta criada automaticamente tem plano Essencial
- `AddonBoasVindasMail` para conta nova inclui link de redefinição de senha
- `AddonBoasVindasMail` para conta existente inclui link direto para `/dashboard`

### Critérios de aceitação:
- [ ] Evento registrado em `webhook_eventos` com prefixo `LASTLINK_`
- [ ] Comprador sem conta GPI → User + Assinante Essencial criados
- [ ] AddonService::ativar chamado com fonte `lastlink`
- [ ] E-mail enviado nos dois casos (conta nova / existente)
- [ ] product_id não reconhecido → sem erro, sem ação
