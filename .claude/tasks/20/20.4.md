# Task 20.4 – Testes de feature: Webhook Lastlink

## Objetivo
Escrever testes de feature cobrindo os quatro cenários principais do webhook Lastlink: compra com conta nova, compra com conta existente, cancelamento e token inválido.

## Tipo
teste

## Arquivos a criar/modificar
- `backend/tests/Feature/WebhookLastlinkTest.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/tests/Feature/WebhookHotmartTest.php` → padrão de testes de webhook existente
- `backend/app/Services/LastlinkHandlerService.php` → após Tasks 20.2 e 20.3
- `backend/app/Http/Controllers/WebhookLastlinkController.php` → após Task 20.1

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Cenários a cobrir:**

`test_compra_aprovada_cria_conta_essencial_e_ativa_addon`:
- POST `/api/webhook/lastlink` com token válido e payload de compra
- Email do comprador não existe no banco
- Verificar: User criado, Assinante com `plano = essencial`, `addons` contém addon_key, registro em `assinante_addons` com status `ativo`
- `Mail::assertSent(AddonBoasVindasMail::class)`

`test_compra_aprovada_ativa_addon_em_conta_existente`:
- Comprador já tem User e Assinante no banco
- Verificar: `addons` atualizado, `assinante_addons` criado, plano NÃO alterado

`test_cancelamento_remove_addon`:
- POST com evento de cancelamento
- Verificar: addon_key removido de `assinantes.addons`, registro em `assinante_addons` com status `cancelado`

`test_token_invalido_registra_evento_mas_nao_processa`:
- Header `x-lastlink-token` errado
- Verificar: evento registrado em `webhook_eventos`, `processado = false`, resposta HTTP 200

`test_produto_desconhecido_nao_causa_erro`:
- product_id não mapeado em config
- Verificar: evento registrado, sem exceção, resposta 200

### Regras de negócio aplicáveis:
- Usar `Mail::fake()` para não enviar e-mails reais
- Mockar `config('services.lastlink.webhook_token')` com valor de teste
- Mockar `config('addons.lastlink_products')` com mapeamento de teste

### Critérios de aceitação:
- [ ] Todos os 5 cenários passando
- [ ] Nenhum e-mail real enviado
- [ ] Nenhuma chamada externa feita
- [ ] Regressão: testes Hotmart existentes continuam passando

## Teste esperado
- `php artisan test --filter=WebhookLastlinkTest`
- `php artisan test --filter=WebhookHotmartTest` (regressão)
