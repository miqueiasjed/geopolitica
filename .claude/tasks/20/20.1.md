# Task 20.1 – WebhookLastlinkController, FormRequest e rota

## Objetivo
Criar o controller do webhook Lastlink com validação de token, registro do evento e delegação ao handler service; e registrar a rota `POST /api/webhook/lastlink`.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/WebhookLastlinkController.php` → criar
- `backend/app/Http/Requests/ReceberWebhookLastlinkRequest.php` → criar
- `backend/routes/api.php` → modificar (adicionar rota)

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/WebhookHotmartController.php` → padrão exato a seguir
- `backend/app/Http/Requests/ReceberWebhookHotmartRequest.php` → padrão de FormRequest
- `backend/routes/api.php` → onde inserir a nova rota

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**`ReceberWebhookLastlinkRequest`:**
- `authorize()`: retorna `true`
- `rules()`: sem regras obrigatórias (payload Lastlink é variável — aceitar qualquer JSON)

**`WebhookLastlinkController`:**
- Injetar `LastlinkHandlerService` via construtor
- Método `receber(ReceberWebhookLastlinkRequest $request)`:
  1. Mesclar `$request->validated() + $request->all()` em `$payload`
  2. Chamar `$this->lastlinkHandlerService->registrarEvento($payload)` → retorna `WebhookEvento`
  3. Validar token: `$request->header('x-lastlink-token') !== config('services.lastlink.webhook_token')`
     - Se inválido: salvar erro no evento e retornar `response()->json(['received' => true])` (sempre 200)
  4. Chamar `$this->lastlinkHandlerService->handle($evento)`
  5. Retornar `response()->json(['received' => true])`

**`routes/api.php`:**
- Adicionar `Route::post('/webhook/lastlink', [WebhookLastlinkController::class, 'receber'])` sem middleware de auth

**`.env.example`:**
- Adicionar as variáveis:
  ```
  LASTLINK_WEBHOOK_SECRET=
  LASTLINK_PRODUCT_ELECTIONS=
  LASTLINK_PRODUCT_WAR=
  HOTMART_PRODUCT_ELECTIONS=
  HOTMART_PRODUCT_WAR=
  ```

### Regras de negócio aplicáveis:
- Webhook deve sempre retornar HTTP 200 — o Lastlink não deve receber erros 4xx/5xx
- Token inválido loga o erro no evento mas não retorna 401

### Critérios de aceitação:
- [ ] `POST /api/webhook/lastlink` acessível sem autenticação
- [ ] Token inválido registra erro no evento mas retorna 200
- [ ] Evento é registrado em `webhook_eventos` antes da validação do token
- [ ] Variáveis de ambiente no `.env.example`
