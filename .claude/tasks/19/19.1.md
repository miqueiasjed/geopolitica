# Task 19.1 – Migrations: addons em assinantes, tabela assinante_addons e vertical_conteudo em conteudos

## Objetivo
Criar as três migrations que suportam o sistema de add-ons: coluna `addons` na tabela `assinantes`, tabela `assinante_addons` para histórico, e coluna `vertical_conteudo` na tabela `conteudos`.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/database/migrations/2026_04_27_000001_add_addons_to_assinantes_table.php` → criar
- `backend/database/migrations/2026_04_27_000002_create_assinante_addons_table.php` → criar
- `backend/database/migrations/2026_04_27_000003_add_vertical_conteudo_to_conteudos_table.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/database/migrations/2026_04_14_160000_create_assinantes_table.php` → ver estrutura atual da tabela assinantes
- `backend/database/migrations/2026_04_14_190000_create_conteudos_table.php` → ver estrutura atual da tabela conteudos

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Migration 1 — `add_addons_to_assinantes_table`:**
- Adicionar coluna `addons` do tipo `json` com default `[]` após a coluna `status`
- O array armazena addon_keys ativos: ex `["elections"]`, `["war"]`, `["elections","war"]`

**Migration 2 — `create_assinante_addons_table`:**
- Tabela de histórico com os campos:
  - `id` bigIncrements PK
  - `user_id` foreignId → users, cascadeOnDelete
  - `addon_key` string — valores: `elections`, `war`
  - `status` string — valores: `ativo`, `cancelado`, `expirado`, `reembolsado`
  - `fonte` string — valores: `hotmart`, `lastlink` (plataforma de origem)
  - `order_id` string nullable — ID do pedido na plataforma de origem
  - `product_id` string nullable — ID do produto na plataforma de origem
  - `iniciado_em` timestamp default now()
  - `expira_em` timestamp nullable
  - `timestamps()`
- Índices: `[user_id, status]`, `[addon_key, status]`

**Migration 3 — `add_vertical_conteudo_to_conteudos_table`:**
- Adicionar coluna `vertical_conteudo` string nullable após `plano_minimo`
- Valores possíveis: `null` (GPI Core), `elections`, `war`
- Índice simples na coluna

### Regras de negócio aplicáveis:
- `addons` em `assinantes` reflete o estado atual (add-ons ativos agora)
- `assinante_addons` é o log imutável de todas as operações (compra, cancelamento, reembolso)
- `vertical_conteudo` em `conteudos` é nullable — NULL significa conteúdo do GPI Core

### Critérios de aceitação:
- [ ] Coluna `addons` JSON existe em `assinantes` com default `[]`
- [ ] Tabela `assinante_addons` criada com todos os campos e índices
- [ ] Coluna `vertical_conteudo` existe em `conteudos` com índice
- [ ] `php artisan migrate` roda sem erros
- [ ] `php artisan migrate:rollback` desfaz tudo corretamente

## Teste esperado
