# Task 19.2 – Model AssinanteAddon e atualização do Model Assinante

## Objetivo
Criar o model `AssinanteAddon` e atualizar o model `Assinante` para incluir o cast de `addons` como array e o relacionamento com add-ons.

## Tipo
backend-estrutura

## Arquivos a criar/modificar
- `backend/app/Models/AssinanteAddon.php` → criar
- `backend/app/Models/Assinante.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/Assinante.php` → ver estrutura atual (fillable, casts, relacionamentos)

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Model `AssinanteAddon`:**
- `$fillable`: `user_id`, `addon_key`, `status`, `fonte`, `order_id`, `product_id`, `iniciado_em`, `expira_em`
- Relacionamento `belongsTo(User::class)`
- Scope `ativo()`: filtra por `status = 'ativo'`
- Constantes de classe: `ADDON_KEYS = ['elections', 'war']`, `FONTES = ['hotmart', 'lastlink']`

**Atualizar `Assinante`:**
- Adicionar `addons` ao `$fillable`
- Adicionar cast `'addons' => 'array'` para serialização JSON automática
- Adicionar relacionamento `hasMany(AssinanteAddon::class, 'user_id', 'user_id')`
- Adicionar método helper `temAddon(string $addonKey): bool` — verifica se `$addonKey` está no array `addons`
- Adicionar método helper `temAcessoVertical(string $addonKey): bool` — combina plano + addons:
  - `elections`: retorna true se `plano` for `pro` ou `reservado`, OU se `temAddon('elections')`
  - `war`: retorna true se `plano` for `reservado`, OU se `temAddon('war')`

### Regras de negócio aplicáveis:
- Plano Pro inclui `elections` sem precisar do add-on
- Plano Reservado inclui `elections` e `war` sem precisar dos add-ons
- `temAcessoVertical` é a fonte de verdade para verificação de acesso

### Critérios de aceitação:
- [ ] `AssinanteAddon` criado com fillable, relacionamentos e scope corretos
- [ ] `Assinante` tem cast `addons => array`
- [ ] `Assinante::temAddon('elections')` retorna bool corretamente
- [ ] `Assinante::temAcessoVertical('elections')` respeita regra do plano Pro
- [ ] `Assinante::temAcessoVertical('war')` respeita regra do plano Reservado

## Teste esperado

- Lógica de `temAcessoVertical` coberta na Task 19.5 (testes)
