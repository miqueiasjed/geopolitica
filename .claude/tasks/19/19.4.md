# Task 19.4 – Atualizar HotmartHandlerService para detectar add-ons e Gate de acesso a verticais

## Objetivo
Atualizar o `HotmartHandlerService` para identificar se um produto comprado é um add-on (em vez de plano) e delegar ao `AddonService`; e registrar o Gate Laravel `acessar-vertical` para controle de acesso nas rotas.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `backend/app/Services/HotmartHandlerService.php` → modificar
- `backend/app/Providers/AppServiceProvider.php` → modificar (registrar Gate)

## Contexto necessário (arquivos para ler antes)
- `backend/app/Services/HotmartHandlerService.php` → ver lógica atual de `ativarAssinatura`, `normalizarPlano`, `extrairEmail`
- `backend/app/Services/AddonService.php` → após Task 19.3
- `backend/app/Providers/AppServiceProvider.php` → ver onde registrar o Gate

## Skills necessárias
- laravel-arquitetura
- permissoes-e-sync

## Especificação

### O que fazer:

**Atualizar `HotmartHandlerService`:**

Adicionar método privado `resolverTipoCompra(array $payload): array`:
- Extrai o `product_id` do payload Hotmart (caminho: `purchase.product.id` ou `product.id`)
- Chama `AddonService::resolverAddonKey($productId, 'hotmart')`
- Retorna `['tipo' => 'addon', 'addon_key' => $key]` se reconhecer um add-on
- Retorna `['tipo' => 'plano']` caso contrário

Atualizar `handlePurchaseApproved` e `handlePurchaseComplete`:
- Chamar `resolverTipoCompra($payload)` antes da lógica atual
- Se `tipo = 'addon'`: chamar `AddonService::ativar(userId, addonKey, 'hotmart', orderId, productId)` em vez de `ativarAssinatura`
- Se `tipo = 'plano'`: manter comportamento atual (chamar `ativarAssinatura`)

Atualizar `handlePurchaseCanceled`, `handlePurchaseRefunded`, `handlePurchaseChargeback`:
- Mesma lógica: se o produto for um add-on, chamar `AddonService::cancelar(userId, addonKey, motivo)`
- Se for plano, manter `desativarAssinatura`

**Registrar Gate `acessar-vertical` em `AppServiceProvider`:**
```php
Gate::define('acessar-vertical', function (User $user, string $addonKey): bool {
    $assinante = $user->assinante;
    return $assinante?->temAcessoVertical($addonKey) ?? false;
});
```

### Regras de negócio aplicáveis:
- Se o product_id não for reconhecido como add-on, tratar como plano (comportamento atual preservado)
- O Gate é usado nos controllers das verticais para retornar 403 se não autorizado

### Critérios de aceitação:
- [ ] Payload com product_id de add-on aciona `AddonService::ativar` (não `ativarAssinatura`)
- [ ] Payload com product_id de plano mantém comportamento atual inalterado
- [ ] Gate `acessar-vertical` registrado e retorna true/false conforme `temAcessoVertical`
- [ ] Testes existentes do Hotmart continuam passando
