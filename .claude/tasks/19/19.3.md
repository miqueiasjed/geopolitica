# Task 19.3 – Config de produtos add-on e AddonService (ativar/cancelar)

## Objetivo
Criar o arquivo de configuração que mapeia IDs de produto (Hotmart e Lastlink) para addon_keys, e implementar o `AddonService` centralizado com os métodos `ativar` e `cancelar`.

- `backend/app/Services/AddonService.php` → criar

## Especificação

### O que fazer:

**`config/addons.php`:**
```php
return [
    'hotmart_products' => [
        env('HOTMART_PRODUCT_ELECTIONS') => 'elections',
        env('HOTMART_PRODUCT_WAR')       => 'war',
    ],
    'lastlink_products' => [
        env('LASTLINK_PRODUCT_ELECTIONS') => 'elections',
        env('LASTLINK_PRODUCT_WAR')       => 'war',
    ],
];
```

**`AddonService`:**

Método `ativar(int $userId, string $addonKey, string $fonte, ?string $orderId = null, ?string $productId = null): void`:
1. Busca `Assinante` pelo `user_id`
2. Atualiza o array `addons`: adiciona `$addonKey` se ainda não estiver presente
3. Salva o `Assinante`
4. Cria registro em `AssinanteAddon` com status `ativo`, fonte, order_id, product_id, iniciado_em = now()

Método `cancelar(int $userId, string $addonKey, string $motivo): void`:
- Valores aceitos para `$motivo`: `cancelado`, `reembolsado`, `expirado`
1. Busca `Assinante` pelo `user_id`
2. Remove `$addonKey` do array `addons`
3. Salva o `Assinante`
4. Atualiza o registro ativo em `AssinanteAddon` (onde `user_id + addon_key + status = ativo`) para o status = `$motivo`

Método estático `resolverAddonKey(string $productId, string $fonte): ?string`:
- Consulta `config("addons.{$fonte}_products")[$productId]`
- Retorna o addon_key ou null se não reconhecido

### Regras de negócio aplicáveis:
- `ativar` é idempotente: adicionar addon_key que já existe no array não duplica
- `cancelar` não lança exceção se o add-on não estiver ativo — simplesmente não há nada a fazer
- O registro em `assinante_addons` é sempre criado (histórico imutável)

### Critérios de aceitação:
- [ ] `config('addons.hotmart_products')` mapeia corretamente via env vars
- [ ] `AddonService::ativar()` atualiza array `addons` e cria `AssinanteAddon`
- [ ] `AddonService::ativar()` é idempotente (não duplica addon_key)
- [ ] `AddonService::cancelar()` remove do array e atualiza status em `assinante_addons`
- [ ] `AddonService::resolverAddonKey()` retorna null para product_id desconhecido
