# Task 17.5 – Integrar PromptTestPanel na página de configurações de IA

## Objetivo
Inserir o componente `PromptTestPanel` abaixo de cada textarea de prompt na página de configurações de IA existente, passando o valor atual do textarea e a flag `exigeJson` correta para cada prompt.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/admin/configuracoes/` → modificar (página de configurações de IA existente, identificar arquivo exato ao ler)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/admin/configuracoes/` → ler o arquivo da página de configurações de IA para identificar onde cada textarea de prompt está renderizada
- `frontend/src/components/admin/PromptTestPanel.tsx` → interface do componente (Task 17.4)

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:
1. Localizar na página existente onde os 6 textareas de prompt são renderizados.
2. Abaixo de **cada** textarea de prompt, adicionar `<PromptTestPanel>` passando:
   - `promptSistema={formValues.nomeDoPrompt}` — o valor atual do input controlado (não o valor salvo)
   - `exigeJson={true}` para: `analise_sistema`, `detector_sistema`, `convergencia_sistema`
   - `exigeJson={false}` para: `chat_sistema`, `perfil_contexto`, `perfil_lideranca`

3. O `promptSistema` deve refletir o texto atual do textarea (estado do formulário), não o valor persistido no banco — assim o admin pode testar uma variação antes de salvar.

### Mapeamento de prompts e exigeJson:
| Chave de configuração         | exigeJson |
|-------------------------------|-----------|
| `ai_prompt_analise_sistema`   | true      |
| `ai_prompt_detector_sistema`  | true      |
| `ai_prompt_convergencia_sistema` | true   |
| `ai_prompt_chat_sistema`      | false     |
| `ai_prompt_perfil_contexto`   | false     |
| `ai_prompt_perfil_lideranca`  | false     |

> Verificar os nomes exatos das chaves lendo a página existente.

### Regras de negócio aplicáveis:
- Não alterar o comportamento de salvar/carregar prompts — apenas adicionar o painel de teste.
- O `PromptTestPanel` é visualmente abaixo do textarea, dentro do mesmo card/seção do prompt.

### Critérios de aceitação:
- [ ] Cada um dos 6 textareas de prompt tem um `PromptTestPanel` logo abaixo.
- [ ] A flag `exigeJson` está correta para cada prompt.
- [ ] Alterar o texto no textarea e clicar "Executar" usa o texto **atual** (não o salvo).
- [ ] A página salva/carrega prompts normalmente (sem regressão).
- [ ] TypeScript compila sem erros, build passa.

## Teste esperado
- Abrir `/admin/ia/configuracoes` no browser.
- Confirmar que cada textarea tem "▶ Testar este prompt" abaixo dele.
- Editar um prompt localmente (sem salvar), executar o teste e confirmar que o texto editado foi usado.
- Salvar configurações e confirmar que o fluxo normal segue funcionando.
