# Task 17.1 – AiTestService: lógica de execução do teste de prompt

## Objetivo
Criar `AiTestService` que executa um prompt via provider ativo, mede duração em ms, extrai tokens usados e retorna um objeto estruturado de resultado — sem efeitos colaterais em produção.

## Tipo
backend-logica

## Arquivos a criar/modificar
- `app/Services/Ai/AiTestService.php` → criar

## Contexto necessário (arquivos para ler antes)
- `app/Contracts/AiProviderInterface.php` → entender a interface `complete()`
- `app/Services/Ai/AiProviderFactory.php` → ver como instanciar o provider ativo
- `app/Services/Ai/ClaudeProvider.php` → entender o que é retornado por `complete()`
- `app/Services/Ai/OpenAiProvider.php` → idem para OpenAI

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:
Criar `AiTestService` no namespace `App\Services\Ai` com um único método público:

```php
public function executar(string $promptSistema, string $mensagemUsuario, int $maxTokens): array
```

O método deve:
1. Capturar `microtime(true)` antes de chamar o provider.
2. Instanciar o provider via `AiProviderFactory::make()`.
3. Chamar `$provider->complete($promptSistema, [['role'=>'user','content'=>$mensagemUsuario]], $maxTokens)`.
4. Capturar `microtime(true)` após a chamada e calcular `$duracaoMs = round(($fim - $inicio) * 1000)`.
5. Retornar array com: `resposta` (string), `provider` (string, ex: 'claude' ou 'openai'), `modelo` (string), `duracao_ms` (int), `tokens_estimados_entrada` (int, estimativa: `strlen($promptSistema . $mensagemUsuario) / 4`), `tokens_estimados_saida` (int, estimativa: `strlen($resposta) / 4`).
6. Se `complete()` lançar exceção, relançar como `\RuntimeException` com a mensagem original.

O `$maxTokens` recebido pelo service já vem com o cap aplicado (isso é feito no Controller). O service não precisa aplicar o cap — apenas usar o valor recebido.

Para saber o nome do provider e modelo ativos, ler `config('ai.provider')` e (se claude) `config('claude.model')`, (se openai) `config('ai.openai.model')`.

### Regras de negócio aplicáveis:
- O service não salva nada no banco (o log é responsabilidade do Controller, e apenas se `AiLog` existir).
- Não usa streaming — usa apenas `complete()`.
- Tokens são estimativas (não há acesso ao count real via `complete()` atualmente).

### Critérios de aceitação:
- [ ] `AiTestService::executar()` retorna array com todos os 6 campos quando a chamada à IA tem sucesso.
- [ ] `duracao_ms` é um inteiro positivo > 0.
- [ ] `provider` e `modelo` refletem a configuração ativa.
- [ ] Exceção da IA é relançada como `\RuntimeException`.
- [ ] Não há escrita em banco de dados dentro do service.

## Teste esperado

- `php artisan tinker` → `app(App\Services\Ai\AiTestService::class)->executar('Você é assistente', 'Diga olá', 100)`
- Deve retornar array com `resposta`, `duracao_ms` etc.
