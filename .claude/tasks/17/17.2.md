# Task 17.2 – AdminAiTestController + FormRequest + Rota + Rate Limit

## Objetivo
Expor o endpoint `POST /api/admin/ai/testar-prompt` protegido por middleware admin e rate limit de 20 req/min, delegando a execução para `AiTestService`.

## Tipo
backend-endpoint

## Arquivos a criar/modificar
- `app/Http/Controllers/Api/Admin/AdminAiTestController.php` → criar
- `app/Http/Requests/Admin/TestarPromptRequest.php` → criar
- `routes/api.php` → modificar (adicionar rota)

## Contexto necessário (arquivos para ler antes)
- `app/Http/Controllers/Api/Admin/AdminConfiguracaoController.php` → ver padrão de controller admin (middleware, injeção de dependência)
- `app/Services/Ai/AiTestService.php` → interface do service (Task 17.1)
- `routes/api.php` → onde adicionar a rota (grupo admin existente)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

**TestarPromptRequest:**
```php
public function rules(): array {
    return [
        'prompt_sistema'   => 'required|string|max:10000',
        'mensagem_usuario' => 'required|string|max:2000',
        'max_tokens'       => 'nullable|integer|min:1|max:1024',
    ];
}
```

**AdminAiTestController:**
```php
public function testar(TestarPromptRequest $request, AiTestService $service): JsonResponse
{
    $maxTokens = min($request->input('max_tokens', 512), 1024); // cap no backend
    try {
        $resultado = $service->executar(
            $request->input('prompt_sistema'),
            $request->input('mensagem_usuario'),
            $maxTokens
        );
        return response()->json(['data' => $resultado]);
    } catch (\RuntimeException $e) {
        return response()->json(['message' => $e->getMessage()], 422);
    }
}
```

**Rota em api.php** (dentro do grupo admin existente):
```php
Route::post('ai/testar-prompt', [AdminAiTestController::class, 'testar'])
    ->middleware('throttle:20,1');
```

O middleware de autenticação e verificação de papel admin já deve existir no grupo — não criar um novo grupo, apenas adicionar a rota dentro do grupo admin existente.

### Regras de negócio aplicáveis:
- `max_tokens` é limitado a 1024 **no backend** independente do que chegar no request.
- Rate limit: 20 requisições por minuto por usuário admin (usar `throttle:20,1`).
- Middleware admin já existente no grupo de rotas — não duplicar.
- Erros da IA retornam HTTP 422 com `message`.

### Critérios de aceitação:
- [ ] `POST /api/admin/ai/testar-prompt` retorna 200 com `data.resposta`, `data.provider`, `data.modelo`, `data.duracao_ms`, `data.tokens_estimados_entrada`, `data.tokens_estimados_saida`.
- [ ] `max_tokens` acima de 1024 é silenciosamente limitado a 1024.
- [ ] Request sem `prompt_sistema` ou `mensagem_usuario` retorna 422 com erros de validação.
- [ ] Após 20 requisições em 1 minuto, retorna 429.
- [ ] Usuário sem permissão admin recebe 403.

## Teste esperado
```bash
curl -X POST http://localhost:8000/api/admin/ai/testar-prompt \
  -H "Authorization: Bearer {token_admin}" \
  -H "Content-Type: application/json" \
  -d '{"prompt_sistema":"Você é assistente","mensagem_usuario":"Diga olá","max_tokens":100}'
```
Deve retornar JSON com `data.resposta` não vazio.
