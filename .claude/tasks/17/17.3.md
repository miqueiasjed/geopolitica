# Task 17.3 – Tipos TypeScript + hook useTestarPrompt

## Objetivo
Definir os tipos TypeScript do resultado do teste de prompt e criar o hook `useTestarPrompt` com React Query mutation para chamar o endpoint backend.

- `frontend/src/hooks/useTestarPrompt.ts` → criar

## Especificação

### O que fazer:

**Tipos em `frontend/src/types/ai.ts`:**
```typescript
export interface TestarPromptPayload {
  prompt_sistema: string;
  mensagem_usuario: string;
  max_tokens?: number;
}

export interface TestarPromptResultado {
  resposta: string;
  provider: string;
  modelo: string;
  duracao_ms: number;
  tokens_estimados_entrada: number;
  tokens_estimados_saida: number;
}
```

**Hook `useTestarPrompt`:**
```typescript
import { useMutation } from '@tanstack/react-query';
import { api } from '@/lib/api'; // ajustar conforme o projeto
import { TestarPromptPayload, TestarPromptResultado } from '@/types/ai';

export function useTestarPrompt() {
  return useMutation<TestarPromptResultado, Error, TestarPromptPayload>({
    mutationFn: async (payload) => {
      const response = await api.post<{ data: TestarPromptResultado }>(
        '/admin/ai/testar-prompt',
        payload
      );
      return response.data.data;
    },
  });
}
```

O hook deve usar o cliente HTTP já configurado no projeto (verificar `src/lib/api.ts` ou equivalente para o padrão exato de chamada POST).

### Regras de negócio aplicáveis:
- `max_tokens` é opcional no payload (o backend usa 512 como default).
- Erro HTTP 429 deve chegar como `Error` — o componente trata a mensagem.
- Sem cache de resultado — cada chamada é uma mutation independente.

### Critérios de aceitação:
- [ ] `useTestarPrompt` retorna `{ mutate, isPending, data, error }` do React Query.
- [ ] Tipos `TestarPromptPayload` e `TestarPromptResultado` exportados corretamente.
- [ ] `isPending` é `true` durante a chamada HTTP.
- [ ] `data` contém todos os 6 campos do resultado quando a chamada tem sucesso.
- [ ] TypeScript compila sem erros.

## Teste esperado
- Verificação via TypeScript: `tsc --noEmit` deve passar sem erros.
- Verificação manual: usar o hook em um componente de teste temporário e conferir que a mutation funciona.
