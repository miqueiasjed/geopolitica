# Task 17.4 – Componente PromptTestPanel (colapsável, formulário, resultado, validação JSON)

## Objetivo
Criar o componente `PromptTestPanel` que aparece abaixo de cada textarea de prompt, permite inserir uma mensagem de teste, executar via IA e exibir o resultado com validação visual de JSON quando aplicável.

## Especificação

### O que fazer:
Criar componente funcional `PromptTestPanel` que recebe:

```typescript
interface PromptTestPanelProps {
  promptSistema: string;     // valor atual do textarea (não salvo, só o estado do input)
  exigeJson?: boolean;       // se true, valida se a resposta é JSON válido
}
```

**Comportamento:**
1. Renderiza um link/botão `▶ Testar este prompt` que expande/colapsa o painel.
2. Quando expandido, exibe:
   - Textarea "Mensagem de teste" (label + placeholder: "Ex: Analise o conflito Rússia-Ucrânia")
   - Slider ou input numérico `max_tokens` (50–1024, padrão 256) — opcional, pode ser ocultado inicialmente
   - Botão "Executar" (desabilitado se mensagem vazia ou isPending)
3. Ao clicar "Executar":
   - Chama `useTestarPrompt().mutate({ prompt_sistema: promptSistema, mensagem_usuario, max_tokens })`
   - Mostra spinner enquanto `isPending`
4. Quando resultado disponível, exibe:
   - Resposta da IA em bloco `<pre>` com scroll vertical (max-height: 200px)
   - Linha de metadados: `[provider] [modelo] · [duracao_ms]ms · ~[entrada]↑ [saida]↓ tokens`
   - **Se `exigeJson=true`**: indicador verde "✓ JSON válido" ou vermelho "✗ JSON inválido" baseado em `JSON.parse(resposta)`
5. Se `error`, exibe mensagem de erro em vermelho.

**Estilo (Tailwind):** usar cores e tamanhos consistentes com o admin existente. Painel colapsável com transição suave (Framer Motion `AnimatePresence`/`motion.div` com `overflow: hidden`).

### Regras de negócio aplicáveis:
- `promptSistema` vem do estado do formulário pai (valor atual digitado, não o salvo no banco).
- O painel não salva nada — é puramente para visualização de resultado.
- `exigeJson` deve ser `true` para os prompts de análise (analise_sistema, detector_sistema, convergencia_sistema) e `false` para prompts de texto livre (chat_sistema, perfil_contexto, perfil_lideranca).

### Critérios de aceitação:
- [ ] Painel colapsa/expande ao clicar no link "▶ Testar este prompt".
- [ ] Botão "Executar" fica desabilitado enquanto `isPending`.
- [ ] Resultado exibe resposta + metadados (provider, modelo, ms, tokens).
- [ ] Com `exigeJson=true` e resposta sendo JSON válido: badge verde "✓ JSON válido".
- [ ] Com `exigeJson=true` e resposta não sendo JSON: badge vermelho "✗ JSON inválido".
- [ ] Erro HTTP exibe mensagem legível.
- [ ] TypeScript compila sem erros.

## Teste esperado
- Verificação visual: abrir página de configurações de IA no browser, clicar "▶ Testar este prompt" em um dos prompts, inserir mensagem e executar.
- Confirmar que resultado aparece com metadados e que o indicador JSON funciona.
