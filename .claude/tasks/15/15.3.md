## Especificação

### O que fazer:

**`LicencaB2BService`:**

```php
public function criarEmpresaComLicenca(array $dados): Empresa
```
- Criar `Empresa` + `LicencaB2B` em transaction
- Gerar slug do subdomínio: `Str::slug($dados['subdominio'])` (lowercase, sem espaços)
- Verificar unicidade do subdomínio antes de criar (lançar `\InvalidArgumentException` se duplicado)
- `contratado_em = now()`, `expira_em = now()->addYear()` (padrão 1 ano)

```php
public function renovarLicenca(Empresa $empresa, int $meses = 12): LicencaB2B
```
- Atualizar `expira_em` da licença atual ou criar nova
- Reativar empresa se estava inativa

```php
public function desativarLicenca(Empresa $empresa): void
```
- `$empresa->update(['ativo' => false])`

**`MembroB2BService`:**

```php
public function convidar(Empresa $empresa, string $email, string $roleB2B = 'reader'): MembroB2B
```
- Verificar limite: `$empresa->membros()->ativos()->count() < $empresa->max_usuarios`
- Se atingiu: lançar `\RuntimeException("Limite de {$empresa->max_usuarios} usuários atingido.")`
- Verificar se email já é membro ativo
- Criar `MembroB2B` com `convite_token = Str::uuid()`, `convite_email = $email`, `role_b2b = $roleB2B`
- Disparar `ConviteB2BMail::to($email)->send(...)` com o token

```php
public function aceitarConvite(string $token, array $dadosUsuario): User
```
- Buscar `MembroB2B` pelo `convite_token` (404 se não encontrado ou já aceito)
- Criar `User` com `$dadosUsuario` (nome + senha) + `email = membro->convite_email`
- `$user->assignRole($membro->role_b2b)`
- Atualizar `MembroB2B`: `user_id = $user->id`, `aceito_em = now()`, `convite_token = null`
- Retornar o `User` criado

```php
public function removerMembro(MembroB2B $membro): void
```
- Soft delete ou hard delete conforme convenção do projeto
- Revogar roles Spatie do usuário se não pertencer a outra empresa

**`ConviteB2BMail`:**
- Mailable com subject "Você foi convidado para [empresa] no Geopolítica para Investidores"
- Variáveis: `$empresa` (nome), `$linkConvite` (URL com token), `$roleB2B`
- URL do convite: `"https://{$empresa->subdominio}.".config('app.domain')."/aceitar-convite/{$token}"`

**Template Blade (`emails/convite-b2b.blade.php`):**
- Layout simples (pode reutilizar layout de email existente)
- Corpo: saudação + descrição da empresa + link de aceite destacado + botão CTA

**`DesativarLicencasExpiradasCommand`:**
- Signature: `b2b:desativar-expiradas`
- Buscar empresas com `expira_em < now()` e `ativo = true`
- Para cada uma: `$empresa->update(['ativo' => false])`
- Logar quantas foram desativadas

**Scheduler:**
```php
$schedule->command('b2b:desativar-expiradas')
    ->dailyAt('00:05')
    ->withoutOverlapping();
```

### Regras de negócio aplicáveis:
- Limite de membros verificado antes de convidar
- `convite_token` destruído após aceite (campo null)
- `aceitarConvite` deve falhar se token já foi usado (`aceito_em != null`)
- Subdomínio deve ser slug válido (apenas lowercase, números e hífen)

### Critérios de aceitação:
- [ ] `criarEmpresaComLicenca` cria empresa + licença em transação atômica
- [ ] Convite envia e-mail com link correto
- [ ] `aceitarConvite` cria User com role B2B atribuído
- [ ] Segundo uso do token retorna erro
- [ ] Limite de membros verificado com 422 correto
- [ ] Scheduler desativa licenças expiradas diariamente
