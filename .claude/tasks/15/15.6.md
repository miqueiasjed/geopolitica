# Task 15.6 – AdminB2BPage, AceitarConvitePage e integração final

## Objetivo
Implementar o painel admin de licenças B2B, a página pública de aceite de convite (criação de conta do novo membro) e a rota de equipe no router protegida por role B2B.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `src/pages/admin/AdminB2BPage.tsx` → criar
- `src/pages/AceitarConvitePage.tsx` → criar
- `src/pages/dashboard/EquipePage.tsx` → criar
- `src/router.tsx` (ou equivalente) → modificar (adicionar rotas B2B)

## Contexto necessário (arquivos para ler antes)
- `src/components/b2b/TeamPanel.tsx` → (15.5)
- `src/pages/admin/` → ver padrão de páginas admin
- `src/router.tsx` → ver estrutura de rotas protegidas e públicas

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

**`AdminB2BPage.tsx`:**
- Título: "Licenças B2B"
- Botão "+ Nova Licença" → abre modal de criação
- Tabela de empresas: Nome, Subdomínio (link clicável → abre em nova aba), Membros, Expira em (badge vermelho se < 30 dias), Status (ativo/inativo), Ações (Renovar | Editar)
- Modal "Nova Licença": campos `nome`, `subdominio`, `max_usuarios`, `email_admin`, `meses` (select: 6, 12, 24)
- Modal "Renovar": select de `meses` + confirmação
- Query: `GET /api/admin/b2b/empresas` via React Query

**`AceitarConvitePage.tsx`:**
- Rota pública: `/convite/:token`
- Ler `token` via `useParams()`
- Formulário: nome completo (input), senha (input password), confirmar senha
- Submit: `POST /api/convite/{token}/aceitar` com `{ nome, password, password_confirmation }`
- Se sucesso: exibir "Conta criada! Acesse o sistema via [subdomínio]." + link
- Se token inválido/usado: exibir erro "Este convite não é mais válido."
- Sem autenticação necessária (página pública)

**`EquipePage.tsx`:**
- Wrapper simples que renderiza `TeamPanel`
- Proteção: verificar se usuário tem role `company_admin` (redirecionar para `/dashboard` se não tiver)

**Rotas a adicionar:**
```tsx
// Pública
{ path: '/convite/:token', element: <AceitarConvitePage /> }

// Admin do canal
{ path: '/admin/b2b', element: <AdminB2BPage /> } // dentro do layout admin

// Dashboard B2B (company_admin)
{ path: '/dashboard/equipe', element: <EquipePage /> } // dentro do layout dashboard
```

### Regras de negócio aplicáveis:
- `AceitarConvitePage` é pública — sem proteção de auth
- `EquipePage` exige role `company_admin` (verificado no frontend e backend)
- Subdomínio na `AdminB2BPage` deve ser link funcional

### Critérios de aceitação:
- [ ] `AdminB2BPage` lista empresas com status e expiração
- [ ] Modal de nova licença cria empresa + convida admin
- [ ] `AceitarConvitePage` cria conta com token válido
- [ ] Página de aceite exibe erro para token inválido
- [ ] `EquipePage` redireciona não-`company_admin` para dashboard
- [ ] Rotas registradas corretamente

## Teste esperado
- Fluxo completo: admin cria licença → convite enviado → abrir link `/convite/{token}` → criar conta → acessar via subdomínio
- Verificação visual de `/admin/b2b`
