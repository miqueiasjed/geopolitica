## Especificação

### O que fazer:

**`TenantContext.tsx`:**
```typescript
interface TenantData {
  nome: string;
  logo_url: string | null;
  subdominio: string;
  max_usuarios: number;
}

interface TenantContextValue {
  tenant: TenantData | null;
  isB2B: boolean;
  isLoading: boolean;
}
```

Lógica:
- Detectar subdomínio atual: `window.location.hostname`
- Se hostname é o domínio raiz ou `www.`: `isB2B = false`, `tenant = null`
- Se é subdomínio: fazer `GET /api/empresa/info` (endpoint público que retorna dados básicos da empresa pelo subdomínio via tenant middleware)
  - `staleTime: Infinity` (dados estáticos durante a sessão)
- Expor via `TenantProvider` e `useTenant()` hook

**Nota:** adicionar endpoint público `GET /api/empresa/info` no backend (retorna `nome`, `logo_url`, `subdominio` — sem auth):
```php
Route::middleware('tenant')->get('/empresa/info', fn() => response()->json(app('tenant')->only(['nome', 'logo_url', 'subdominio'])));
```

**`DashboardLayout.tsx` (modificar):**
- Usar `useTenant()` no topo do componente
- Se `isB2B && tenant`:
  - Exibir logo da empresa (`logo_url`) na TopNav no lugar do logo padrão
  - Se `logo_url` null: exibir nome da empresa em texto
  - Adicionar badge "B2B" ou nome da empresa abaixo do logo
- Se não B2B: comportamento atual sem alteração
- Envolver `DashboardLayout` no `TenantProvider` no `App.tsx`/`router.tsx`

**`MembersList.tsx`:**
Props: `membros: MembroB2B[]`, `onRemover: (id: number) => void`, `isLoading: boolean`

Tabela com colunas:
- Nome (se aceito) / E-mail (se pendente)
- Role: badge "Admin" (dourado) ou "Leitor" (cinza)
- Status: "Ativo" (verde) / "Convite pendente" (âmbar)
- Data de entrada (aceito_em formatado) / "Aguardando..."
- Ação: botão "Remover" (ícone lixeira, vermelho)
- Skeleton loading

**`InviteMemberModal.tsx`:**
Props: `isOpen: boolean`, `onClose: () => void`, `onSuccess: () => void`
- Modal com `Dialog` do Radix UI
- Campos: e-mail (input), role (select: "Leitor" | "Administrador")
- Botão "Enviar convite"
- Mutation: `POST /api/empresa/convidar`
- Invalidar query de membros após sucesso
- Toast de sucesso + fechar modal

**`TeamPanel.tsx`:**
- Título: "Minha Equipe"
- Badge: "X / Y membros"
- Botão "+ Convidar" abre `InviteMemberModal`
- `MembersList` com dados de `GET /api/empresa/equipe`
- Rota: `/dashboard/equipe` (disponível apenas em contexto B2B)

### Regras de negócio aplicáveis:
- `TenantContext` deve ser silencioso quando não é B2B (não exibir erros)
- Logo da empresa sobrescreve logo padrão no layout B2B
- `TeamPanel` só aparece para `company_admin`
