## Especificação

### O que fazer:

**`EmpresaController` (para `company_admin` da empresa):**

Todos os endpoints verificam: `auth()->user()` é `company_admin` da empresa identificada pelo tenant middleware.

`GET /api/empresa/equipe` — listar membros:
- Usar `app('tenant')` para obter empresa atual
- Retornar membros ativos + pendentes: `id`, `convite_email`, `role_b2b`, `aceito_em`, `usuario.name` (se aceito)
- Incluir contadores: `ativos`, `pendentes`, `max_usuarios`

`POST /api/empresa/convidar` — convidar membro:
- Validar: `email` (required|email), `role_b2b` (required|in:company_admin,reader)
- Chamar `MembroB2BService::convidar()`
- Retornar 201 com o membro criado

`DELETE /api/empresa/membros/{membroId}` — remover membro:
- Verificar que o membro pertence à empresa do tenant
- Não permitir remover a si mesmo se for o único `company_admin`
- Chamar `MembroB2BService::removerMembro()`
- Retornar 204

**`AdminB2BController` (para admin do canal):**

`GET /api/admin/b2b/empresas` — listar todas as licenças:
- Retornar: `id`, `nome`, `subdominio`, `ativo`, `expira_em`, `membros_count`, `licenca.tipo`
- Ordenar por `expira_em ASC` (próximas a vencer primeiro)

`POST /api/admin/b2b/empresas` — criar nova licença:
- Validar: `nome`, `subdominio`, `max_usuarios`, `meses` (padrão 12), `email_admin` (e-mail do primeiro `company_admin`)
- Chamar `LicencaB2BService::criarEmpresaComLicenca()`
- Criar e convidar o primeiro `company_admin` via `MembroB2BService::convidar()`
- Retornar 201

`PATCH /api/admin/b2b/empresas/{id}` — atualizar:
- Campos: `nome`, `logo_url`, `max_usuarios`, `ativo`
- Retornar empresa atualizada

`POST /api/admin/b2b/empresas/{id}/renovar` — renovar licença:
- Parâmetro: `meses` (default 12)
- Chamar `LicencaB2BService::renovarLicenca()`
- Retornar licença atualizada

**Rotas:**
```php
// company_admin gerenciando equipe (requer tenant middleware)
Route::middleware(['auth:sanctum', 'tenant', 'role:company_admin'])->group(function () {
    Route::get('/empresa/equipe', [EmpresaController::class, 'equipe']);
    Route::post('/empresa/convidar', [EmpresaController::class, 'convidar']);
    Route::delete('/empresa/membros/{membroId}', [EmpresaController::class, 'removerMembro']);
});

// Admin do canal — gestão de licenças
Route::middleware(['auth:sanctum', 'role:admin'])->prefix('admin')->group(function () {
    Route::get('/b2b/empresas', [AdminB2BController::class, 'index']);
    Route::post('/b2b/empresas', [AdminB2BController::class, 'store']);
    Route::patch('/b2b/empresas/{id}', [AdminB2BController::class, 'update']);
    Route::post('/b2b/empresas/{id}/renovar', [AdminB2BController::class, 'renovar']);
});
```

Rota pública para aceite de convite:
```php
Route::post('/convite/{token}/aceitar', [EmpresaController::class, 'aceitarConvite']);
```

### Regras de negócio aplicáveis:
- `company_admin` só acessa dados da sua própria empresa (verificado via tenant)
- Admin do canal acessa todas as empresas
- Convite de aceite é rota pública (sem auth) pois o usuário ainda não existe

### Critérios de aceitação:
- [ ] `GET /api/empresa/equipe` retorna membros com contadores
- [ ] `POST /api/empresa/convidar` envia convite e retorna 201
- [ ] `DELETE /api/empresa/membros/{id}` retorna 403 se membro de outra empresa
- [ ] `POST /api/admin/b2b/empresas` cria empresa + convida primeiro admin
- [ ] `POST /api/admin/b2b/empresas/{id}/renovar` estende `expira_em`
- [ ] Usuário sem role `company_admin` recebe 403 nas rotas da empresa
