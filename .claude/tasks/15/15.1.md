# Task 15.1 – Infraestrutura: wildcard DNS, SSL e Nginx multi-subdomínio

## Objetivo
Configurar a infraestrutura de servidor para suportar multi-tenant via subdomínio: wildcard DNS, certificado SSL wildcard e Nginx aceitando `*.geopoliticainvestidores.com.br` na mesma aplicação Laravel.

## Tipo
config

## Arquivos a criar/modificar
- `nginx/sites-available/geopolitica.conf` → criar (configuração de exemplo documentada)
- `.env.example` → modificar (adicionar `APP_DOMAIN` e `TENANT_SUBDOMAIN`)
- `config/app.php` → modificar (adicionar `'domain' => env('APP_DOMAIN', 'geopoliticainvestidores.com.br')`)

## Contexto necessário (arquivos para ler antes)
- `.env.example` → ver variáveis existentes
- `config/app.php` → ver configurações existentes

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**Documentar configuração Nginx (`nginx/sites-available/geopolitica.conf`):**
```nginx
# Wildcard para todos os subdomínios + domínio raiz
server {
    listen 443 ssl http2;
    server_name geopoliticainvestidores.com.br *.geopoliticainvestidores.com.br;

    # Certificado wildcard (Let's Encrypt ou Cloudflare)
    ssl_certificate /etc/letsencrypt/live/geopoliticainvestidores.com.br/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/geopoliticainvestidores.com.br/privkey.pem;

    root /var/www/geopolitica/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # Desabilitar buffering para SSE do chat
    location /api/chat/perguntar {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_buffering off;
        proxy_buffering off;
    }
}

server {
    listen 80;
    server_name geopoliticainvestidores.com.br *.geopoliticainvestidores.com.br;
    return 301 https://$host$request_uri;
}
```

**Instruções de setup do SSL wildcard:**
Adicionar comentário no arquivo Nginx com os comandos Let's Encrypt:
```bash
# certbot certonly --dns-cloudflare \
#   --dns-cloudflare-credentials ~/.secrets/cloudflare.ini \
#   -d geopoliticainvestidores.com.br \
#   -d "*.geopoliticainvestidores.com.br"
```

**Variáveis de ambiente (`.env.example`):**
```dotenv
APP_DOMAIN=geopoliticainvestidores.com.br
```

**`config/app.php`:**
Adicionar:
```php
'domain' => env('APP_DOMAIN', 'geopoliticainvestidores.com.br'),
```

**Configuração de CORS (`config/cors.php`):**
Atualizar `allowed_origins` para aceitar wildcard de subdomínio:
```php
'allowed_origins_patterns' => [
    '#^https?://([a-z0-9-]+\.)?geopoliticainvestidores\.com\.br$#',
],
```

### Regras de negócio aplicáveis:
- `fastcgi_buffering off` no endpoint `/api/chat/perguntar` para SSE funcionar corretamente
- Wildcard DNS deve ser `*.geopoliticainvestidores.com.br` apontando para o mesmo IP
- SSL wildcard requer validação DNS (não HTTP) — usar Cloudflare ou DNS manual
