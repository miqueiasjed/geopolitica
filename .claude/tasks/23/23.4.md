## Especificação

### O que fazer:

#### `RelatorioIaController.php`

**`gerar(Request $request): StreamedResponse`**
```php
$request->validate([
    'topico' => 'required|string|max:300',
    'escopo' => 'nullable|string|max:500',
]);

return response()->stream(function () use ($request): void {
    $relatorio = $this->relatorioIaService->gerarComStreaming(
        usuario:        auth()->user(),
        topico:         $request->input('topico'),
        escopo:         $request->input('escopo', ''),
        aoReceberToken: function (string $token): void {
            echo 'data: ' . json_encode(['token' => $token]) . "\n\n";
            ob_flush();
            flush();
        },
    );

    echo 'data: ' . json_encode(['done' => true, 'relatorio_id' => $relatorio->id]) . "\n\n";
    ob_flush();
    flush();
}, 200, [
    'Content-Type'      => 'text/event-stream',
    'Cache-Control'     => 'no-cache',
    'X-Accel-Buffering' => 'no',
]);
```
Capturar `TooManyRequestsHttpException` → echo SSE com `['error' => 'limit_reached', 'mensagem' => ...]` e encerrar stream.

**`historico(Request $request): JsonResponse`**
- Retorna `$this->relatorioIaService->listarRelatorios(auth()->id())` com campos resumidos (sem `body` completo): `id, title, topic, word_count, status, created_at`

**`show(Request $request, string $id): JsonResponse`**
- Retorna o relatório completo via `buscarRelatorio($id, auth()->id())`
- 404 se não encontrado

#### `routes/api.php`
Dentro do grupo `auth:sanctum`:
```php
Route::post('/relatorios', [RelatorioIaController::class, 'gerar']);
Route::get('/relatorios',  [RelatorioIaController::class, 'historico']);
Route::get('/relatorios/{id}', [RelatorioIaController::class, 'show']);
```

### Regras de negócio aplicáveis:
- `Content-Type: text/event-stream` para streaming SSE.
- O evento final `done: true` deve incluir o `relatorio_id` para o frontend redirecionar.
- Erro de limite enviado como evento SSE (não como status HTTP) para chegar ao consumer do stream.

### Critérios de aceitação:
- [ ] `POST /api/relatorios` retorna stream SSE com tokens em tempo real
- [ ] Stream termina com evento `{"done":true,"relatorio_id":"..."}` 
- [ ] `GET /api/relatorios` retorna lista de relatórios do assinante autenticado
- [ ] `GET /api/relatorios/{id}` retorna relatório completo ou 404
- [ ] Limite atingido retorna evento SSE com `error: limit_reached`
- [ ] Rotas sem autenticação retornam 401
