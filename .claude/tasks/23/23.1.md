## Especificação

### O que fazer:

#### `create_relatorios_ia_table`
```php
Schema::create('relatorios_ia', function (Blueprint $table) {
    $table->uuid('id')->primary()->default(DB::raw('(UUID())'));
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('title');
    $table->text('topic');
    $table->text('scope')->nullable();
    $table->longText('body');
    $table->json('sources_used')->nullable();
    $table->unsignedInteger('word_count')->nullable();
    $table->string('status')->default('completed');
    // status: generating | completed | failed
    $table->timestamps();

    $table->index(['user_id', 'created_at']);
});
```

#### `create_uso_relatorios_table`
```php
Schema::create('uso_relatorios', function (Blueprint $table) {
    $table->bigIncrements('id');
    $table->foreignId('user_id')->constrained()->cascadeOnDelete();
    $table->string('month_key', 7); // formato: "2026-04"
    $table->unsignedInteger('count')->default(0);
    $table->timestamps();

    $table->unique(['user_id', 'month_key']);
});
```

### Regras de negócio aplicáveis:
- `relatorios_ia.id` é UUID para evitar enumeração.
- `uso_relatorios` tem constraint `unique(user_id, month_key)` para garantir upsert seguro.
- `status` controla o estado durante geração em streaming (saving `generating` antes de iniciar).

### Critérios de aceitação:
- [ ] Migration `relatorios_ia` cria tabela com todos os campos corretos
- [ ] Migration `uso_relatorios` cria tabela com unique constraint em (user_id, month_key)
- [ ] `php artisan migrate` executa sem erros
- [ ] `php artisan migrate:rollback` reverte ambas as tabelas
