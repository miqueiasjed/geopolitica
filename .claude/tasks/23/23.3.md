# Task 23.3 – RelatorioIaService: limite, contexto, geração e persistência

## Objetivo
Criar o `RelatorioIaService` com toda a lógica de negócio: verificação de limite mensal por plano, coleta de contexto nas 4 fontes da plataforma, geração via Claude API com streaming e persistência do relatório completo.

backend-logica

## Especificação

### O que fazer:
Criar `App\Services\RelatorioIaService` com os métodos:

#### `verificarLimite(User $usuario): void`
Limites por plano:
```php
private const LIMITES_POR_PLANO = [
    'essencial' => 2,
    'pro'       => 10,
];
private const PLANOS_SEM_LIMITE = ['reservado', 'admin'];
```
- Busca o registro de `uso_relatorios` para `user_id` + `month_key` (formato `Y-m`, fuso `America/Sao_Paulo`)
- Lança `TooManyRequestsHttpException` se `count >= limite`
- Planos sem limite (reservado, admin) retornam imediatamente

#### `coletarContexto(string $topico, string $escopo = ''): array`
- Reutiliza os métodos privados de `ChatRecuperacaoService` (injetar e chamar ou replicar a chamada SQL)
- Busca em paralelo (ou sequencial): 5 conteúdos, 5 eventos, 3 perfis de países, 3 crises históricas
- Retorna `['texto' => string, 'fontes' => array]`
- Formato do texto: `[FONTE N — TIPO — TÍTULO]\n{excerpt}\n\n---\n\n`

#### `gerarComStreaming(User $usuario, string $topico, string $escopo, callable $aoReceberToken): RelatorioIa`
Fluxo:
1. `verificarLimite($usuario)`
2. `coletarContexto($topico, $escopo)`
3. Criar `RelatorioIa` com `status = 'generating'` e `body = ''` (persistir já para ter o ID)
4. Montar system prompt com estrutura obrigatória (5 seções) + regras de tom + contexto coletado
5. Chamar o AI provider com streaming, acumular tokens, chamar `$aoReceberToken($token)` a cada chunk
6. Após conclusão: atualizar `RelatorioIa` com `body`, `word_count`, `sources_used`, `status = 'completed'`
7. Incrementar `uso_relatorios` via `updateOrCreate` + `increment('count')` ou `upsert`
8. Retornar o `RelatorioIa` atualizado

System prompt obrigatório:
```
Você é um analista geopolítico sênior. Gere um relatório analítico formal sobre o tema fornecido,
baseado EXCLUSIVAMENTE no contexto abaixo.

ESTRUTURA OBRIGATÓRIA (use estes títulos exatos em Markdown):
# [Título do relatório]
## Contexto geopolítico
## Dinâmicas em curso
## Implicações econômicas para o Brasil
## Cenários e riscos
## O que monitorar

REGRAS:
- Tom: analítico, direto, sem eufemismos
- Extensão: 800 a 1.200 palavras no total
- Cite as fontes como [Fonte: título] ao longo do texto
- Não faça recomendações de investimento
- Se o contexto for insuficiente para alguma seção, indique explicitamente

CONTEXTO DISPONÍVEL:
{contexto}
```

#### `buscarRelatorio(string $id, int $userId): RelatorioIa`
- `RelatorioIa::where('id', $id)->where('user_id', $userId)->firstOrFail()`

#### `listarRelatorios(int $userId): Collection`
- `RelatorioIa::where('user_id', $userId)->orderByDesc('created_at')->get()`

### Regras de negócio aplicáveis:
- `month_key` sempre calculado no fuso `America/Sao_Paulo` para evitar virada de mês incorreta.
- Relatório criado como `generating` antes de iniciar streaming — garante que o ID existe se o usuário atualizar a página.
- Se o streaming falhar, atualizar `status = 'failed'` para não deixar relatório em estado inconsistente.

### Critérios de aceitação:
- [ ] `verificarLimite` bloqueia corretamente para plano `essencial` com 2+ relatórios no mês
- [ ] `coletarContexto` retorna resultados das 4 fontes
- [ ] System prompt enviado com a estrutura das 5 seções obrigatórias
- [ ] Streaming funciona: callback `$aoReceberToken` chamado a cada token
- [ ] `RelatorioIa` persistido com `body`, `word_count` e `sources_used` após conclusão
- [ ] `uso_relatorios` incrementado após geração bem-sucedida
- [ ] `status = 'failed'` em caso de erro no streaming
