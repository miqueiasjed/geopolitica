# Task 23.6 – Página NovoRelatorio: formulário e preview em streaming

## Objetivo
Criar a página `NovoRelatorio.tsx` com o formulário de geração de relatórios: campo de tema com 5 sugestões clicáveis, campo de escopo opcional, botão gerar, e preview em tempo real do relatório sendo gerado com Markdown renderizado.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/dashboard/NovoRelatorio.tsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useGerarRelatorio.ts` → interface do hook (Task 23.5)
- `frontend/src/pages/ChatBriefings.tsx` → padrão visual de página com campo de input e estado de loading
- `frontend/src/components/ExportPdfButton.tsx` → componente a usar após geração (Task 22.6)

## Skills necessárias
- frontend-design-system
- animacoes-motion

## Especificação

### O que fazer:
Criar página que usa o hook `useGerarRelatorio` e renderiza:

#### Cabeçalho da página
- Label "Geopolítica para Investidores" em dourado/60, 10px uppercase tracking-widest
- Título "Relatório Personalizado por IA" em serif, 20px
- Contador de uso: "N relatórios restantes este mês" em branco/30 (quando há limite; nada para plano sem limite)

#### Formulário
- Label "Tema do relatório" + `<input>` para o tema
- 5 sugestões de tema como botões clicáveis (ao clicar, preenche o campo de tema):
  1. "Risco energético para portfólios com exposição europeia"
  2. "Impacto da guerra no Oriente Médio nas cadeias de fertilizantes para o Brasil"
  3. "Eleições de 2026 nos EUA — cenários e implicações para emergentes"
  4. "Risco geopolítico no setor de semicondutores — tensão EUA-China"
  5. "Panorama de risco para o agronegócio brasileiro — próximos 90 dias"
- Label "Escopo adicional (opcional)" + `<textarea>` para escopo (2 linhas, placeholder: "Ex: Foco no impacto para exportadores de soja. Horizonte de 90 dias.")
- Botão principal: "Gerar relatório →" (dourado/ouro quando disponível, 40% opacity quando desabilitado)

#### Aviso de limite atingido
- Se `erroLimite` estiver ativo: caixa de aviso com mensagem "Limite de relatórios mensais atingido. Faça upgrade do seu plano." e link/botão para upgrade

#### Preview em streaming
- Seção que aparece quando `preview` não está vazio
- Markdown renderizado usando `react-markdown` (ou `marked`) em container `prose-invert`
- Borda sutil `border border-white/8 p-6`
- Após `relatorioId` disponível:
  - `ExportPdfButton tipo="report" id={relatorioId} label="Exportar PDF"`
  - Botão secundário "Ver histórico" que navega para `/dashboard/relatorios`

### Regras de negócio aplicáveis:
- Botão "Gerar" desabilitado quando `topico.trim()` está vazio, `gerando` é true ou `erroLimite` é true.
- Preview só aparece quando `preview !== ''`.
- Após `relatorioId` disponível, os botões de ação aparecem abaixo do preview.

### Critérios de aceitação:
- [ ] Formulário com input de tema e textarea de escopo funcionais
- [ ] 5 sugestões clicáveis que preenchem o campo de tema
- [ ] Preview em tempo real renderizando Markdown enquanto IA gera
- [ ] `ExportPdfButton` visível após conclusão com `tipo="report"`
- [ ] Aviso de limite atingido exibido quando `erroLimite` é true
- [ ] Botão de geração desabilitado nos estados corretos
- [ ] Layout consistente com o design system escuro do projeto

## Teste esperado
- Verificação manual: abrir `/dashboard/relatorios/novo`, digitar tema, clicar gerar e verificar que o preview cresce
- Testar sugestões: clicar em uma sugestão e confirmar que o campo é preenchido
- Testar export: após geração, clicar "Exportar PDF" e verificar download
