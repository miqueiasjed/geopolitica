## Especificação

### O que fazer:

#### `RelatorioIa.php`
```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class RelatorioIa extends Model
{
    protected $table = 'relatorios_ia';

    public $incrementing = false;
    protected $keyType = 'string';

    protected $fillable = [
        'id', 'user_id', 'title', 'topic', 'scope',
        'body', 'sources_used', 'word_count', 'status',
    ];

    protected function casts(): array
    {
        return [
            'sources_used' => 'array',
        ];
    }

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }
}
```

#### `UsoRelatorio.php`
```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class UsoRelatorio extends Model
{
    protected $table = 'uso_relatorios';

    protected $fillable = ['user_id', 'month_key', 'count'];

    public function user(): BelongsTo
    {
        return $this->belongsTo(User::class);
    }
}
```

### Regras de negócio aplicáveis:
- `RelatorioIa` usa UUID como chave primária (`$incrementing = false`, `$keyType = 'string'`).
- `sources_used` é cast para `array` — armazena lista de fontes consultadas como JSON.
- Ambos os models têm `user_id` e relacionamento `belongsTo(User::class)`.

### Critérios de aceitação:
- [ ] `RelatorioIa` com UUID como PK, fillable completo e cast de `sources_used`
- [ ] `UsoRelatorio` com fillable e relacionamento com User
- [ ] Ambos os models instanciam sem erros: `new RelatorioIa()`, `new UsoRelatorio()`
