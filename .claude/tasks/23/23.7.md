# Task 23.7 – Página RelatoriosIA (histórico), rotas e navegação

## Objetivo
Criar a página de histórico de relatórios, adicionar as duas rotas no router (`/dashboard/relatorios` e `/dashboard/relatorios/novo`) e inserir "Relatórios" na navegação lateral do `TopNav`.

## Tipo
frontend-pagina

## Arquivos a criar/modificar
- `frontend/src/pages/dashboard/RelatoriosIA.tsx` → criar
- `frontend/src/router/index.tsx` → modificar (adicionar 2 rotas)
- `frontend/src/components/TopNav.tsx` → modificar (adicionar item de navegação)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/pages/dashboard/Biblioteca.tsx` → padrão de página de listagem com cards
- `frontend/src/router/index.tsx` → onde adicionar as novas rotas (após `/dashboard/chat`)
- `frontend/src/components/TopNav.tsx` → como adicionar item à lista `navItems`

## Skills necessárias
- frontend-design-system
- animacoes-motion

## Especificação

### O que fazer:

#### `RelatoriosIA.tsx`
Página de histórico com:
- Cabeçalho: "Relatórios" (título serif) + botão "Novo relatório →" que navega para `/dashboard/relatorios/novo`
- Lista de relatórios via `useQuery` → `GET /api/relatorios`
- Cada card exibe: título do relatório, tema resumido (topic), contagem de palavras, data de criação formatada em pt-BR
- Estado vazio: "Nenhum relatório gerado ainda. Crie seu primeiro relatório →" com link
- Estado de loading: skeleton cards
- Ao clicar em um card: exibir o relatório completo (expandir inline ou navegar para `GET /api/relatorios/{id}` e renderizar com react-markdown)

#### `router/index.tsx`
Adicionar após a rota `/dashboard/chat`:
```tsx
import { NovoRelatorio } from '../pages/dashboard/NovoRelatorio'
import { RelatoriosIA } from '../pages/dashboard/RelatoriosIA'
// ...
<Route path="/dashboard/relatorios" element={<RelatoriosIA />} />
<Route path="/dashboard/relatorios/novo" element={<NovoRelatorio />} />
```

#### `TopNav.tsx`
Adicionar item na lista `navItems` após "Chat":
```typescript
{ to: '/dashboard/relatorios', label: 'Relatórios', isActive: (p) => p.startsWith('/dashboard/relatorios') }
```
O item é visível para todos os assinantes (sem restrição de plano — o limite é controlado pelo contador de uso dentro da página).

### Regras de negócio aplicáveis:
- "Relatórios" aparece na navegação para todos os planos (a limitação é quantidade, não acesso).
- O item de "Relatórios" deve estar ativo (`aria-current`) quando a rota começa com `/dashboard/relatorios`.
- Histórico ordenado por `created_at DESC` (já retornado assim pelo backend).

### Critérios de aceitação:
- [ ] `/dashboard/relatorios` exibe lista de relatórios com dados corretos
- [ ] `/dashboard/relatorios/novo` exibe o formulário de geração
- [ ] "Relatórios" visível na navegação lateral para todos os assinantes
- [ ] Item de navegação ativo quando rota começa com `/dashboard/relatorios`
- [ ] Estado vazio exibe CTA para criar primeiro relatório
- [ ] Estado de loading com skeletons visível durante fetch

- Confirmar que item "Relatórios" aparece no menu e fica ativo nas duas sub-rotas
