# Task 23.5 – Hook useGerarRelatorio: consumer SSE com preview em tempo real

## Objetivo
Criar o hook `useGerarRelatorio` que gerencia o estado do formulário, faz POST para `/api/relatorios`, consome o stream SSE e atualiza o preview do relatório token a token em tempo real.

## Especificação

### O que fazer:
Interface retornada pelo hook:
```typescript
interface UseGerarRelatorioReturn {
  topico: string
  setTopico: (v: string) => void
  escopo: string
  setEscopo: (v: string) => void
  gerando: boolean
  preview: string
  relatorioId: string | null
  erroLimite: boolean
  gerar: () => Promise<void>
  resetar: () => void
}
```

Lógica da função `gerar()`:
1. Validar que `topico.trim()` não está vazio
2. `setGerando(true)`, `setPreview('')`, `setRelatorioId(null)`, `setErroLimite(false)`
3. Fazer fetch `POST /api/relatorios` com `{ topico, escopo }` e `Authorization: Bearer`
4. Verificar `res.ok` — se não, tratar erro genérico
5. Usar `ReadableStream` + `TextDecoder` para consumir o SSE:
   ```typescript
   const reader = res.body!.getReader()
   const decoder = new TextDecoder()
   let buffer = ''
   while (true) {
     const { done, value } = await reader.read()
     if (done) break
     buffer += decoder.decode(value, { stream: true })
     const lines = buffer.split('\n\n')
     buffer = lines.pop() ?? ''
     for (const line of lines) {
       if (!line.startsWith('data: ')) continue
       const data = JSON.parse(line.slice(6))
       if (data.token) setPreview(prev => prev + data.token)
       if (data.done)  setRelatorioId(data.relatorio_id)
       if (data.error === 'limit_reached') setErroLimite(true)
     }
   }
   ```
6. `setGerando(false)` no `finally`

### Regras de negócio aplicáveis:
- `preview` é construído concatenando cada token recebido — nunca substituir, sempre concatenar.
- `erroLimite` sinaliza para o componente exibir mensagem de upgrade de plano.
- `resetar()` limpa todo o estado para permitir nova geração.

### Critérios de aceitação:
- [ ] Hook exportado como `useGerarRelatorio`
- [ ] `preview` atualizado token a token durante o streaming
- [ ] `relatorioId` preenchido quando evento `done` chega
- [ ] `erroLimite` ativado quando API retorna `error: limit_reached`
- [ ] `gerando` retorna a `false` após conclusão ou erro
- [ ] `resetar()` limpa todos os estados
