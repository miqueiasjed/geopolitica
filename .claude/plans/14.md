# Plano 14 – Chat com os Briefings (M09)

## Objetivo
Implementar o chat com IA onde o assinante faz perguntas em linguagem natural e recebe respostas contextualizadas com base no conteúdo do canal — Biblioteca, eventos, perfis de países e crises históricas — com streaming SSE e histórico persistente por sessão diária.

## Contexto de Negócio
O chat consulta simultaneamente 4 fontes via FULLTEXT MySQL: `conteudos` (M03), `events` (M01), `perfis_paises` (M06) e `crises_historicas` (M07). Respostas via streaming Server-Sent Events (SSE). Limite diário por plano: `essencial` → 5 perguntas; `pro` → 20; `reservado` → ilimitado. Limite reinicia à meia-noite (Brasília). Contagem via Redis. Modelo: `claude-sonnet-4-5` (ou versão mais recente disponível). Histórico de conversa persistido por sessão diária no banco.

Detalhes completos em: `prd/modulo-09-chat-briefings.md`

## Escopo

### Inclui
**Backend:**
- Migrations: `chat_sessoes` (user_id, data_sessao, pergunta_count), `chat_mensagens` (sessao_id, role, conteudo, created_at)
- Models `ChatSessao` e `ChatMensagem`
- `ChatRecuperacaoService` — busca FULLTEXT nas 4 fontes, retorna contexto relevante
- `ChatService` — monta prompt com contexto, chama Claude API com streaming, salva mensagens, verifica limite
- `ChatController` — `POST /api/chat/perguntar` (inicia streaming SSE via `StreamedResponse`)
- `ChatHistoricoController` — `GET /api/chat/historico` (sessão do dia)
- Contagem de perguntas via Redis com TTL até meia-noite (Brasília)
- Verificação de limite antes de chamar Claude API (HTTP 429 com mensagem de upgrade)
- Middleware para verificar plano e limite
- Variáveis `.env`: `ANTHROPIC_API_KEY` (se não existir do M04)

**Frontend:**
- Types para chat (mensagem, sessão)
- Hook `useChat.ts` — gerencia estado de mensagens, streaming, loading
- Serviço `chatApi.ts` — fetch com `text/event-stream` para SSE
- `ChatInput.tsx` — campo de texto com botão enviar, ctrl+enter
- `ChatMessage.tsx` — bolha de mensagem com Markdown renderizado (user → direita, IA → esquerda)
- `ChatMessages.tsx` — lista com auto-scroll ao fundo
- `ChatLimitWarning.tsx` — aviso de limite atingido com CTA de upgrade
- Página `ChatBriefings.tsx` — `/dashboard/chat`
- Atualizar `TopNav.tsx` com link "Chat"
- Instalar `react-markdown` para renderização
