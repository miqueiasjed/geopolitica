# Plano 35 – Mobile: Componentes Compartilhados (Painéis, Modais, Toasts)

## Objetivo
Garantir que os componentes de UI compartilhados entre admin e assinantes (drawers laterais, modais, toasts/notificações) funcionem corretamente em telas muito pequenas (375px), sem ocultar conteúdo ou bloquear interações.

## Contexto de Negócio
Painéis como `AlertaPanel`, `EleicaoDetailPanel` e `CriseDetailPanel` já têm algumas classes responsivas (`sm:w-[440px]`), mas precisam de revisão em 375px (iPhone SE). Modais criados com Radix UI Dialog podem ter largura inadequada em mobile. Toasts/notificações fixos podem sobrepor botões de ação.

## Escopo

### Inclui

**Drawers laterais (Sheet/fixed panels):**
- `AlertaPanel` — painel de alertas fixo à direita (`sm:w-80 md:w-96`) → garantir `w-full` em mobile com swipe para fechar
- `EleicaoDetailPanel` — detalhe de eleição (`sm:w-[440px]`) → `w-full` em 375px, scroll interno correto
- `CriseDetailPanel` — detalhe de crise (`sm:w-[480px]`) → idem

**Modais (Radix UI Dialog):**
- Auditar todos os `<Dialog.Content>` do projeto para verificar `max-w` em mobile
- Garantir que modais não excedam a viewport em largura (`max-w-[calc(100vw-2rem)]` ou similar)
- Verificar que o scroll interno funciona quando o conteúdo é maior que a tela
- Confirmar que o overlay fecha o modal ao clicar fora em touch

**Toasts / Notificações:**
- Verificar posição dos toasts (bottom-right por padrão) → não devem sobrepor navegação mobile
- Em mobile (< sm), toasts devem ser `bottom-center` com `w-[calc(100vw-2rem)]` ou similar
- Confirmar que não bloqueiam botões de ação fixos (ex: "Enviar relatório", "Salvar")

**Formulários em modais:**
- Campos de formulário dentro de dialogs em mobile → garantir `w-full`, sem overflow

### Não inclui
- Layout principal (admin/assinantes) → Planos 32, 33, 34
- Páginas específicas → Planos 33, 34
