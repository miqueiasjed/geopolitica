# Plano 15 – Licenciamento B2B (M11)

## Objetivo
Transformar o produto em negócio B2B multi-tenant — gestoras, family offices e consultorias acessam o dashboard via subdomínio próprio com logo e identidade da empresa, gerenciados por um `company_admin` com self-service de equipe.

## Contexto de Negócio
Single database, shared schema com `empresa_id`. Subdomínio identifica o tenant via middleware Laravel. URL padrão: `subdominio.geopoliticainvestidores.com.br`. Cada licença tem: URL própria, logo + nome da empresa na nav, usuários configuráveis (5–20 típico), roles internos `company_admin` e `reader`, acesso idêntico ao plano Reservado (todos M01–M09), 50 perguntas/dia por usuário no chat. Infraestrutura: wildcard DNS + wildcard SSL + Nginx multi-subdomínio. Ticket: R$ 15.000–50.000/ano.

Detalhes completos em: `prd/modulo-11-licenciamento-b2b.md`

## Escopo

### Inclui
**Infraestrutura:**
- Configurar wildcard DNS `*.geopoliticainvestidores.com.br` → IP do servidor
- Configurar wildcard SSL (Let's Encrypt ou similar)
- Configurar Nginx para aceitar todos os subdomínios na mesma app Laravel

**Backend:**
- Migrations: `empresas` (nome, subdominio UNIQUE, logo_url, ativo, max_usuarios, expira_em), `licencas_b2b` (empresa_id, tipo, ativa, datas), `membros_b2b` (empresa_id, user_id, role_b2b, convite_token, aceito_em)
- Seeders: roles B2B Spatie (`company_admin`, `reader`)
- Models: `Empresa`, `LicencaB2B`, `MembroB2B`
- Middleware `IdentificarTenantMiddleware` — resolve empresa pelo subdomínio, injeta `empresa_id` no request
- `LicencaB2BService` — CRUD de empresas/licenças, verificação de expiração
- `MembroB2BService` — convites (token único), aceite, remoção, verificação de limite
- `EmpresaController` — endpoints para `company_admin` gerenciar sua equipe
- `AdminB2BController` — admin do canal: CRUD de licenças em `/api/admin/b2b`
- Fluxo de convite: envio (e-mail com link token) → aceite (cria User + MembroB2B + assignRole)
- Template Blade para e-mail de convite
- Scheduler: verificação diária de licenças expiradas (desativa empresa)
- Escopo automático de queries por `empresa_id` em middleware

**Frontend:**
- `TenantContext.tsx` — carrega dados da empresa (logo, nome) pelo subdomínio
- Atualizar `DashboardLayout.tsx` — exibe logo e nome da empresa quando contexto B2B
- CSS Custom Properties para tema da empresa (logo + nome na nav)
- Painel `TeamPanel.tsx` — lista membros, badge de role, invite
- `MembersList.tsx` — tabela de membros com status de convite
- `InviteMemberModal.tsx` — formulário de convite por e-mail
- Painel admin `/admin/b2b` — CRUD de licenças com status e expiração
- `AdminB2BPage.tsx` — listagem de empresas licensiadas
