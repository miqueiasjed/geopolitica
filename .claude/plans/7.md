# Plano 7 – Biblioteca de Conteúdo: Backend (M03)

## Objetivo
Implementar a API completa da Biblioteca de Conteúdo — CRUD admin para publicação, endpoint de listagem com busca full-text e filtros, e endpoint de leitura individual com controle de acesso por plano.

## Contexto de Negócio
Conteúdo publicado manualmente pelo admin (briefings, mapas, teses). Controle de acesso por plano: `essencial` → apenas briefings (90 dias); `pro` → briefings + mapa + tese (90 dias); `reservado` → tudo sem limite. Busca full-text MySQL via `MATCH AGAINST`. Paginação cursor-based. Slug único gerado com timestamp. Plano insuficiente retorna HTTP 404 (não revela que o conteúdo existe).

Detalhes completos em: `prd/modulo-03-biblioteca-conteudo.md`

## Escopo

### Inclui
- Migration `conteudos` (tipo, titulo, slug, corpo HTML, resumo, regiao, tags[], tese_manchete, plano_minimo, publicado_em, publicado) com índices FULLTEXT e UNIQUE no slug
- Model `Conteudo` com scopes `publicados()` e `acessivelPor(role)` e `gerarSlug()`
- `ConteudoService` — `listar()`, `buscarPorSlug()`, `criar()`, `atualizar()`, `despublicar()`
- `BibliotecaController` — `GET /api/biblioteca` (listagem filtrada por role)
- `ConteudoController` — `GET /api/biblioteca/{slug}` (leitura com controle de plano)
- `AdminConteudoController` — CRUD completo para role `admin`
- `BibliotecaFiltroRequest` — validação de `q`, `tipo`, `regiao`, `de`, `ate`, `cursor`, `limite`
- `CriarConteudoRequest` e `AtualizarConteudoRequest` — validação admin
- `ConteudoResource` (completo) e `ConteudoCardResource` (resumido para listagem)
- `ConteudoPolicy` — regras de acesso por role
- Seeder: 1 briefing, 1 mapa, 1 tese por nível de plano (para testes)
- Rotas: `GET/POST /api/admin/conteudos`, `PATCH/DELETE /api/admin/conteudos/{id}`

### Não inclui (fica para outro plano)

- Editor TipTap → Plano 8
