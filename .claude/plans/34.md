# Plano 34 – Mobile: Subscriber Pages (Refinamento)

## Objetivo
Corrigir as páginas do dashboard de assinantes que ainda têm problemas em mobile (375px–768px), principalmente páginas com visualizações complexas como mapas, timelines, monitores e layouts de dois painéis.

## Contexto de Negócio
O `DashboardLayout` + `TopNav` já tem hamburger menu funcional (Plano 5). O problema está nas páginas filhas que assumem viewport larga. As páginas mais críticas são as que possuem painéis laterais side-by-side, grids fixos e visualizações de dados horizontais. O assinante comum acessa o app majoritariamente em mobile.

## Escopo

### Inclui

**Páginas com layout de dois painéis (painel lateral + conteúdo):**
- `EleicaoPage` (assinante) — grid de cards + `EleicaoDetailPanel` → painel vira drawer/sheet em mobile (painel já tem `sm:w-[440px]`, mas revisar comportamento em 375px)
- `MonitorGuerraPage` — mapa + painel de notícias lateral → painel empilha abaixo do mapa em mobile ou vira drawer
- `MonitorEleitoralPage` — idem ao monitor de guerra

**Páginas com visualização de dados horizontal:**
- `MapaPage` — mapa de calor Leaflet/similar → garantir `touch-action: pan-y` + controles de zoom acessíveis em touch + container sem overflow-x oculto
- `TimelinePage` — timeline horizontal → adicionar scroll horizontal nativo com `overflow-x-auto` + snap points ou converter para lista vertical em mobile
- `RiskScorePage` — gráficos + tabela de portfólio → gráficos responsivos (`ResponsiveContainer` já em uso?), tabela com `overflow-x-auto`

**Páginas com grid/layout que precisa de 1 coluna em mobile:**
- `BibliotecaPage` (assinante) — filtros + grid de conteúdos → filtros em drawer/sheet em mobile, grid `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- `ChatPage` — layout de dois painéis (histórico de conversas + chat ativo) → em mobile: mostrar apenas uma tela por vez com botão "Voltar"
- `RelatoriosPage` — lista de relatórios + formulário → verificar empilhamento, botão "Novo Relatório" acessível em mobile
- `PaisPage` (perfil de país) — seções de gráficos side-by-side → `flex-col md:flex-row`

### Não inclui
- `AdminLayout` e páginas admin → Planos 32 e 33
- `AlertaPanel`, `EleicaoDetailPanel`, `CriseDetailPanel` (drawers compartilhados) → Plano 35
- `TopNav` / `DashboardLayout` (já funcional) → sem alteração

- [ ] `RiskScorePage`: tabela de portfólio com `overflow-x-auto`, gráficos usando `width: "100%"`
- [ ] `EleicaoPage`: `EleicaoDetailPanel` como drawer full-screen ou modal em mobile (< sm)
- [ ] `MonitorGuerraPage` / `MonitorEleitoralPage`: painel lateral empilhado abaixo do mapa em mobile
- [ ] `ChatPage`: uma tela por vez em mobile (lista de conversas OU chat ativo), com botão voltar
- [ ] `BibliotecaPage`: filtros acessíveis em mobile (drawer ou accordion), grid 1 coluna
- [ ] `PaisPage`: seções de gráficos empilhadas verticalmente em mobile
