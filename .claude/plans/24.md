# Plano 24 – Risk Score de Portfólio (M16)

## Objetivo
Permitir ao assinante informar os ativos da sua carteira (tickers + pesos percentuais) e receber um score de risco geopolítico do portfólio calculado em tempo real, com breakdown por categoria (Energia, Alimentos, Câmbio, Geopolítica Militar), alertas preditivos relacionados e exportação como PDF. Ativos desconhecidos são mapeados automaticamente via Claude API e cacheados.

## Contexto de Negócio
O Risk Score é a funcionalidade analítica mais avançada da plataforma — transforma os dados geopolíticos do feed em um número acionável para o gestor. Cada ativo é mapeado para categorias de risco com pesos (ex: PETR4 → energia 0.85, câmbio 0.35); o score de cada categoria é calculado multiplicando esses pesos pelo nível de tensão atual extraído dos eventos ativos das últimas 48h.

Acesso por plano:
| Plano | Acesso |
|---|---|
| Essencial | ❌ — exibe `AddonGate` com CTA de upgrade |
| Pro | ✅ — portfólio salvo, recalcula manualmente |
| Reservado | ✅ — portfólio salvo, recalcula manualmente + automaticamente quando novo alerta é gerado |
| B2B | ✅ — todos os membros da licença |

Categorias de risco e mapeamento do feed:
| Categoria | Tags do feed mapeadas |
|---|---|
| `energy` | energy, oil, gas |
| `food` | food, agriculture, grain |
| `fx` | currency, trade, sanctions |
| `military` | military, war, conflict |

Como o score total é calculado:
1. Tensão por categoria = média ponderada do `impact_score` dos eventos ativos das últimas 48h com aquela categoria, normalizada para 0–100.
2. Score do portfólio por categoria = soma(tensão_categoria × peso_risco_ativo × peso_carteira) / soma(pesos_carteira).
3. Score total = média dos 4 scores de categoria.

## Escopo

### Inclui
- Migration: tabela `mapa_risco_ativos` (id, ticker unique, name, asset_type, risk_weights JSON, regions JSON, notes, updated_at) — pré-populada com 30+ ativos B3 e 10+ internacionais
- Migration: tabela `carteiras` (id, user_id, name, assets JSON, last_score JSON, created_at, updated_at) com RLS — uma carteira por usuário (upsert por user_id)
- Seeder `MapaRiscoAtivosSeeder` — insere os ativos iniciais
- `RiskScoreService` com dois métodos principais:
  - `calcularTensaoPorCategoria()` — busca eventos ativos das últimas 48h, mapeia tags para categorias, normaliza para 0–100
  - `calcularRiscoPortfolio(assets[])` — monta breakdown e score total; para ativos não mapeados chama `mapearAtivoComIA()`
  - `mapearAtivoComIA(tickers[])` — chama Claude API, retorna e cacheia os pesos na `mapa_risco_ativos`
- `CarteirRiscoController` com endpoints:
  - `GET /api/carteira` — retorna carteira salva do assinante (ou null)
  - `POST /api/carteira` — valida soma dos pesos (100% ± 5%), calcula score, salva e retorna
- Middleware de acesso: retorna 403 para plano Essencial (frontend renderiza `AddonGate`)
- Página `RiskScore.tsx` — formulário de ativos (ticker + peso%), botão calcular, score total com cor dinâmica (verde/amarelo/vermelho), breakdown com 4 gauges, alertas preditivos ativos relacionados, botão ExportPdfButton (Plano 22)
- Componente `ScoreGauge.tsx` — barra de progresso + valor numérico + label BAIXO/MÉDIO/ALTO
- Suporte a `type = "risk_score"` no `PdfTemplateService` e template `pdf/risk_score.blade.php` (atualiza Plano 22)
- Adição de "Risk Score" à navegação lateral apenas para Pro, Reservado e B2B
- `AddonGate` para Essencial com CTA de upgrade de plano

### Não inclui
- Recalculo automático por evento (apenas manual via botão — automático fica para fase futura)
- Múltiplas carteiras por usuário
- Histórico de scores anteriores
- Suporte a cotas de fundos ou CDBs
- Importação de carteira por arquivo CSV
