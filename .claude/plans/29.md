# Plano 29 – Multi-Produtos: Frontend Admin UI (Produtos, Addons e Importação)

## Objetivo
Entregar as três páginas/painéis de administração: gerenciamento de produtos, gestão de addons por assinante e importação em lote com CSV/XLS.

## Contexto de Negócio
Com o backend dos Planos 26 e 27 operacional e os serviços do Plano 28 criados, este plano fecha o ciclo com a interface que permite ao admin configurar produtos (links de compra/reativação), gerenciar addons individualmente por assinante e realizar importações em lote.

Detalhes completos em: `/Users/miqueias/.claude/plans/alguns-assinantes-tem-planos-swirling-lamport.md`

## Escopo

### Inclui
- `frontend/src/pages/admin/AdminProdutos.tsx`:
  - Tabela: Nome, Chave, Preço, Links (ícones clicáveis), Status, Ordem, Ações (Editar, Excluir)
  - Modal criar/editar: chave (read-only no edit), nome, descricao, preco_label, link_compra, link_reativar, ativo, ordem
  - Paleta `[#C9B882]` gold do painel admin
- `frontend/src/pages/admin/AdminAssinanteAddons.tsx`:
  - Lista de addons do assinante (addon_key, status, fonte, iniciado_em, expira_em)
  - Modal "+ Adicionar Addon" com select de produtos ativos, status, iniciado_em, expira_em
  - Editar status na linha e remover addon
  - Acessível a partir da página de detalhe do assinante
- `frontend/src/pages/admin/AdminImportarAddons.tsx`:
  - Upload de arquivo (.csv ou .xlsx)
  - Preview das primeiras 5 linhas após seleção
  - Link para baixar template CSV com cabeçalhos corretos
  - Botão "Exportar addons atuais" (chama `exportarAddons()`)
  - Após importar: exibir relatório `{ importados: N, erros: [{linha, motivo}] }`
- Router: adicionar rotas `"produtos"` e `"assinantes/importar-addons"` no admin
- Sidebar admin: adicionar "Produtos" (após "Planos") e "Importar Addons" (seção de assinantes)

### Não inclui
- Lógica de backend — já entregue nos Planos 26 e 27
- Componentes de usuário final — já entregues no Plano 28
