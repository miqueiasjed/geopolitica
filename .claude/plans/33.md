# Plano 33 – Mobile: Admin Pages (Tabelas, Formulários e Grids)

## Objetivo
Tornar todas as páginas do painel admin utilizáveis em mobile (375px+), corrigindo tabelas sem scroll, formulários que transbordam e grids de coluna fixa que quebram o layout.

## Contexto de Negócio
Após o Plano 32 resolver a navegação, as páginas admin ainda terão conteúdo inacessível em telas pequenas. Tabelas com 6-10 colunas, formulários side-by-side e filtros horizontais são os principais problemas. O padrão a adotar é: tabelas com `overflow-x-auto` + `min-w-[600px]` na `<table>`, formulários com `grid-cols-1 md:grid-cols-2`, filtros empilhados em mobile.

## Escopo

### Inclui
**Tabelas com overflow horizontal:**
- `AssinantesPage` — tabela principal com muitas colunas (nome, e-mail, plano, addons, status, ações)
- `UsuariosPage` — tabela de usuários com ações
- `SuportePage` — lista de tickets
- `WebhookEventosPage` — log de eventos (timestamp, tipo, payload, status)
- `WebhookTokensPage` — tabela de tokens
- `EleicaoPage (admin)` — tabela de eleições
- `CrisesPage (admin)` — tabela da linha do tempo
- `BibliotecaPage (admin)` — lista de conteúdos com filtros
- `FontesPage (admin)` — tabela de feeds RSS

**Grids / Cards:**
- `ProdutosPage (admin)` — grid de cards de addons → `grid-cols-1 sm:grid-cols-2`
- `PlanosPage (admin)` — tabela/cards comparativos → cards empilhados em mobile
- `B2BPage (admin)` — lista de empresas → cards responsivos

**Formulários:**
- `NovoConteudoPage` — campos side-by-side → `grid-cols-1 md:grid-cols-2`
- `ConfiguracoesPage` — seções de configuração → empilhar em mobile
- `ImportarAddonsPage` — área de upload + preview → verificar layout

**Filtros:**
- Qualquer barra de filtros horizontal → `flex-wrap` ou `flex-col sm:flex-row`

### Não inclui
- `AdminLayout` (navbar/sidebar) → Plano 32
- Páginas de assinantes → Plano 34
- Painéis laterais e modais compartilhados → Plano 35

## Dependências
- **Requer concluído:** Plano 32 (AdminLayout mobile)
- **Bloqueia:** —

## Entidades/Models envolvidos
Apenas componentes React em `frontend/src/pages/admin/`

## Entregáveis esperados
- [ ] Todas as tabelas com `<div className="overflow-x-auto">` + `<table className="min-w-[560px]">`
- [ ] Grids de cards usando `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
- [ ] Formulários de criação/edição com `grid-cols-1 md:grid-cols-2`
- [ ] Barras de filtro com `flex-wrap gap-2` ou separadas por linha em mobile
- [ ] Botões de ação com texto abreviado em mobile (usar `hidden sm:inline` para rótulos longos)
- [ ] Sem overflow horizontal em nenhuma página admin em 375px
