# Plano 17 – Admin IA v2: Modo de Teste de Prompt (F4)

## Objetivo
Permitir que admins validem o resultado de um prompt editado antes de salvar em produção, executando a IA com uma entrada de amostra diretamente na interface de configuração.

## Contexto de Negócio
Editar um prompt e só descobrir o resultado em produção é arriscado — um prompt mal formulado pode quebrar o parsing JSON ou degradar a qualidade das análises. O teste usa o texto atual do textarea (não o valor salvo no banco) e não afeta nenhum dado de produção.

## Escopo

### Inclui
- **Backend:** endpoint `POST /api/admin/ai/testar-prompt` que recebe `prompt_sistema`, `mensagem_usuario` e `max_tokens` (limitado a 1024 no backend)
- **Backend:** endpoint usa o provider ativo nas configurações; registra no `ai_logs` se já existir (não é bloqueante)
- **Backend:** rate limit de 20 testes/minuto por admin
- **Frontend:** painel colapsável abaixo de cada textarea com link "▶ Testar este prompt"
- **Frontend:** campo de entrada de teste + botão "Executar" + área de resultado (resposta, tempo em ms, tokens)
- **Frontend:** para prompts que exigem JSON, validação automática com indicador verde/vermelho

### Não inclui (fica para outro plano)
- Painel de uso/logs → Plano 18
- Salvar resultados de teste

## Dependências
- **Requer concluído:** Plano 16 (interface de prompts melhorada)
- **Bloqueia:** nenhum (F4 é independente de F3)

## Entidades/Models envolvidos
- `AiProviderInterface` (existente) – usado para executar o teste
- `AiLog` (pode ser criado antecipadamente aqui ou aguardar Plano 18)

## Entregáveis esperados
- [ ] Endpoint `POST /api/admin/ai/testar-prompt` implementado com validação e rate limit
- [ ] Endpoint protegido por middleware admin
- [ ] `max_tokens` limitado a 1024 no backend independente do input
- [ ] Painel colapsável "▶ Testar este prompt" abaixo de cada textarea
