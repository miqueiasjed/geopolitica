# Plano 2 – Autenticação e Controle de Acesso (M00)

## Objetivo
Implementar o sistema completo de autenticação via Laravel Sanctum, controle de acesso por roles com Spatie Permission, e as páginas de login/reset de senha no frontend React.

## Contexto de Negócio
Não há cadastro público — toda conta é criada pelo webhook da Hotmart (M10). O primeiro acesso é via link de definição de senha (Laravel Password Reset). O token Sanctum é armazenado em `localStorage` no SPA. Toda rota protegida verifica `auth:sanctum` + `EnsureAssinanteAtivo` (campo `ativo = true` na tabela `assinantes`). Quatro roles: `assinante_essencial`, `assinante_pro`, `assinante_reservado`, `admin`.

Detalhes completos em: `prd/modulo-00-autenticacao.md`

## Escopo

### Inclui
- Migration e Model `Assinante` (`user_id`, `plano`, `ativo`, `status`, datas)
- `AuthController` (login, logout, me) com Sanctum
- `SenhaController` (esqueci senha, reset com token)
- `PerfilController` (GET/PATCH perfil do usuário autenticado)
- Middleware `EnsureAssinanteAtivo` (bloqueia se `ativo = false`)
- Seeder de roles Spatie: `assinante_essencial`, `assinante_pro`, `assinante_reservado`, `admin`
- Seeder de usuário admin inicial
- Rotas de API: `/api/auth/login`, `/api/auth/logout`, `/api/auth/me`, `/api/auth/senha/*`, `/api/perfil`
- Frontend: `AuthContext`, hook `useAuth`, componente `RotaProtegida`
- Páginas React: `Login.tsx`, `EsqueciSenha.tsx`, `RedefinirSenha.tsx`, `Perfil.tsx`
- Interceptor Axios com header `Authorization: Bearer {token}`

### Não inclui (fica para outro plano)
- Criação automática de usuários via webhook → Plano 3 (M10)
- Dashboard e rotas protegidas dos demais módulos → Planos 4+
