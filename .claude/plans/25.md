# Plano 25 – Admin: Gerenciamento de Planos e Recursos (M17)

## Objetivo
Criar um sistema dinâmico de gerenciamento de planos e recursos no admin, substituindo os valores hardcoded dos limites e permissões por registros em banco de dados editáveis via interface administrativa.

## Contexto
Atualmente os 3 planos (essencial, pro, reservado) e seus limites (chat: 5/20/ilimitado, relatórios: 2/10/ilimitado, acesso à biblioteca, monitors, risk score, etc.) estão hardcoded em constantes dentro dos Services e Controllers. O objetivo é mover isso para banco de dados, criar uma API de admin para CRUD e uma página no admin para gerenciar tudo visualmente.

## Escopo

### Backend
1. **Migration**: tabelas `planos` e `plano_recursos`
2. **Models**: `Plano` e `PlanoRecurso`
3. **PlanoService**: serviço central que lê recursos do banco com cache Redis (TTL 10min)
4. **Seeder**: `PlanoSeeder` popula as 3 planos com todos os recursos existentes
5. **Refatoração**: `ChatService`, `RelatorioIaService`, `CarteiraRiscoController` passam a usar `PlanoService`
6. **API Admin**: endpoints CRUD para planos e recursos (`AdminPlanoController`)

### Frontend
7. **Página Admin**: `AdminPlanos.tsx` — lista planos, edita recursos (toggle on/off e edição de limites numéricos)
