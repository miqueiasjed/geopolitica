# Plano 20 – Webhook Lastlink e E-mail de Boas-vindas do Add-on

## Objetivo
Implementar a integração com o Lastlink como segunda plataforma de pagamento para add-ons avulsos, incluindo webhook de compra/cancelamento, criação automática de conta Essencial para novos compradores, e e-mail de boas-vindas.

## Contexto de Negócio
Os add-ons (`elections` e `war`) podem ser comprados via Lastlink (plataforma independente da Hotmart). Quando alguém compra um add-on no Lastlink:
- Se já tem conta GPI → add-on é ativado na conta existente
- Se não tem conta → conta Essencial é criada automaticamente + e-mail com link de primeiro acesso
- Cancelamento/reembolso no Lastlink → remove o add-on da conta

O padrão de implementação segue exatamente o Webhook Hotmart já existente (Plano 3).

Variáveis de ambiente necessárias:
- `LASTLINK_WEBHOOK_SECRET` — token de validação do webhook
- `LASTLINK_PRODUCT_ELECTIONS` — ID do produto add-on eleitoral no Lastlink
- `LASTLINK_PRODUCT_WAR` — ID do produto add-on guerra no Lastlink

## Escopo

### Inclui
- `WebhookLastlinkController` (POST `/api/webhook/lastlink`) com validação de token via header `x-lastlink-token`
- `LastlinkHandlerService` com handlers de compra aprovada e cancelamento/reembolso
- Detecção de addon_key por product_id via config `addons.lastlink_products`
- Criação automática de conta Essencial (User + Assinante) se comprador não existir
- Delegação ao `AddonService` (Plano 19) para ativar/cancelar
- Log de todos os eventos na tabela `webhook_eventos` existente (fonte: `lastlink`)
- Template de e-mail `AddonBoasVindasEmail` via Resend com dois fluxos: conta nova (com link de redefinição de senha) e conta existente (link direto ao dashboard)
- Entrada na tabela `assinante_addons` em todos os eventos
- Endpoint retorna sempre HTTP 200 para o Lastlink (mesmo em erro interno)
- Variáveis de ambiente documentadas no `.env.example`
- Testes de feature do webhook (compra, cancelamento, comprador novo, comprador existente)

### Não inclui
- Interface frontend de add-ons → Plano 21
- Add-ons via Hotmart → coberto no Plano 19 (`HotmartHandlerService`)
