# Plano 16 – Admin IA v2: Descrições Enriquecidas e Restaurar Padrão (F1 + F2)

## Objetivo
Melhorar a experiência de edição de prompts no painel admin: adicionar metadados ricos em cada prompt (tela afetada, trigger, saída esperada, variáveis) e permitir que o admin restaure o texto padrão de um prompt sem precisar lembrar o original.

## Contexto de Negócio
O painel admin já possui edição de prompts via banco de dados com suporte a Claude e OpenAI. Os prompts atuais têm descrições genéricas que não indicam impacto real no sistema. Admins que editam prompts não têm como reverter ao padrão sem buscar o código-fonte.

## Escopo

### Inclui
- **F1 – Descrições enriquecidas:** atualizar os metadados dos 6 prompts no `ConfiguracaoService` com tela afetada, trigger, formato de saída e variáveis disponíveis
- **F1 – UI:** exibir metadados estruturados na interface (badges de variáveis, label de tela/trigger/saída)
- **F2 – Endpoint de defaults:** `GET /api/admin/configuracoes/defaults` retornando textos padrão de `config/ai.php` (nunca do banco)
- **F2 – Botão "↺ Restaurar padrão":** visível quando o prompt foi personalizado (valor no banco ≠ null), preenche o textarea sem salvar automaticamente
- **F2 – Feedback visual:** borda âmbar no textarea restaurado + tooltip "Clique em Salvar para confirmar"

### Não inclui (fica para outro plano)
- Teste de prompt → Plano 17
- Painel de uso/logs → Plano 18
- Versionamento/histórico de prompts

## Dependências
- **Requer concluído:** Plano 15 (sistema multi-provider e tela admin de configurações já existem)
- **Bloqueia:** nenhum (F1/F2 são independentes de F3/F4)

## Entidades/Models envolvidos
- `Configuracao` (model existente) – sem alteração estrutural
- `ConfiguracaoService` (service existente) – atualizar metadados dos prompts
- `config/ai.php` – fonte dos textos padrão (leitura apenas)

## Entregáveis esperados
- [ ] Metadados dos 6 prompts atualizados no `ConfiguracaoService` (tela, trigger, saída, variáveis)
- [ ] Interface do admin exibe metadados estruturados por prompt (badges de variáveis, labels)
- [ ] Endpoint `GET /api/admin/configuracoes/defaults` implementado e protegido por middleware admin
- [ ] Botão "↺ Restaurar padrão" funcional no frontend (visível apenas quando customizado)
- [ ] Feedback visual (borda âmbar + tooltip) ao restaurar sem salvar
- [ ] Ao salvar após restaurar, o valor no banco volta a null (padrão do config volta a valer)
