# Plano 4 – Feed de Tensões: Backend (M01)

## Objetivo
Implementar o coração do dashboard — o sistema de coleta automática de notícias via RSS, análise com Claude API e persistência dos eventos geopolíticos relevantes para investidores brasileiros.

## Contexto de Negócio
O feed coleta notícias de 8 fontes RSS (Reuters, Al Jazeera, BBC, etc.), filtra com IA (Claude API) usando critérios de relevância para o Brasil (energia, câmbio, alimentos, sanções, etc.) e classifica por impacto numa escala de 1–10. Atualiza a cada hora via scheduler. Deduplicação por `fonte_url`. Análise em lotes de 5 para controlar rate limit da Claude API.

Detalhes completos em: `prd/modulo-01-feed-tensoes.md`

## Escopo

### Inclui
- Migrations: `sources` e `events` (com índices e FULLTEXT)
- Models: `Source` e `Event` com scopes (`ativos`, `ultimas48h`)
- `SourcesSeeder` — 8 fontes RSS iniciais
- `RssFetcherService` — coleta RSS com filtro de 24h e deduplicação
- `AiAnalyzerService` — análise via Claude API (claude-sonnet-4-5) em lotes de 5
- `FeedUpdaterService` — orquestra coleta + análise + persistência
- `ProcessFeedUpdateJob` — job assíncrono (fila: `default`, tries: 3, timeout: 300)
- `UpdateFeedCommand` — comando artisan para forçar atualização
- `FeedController` — GET com filtros, paginação cursor-based; POST atualizar (admin)
- `FeedFilterRequest` — validação de filtros (categoria, regiao, label, cursor, limite)
- `EventResource` e `EventCollection` — transformação de resposta
- Scheduler: job a cada hora com `withoutOverlapping()`
- Controle de acesso por role: `essencial` → máx 20 eventos/48h; `reservado` → histórico completo
- Error handling global: 401/403 padronizados no `Handler.php`
