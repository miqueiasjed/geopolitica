# Plano 5 – Feed de Tensões: Frontend (M01)

## Objetivo
Construir o dashboard principal do feed de tensões — layout escuro estilo Bloomberg Terminal, com cards de eventos, filtros, scroll infinito e atualização automática a cada 5 minutos.

## Contexto de Negócio
O assinante abre o dashboard e em 30 segundos sabe quais eventos geopolíticos importam para o Brasil. Visual denso e escuro (`#0a0a0b`), tipografia monospace para dados. Cores de impacto: CRÍTICO (vermelho `#EF4444`), ALTO (laranja `#FB923C`), MÉDIO (amarelo `#FACC15`), MONITORAR (azul `#60A5FA`). Cor dourada `#C9B882` como destaque da marca.

Detalhes completos em: `prd/modulo-01-feed-tensoes.md` (seção 7)

## Escopo

### Inclui
- Serviço `api.ts` — instância Axios com interceptor Bearer token
- Types: `feed.ts` (interfaces TypeScript: `Event`, `FeedFilters`, etc.)
- Hook `useFeed.ts` — React Query com paginação cursor, `refetchInterval: 5min`
- Hook `useAuth.ts` — contexto de autenticação (token, user, logout)
- Componente `EventCard.tsx` — score, badge, título (link), análise IA, metadados
- Componente `ImpactBadge.tsx` — badge colorido por nível de impacto
- Componente `EventList.tsx` — lista com scroll infinito (IntersectionObserver)
- Componente `FilterBar.tsx` — filtros: Todos | Energia | Alimentos | Câmbio | Conflitos | Sanções | Eleições | Commodities
- Componentes de layout: `DashboardLayout.tsx`, `TopNav.tsx`
- Componentes utilitários: `LoadingSpinner.tsx`, `EmptyState.tsx`
- Página `Feed.tsx` — página principal `/dashboard/feed`
- Página `Login.tsx` — formulário de login (se não criado no Plano 2)
- Configuração React Router v7 com rota protegida

### Não inclui (fica para outro plano)
- Barra de indicadores de risco → Plano 9 (M04)
- Navegação para Mapa, Biblioteca → adicionada em cada plano respectivo
