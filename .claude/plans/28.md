# Plano 28 – Multi-Produtos: Frontend Core (Tipos, Serviços, Hooks e Componentes)

## Objetivo
Conectar o frontend ao novo endpoint `/api/meus-produtos`, adaptar o hook `useAddonAccess` e reescrever `AddonGate` para exibir CTA dinâmico por estado do addon (nunca comprou / cancelado / expirado / reembolsado).

## Contexto de Negócio
Hoje `AddonGate.tsx` e `useAddonAccess.ts` usam dados hardcoded (links fixos, sem distinção de estado). Com o backend do Plano 26 entregando `status_usuario` e links dinâmicos, o frontend precisa ser atualizado para refletir essas informações e mostrar CTAs corretos por estado.

Detalhes completos em: `/Users/miqueias/.claude/plans/alguns-assinantes-tem-planos-swirling-lamport.md`
